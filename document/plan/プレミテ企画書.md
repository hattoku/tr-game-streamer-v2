## 0. プレミテの成長戦略

### 0.1 二段階成長モデルの全体像

プレミテは、**立ち上げ初期から段階的に成長する二段階モデル**を採用する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  【初期フェーズ】                      【将来フェーズ】
  視聴記録＆ライフログ                    再生リスト格付けサイト

  「ゲーム実況動画の                →    「本当に面白い実況シリーズに
   再生リスト視聴記録                      出会える場」
   ＆新着通知サービス」

  コアバリュー：                          コアバリュー：
  ・視聴進捗の管理                         ・信頼できるレビューの集積
  ・新着動画の追跡                         ・コミュニティによる格付け
  ・ライフログとしての記録                   ・人間による本物の評価

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

この二段階モデルは、**データ成熟度に応じた自然な進化**を想定している。初期フェーズでは「視聴記録」という日常的な行為からユーザーを獲得し、データが蓄積されることで、格付けサイトとして機能できる状態に到達する。

### 0.2 初期フェーズ：「視聴記録＆ライフログ」の時代

**キャッチコピー**：「ゲーム実況動画の再生リスト視聴記録＆新着通知サービス」

立ち上げ直後はレビューの絶対数が少なく、格付けサイトとしての機能は未成熟である。この時期は、**ユーザーの視聴ライフログを蓄積することに主眼を置く**。

初期フェーズで前面に出す価値：
- **どこまで見たかの記録**: シリーズの視聴進捗を「ライフログ」として管理
- **新着通知**: マイリストに登録した再生リストに新しい動画が追加されたら通知
- **並走視聴の効率化**: 複数シリーズをプレミテ一か所で管理

レビュー・スコアリング機能は初期フェーズから実装されているが、データが少ない段階ではUIのメッセージングで積極的に前面には出さない。

### 0.3 将来フェーズ：「再生リスト格付けサイト」への転換

データが成熟した段階で、プレミテは**「信頼できるレビューを通じて本当に面白い実況シリーズと出会える場」**としての訴求に主軸を移す。

将来フェーズで前面に出す価値：
- **信頼度スコアリングによる格付け**: 視聴深度で重み付けされた質の高いレビュー群
- **コミュニティによる評価の可視化**: 「参考になった」数で浮かび上がる良質なレビュー
- **AI時代における人間の評価の希少性**: AIが生成できない、実際に視聴した人間の声

### 0.4 転換トリガー

初期フェーズから将来フェーズへの転換は、以下の**複合指標**と**補助的確認指標**の両方が満たされた段階で開始する。

#### メイン転換トリガー（複合指標）

以下の2条件を**同時に**満たした時点で転換を開始する：

| 条件 | 閾値 |
|------|------|
| コメント付きレビュー投稿数 | 500件以上 |
| レビュー投稿者のうち複数回投稿したユーザーの割合 | 30%以上 |

**なぜこの指標か**：単なるレビュー数ではなく「コメント付き」に限定することで品質を担保する。加えて、リピート投稿者の割合がコミュニティの継続性を示し、「格付けサイト」に転換した後も評価が育ち続けることを保証する。

#### 補助的確認指標（比率トリガー）

メイン転換トリガーの達成後、以下の指標でデータ成熟度を確認する：

| 条件 | 閾値 |
|------|------|
| 再生リスト登録数 | 1,000件以上 |
| レビュー投稿数 ÷ 再生リスト登録数 | 20%以上 |

**なぜこの指標か**：登録されたコンテンツに対してどの程度の割合で評価が付いているかを相対的に計測することで、データの成熟度を検証する。

## 1. プレミテの本質的なミッション

### 1.1 初期フェーズのミッションステートメント

**「ゲーム実況の視聴をもっと豊かにする、ライフログ＆新着追跡サービス」**

好きな実況シリーズをマイリストに登録し、どこまで見たかを記録する。新しい動画がアップされたらすぐに分かる。複数のシリーズを並走で追いかけながら、視聴体験そのものを楽しむ。プレミテは、そのための場所である。

### 1.2 将来フェーズのミッションステートメント

**「本当に面白い実況シリーズに出会える場を作ること」**

プレミテは、単なる視聴記録ツールでもなく、実況者の宣伝プラットフォームでもない。**真の視聴者の声が評価に反映されたレビューを通じて、ユーザーが本当に価値あるゲーム実況シリーズと出会える場**を提供する。

### 1.3 なぜプレミテが必要なのか

#### YouTubeの構造的な限界

YouTubeは**「囲い込みの最大化」を目的としたマーケティング**に最適化されている：

- **サジェスト・レコメンデーション**: アルゴリズムが「視聴時間を最大化する動画」を優先
- **エンゲージメント最適化**: 視聴者の継続的な訪問を促す設計
- **シリーズ全体への評価ができない**: ユーザーが「いいね」できる単位が個別動画であり、シリーズ全体の質を評価する仕組みがない

結果として、**本当の視聴者の評価とYouTubeのランキング・レコメンドに乖離**が生じる。

#### プレミテの価値提案

プレミテは以下を実現する：

1. **シリーズ単位の評価**: 複数話を視聴したユーザーの信頼できる評価
1. **視聴深度に基づく信頼度**: 全話見た人のレビューと、数話で辞めた人のレビューを差別化
1. **コミュニティの声の可視化**: 参考になったボタンにより、評判の良いレビューがより影響力を持つ
1. **YouTube APIへの依存を避ける**: YouTubeの再生数などに頼らず、自ユーザーの動線から信頼できる情報を創出

### 1.4 AI生成時代における役割

**AI生成コンテンツが氾濫する近い未来において、プレミテは「人間による本当の評価」が最後の砦となる**。

- 誰が作ったのか、何のためか不透明なAI生成コンテンツ
- YouTubeのアルゴリズムに流されやすい低品質コンテンツ
- こうした濁流の中で、**実際に視聴して、評価した人間の声**がますます希少で価値を持つようになる

プレミテは、AI時代における**「人間による質の証明」の無二の存在**になる。この価値は初期フェーズから積み上げられ、将来フェーズで一気に開花する。

## 2. ターゲットユーザーと使用コンテキスト

### 2.1 ターゲットペルソナ

#### 主要ペルソナ：「複数シリーズ並走視聴者」

**基本属性**

- 複数のゲーム実況シリーズを同時に追いかけている
- Google検索で「ゲーム名 + 実況」というキーワードで検索してきた
- 日本語での検索・利用が基本

**行動パターン**

- 新しいゲーム実況を見つけたら、プレミテで関連する再生リストを検索
- 気になった再生リストをマイリストに追加
- 複数のゲーム実況を「並走」で追いかける
- 作業しながら「ラジオ感覚」で視聴する（バックグラウンド視聴）
- シリーズの完結まで、継続的に視聴

**動機**

- ゲームの攻略情報を得たい
- 実況者の反応や人間ドラマを楽しみたい
- 複数のシリーズを効率的に管理したい
- YouTube上で埋もれたコンテンツを発見したい

### 2.2 使用シーン

| シーン | 詳細 |
|--------|------|
| **発見シーン** | Google検索で「ゲーム名 + 実況」を検索 → プレミテで関連する再生リストを一覧表示 |
| **追跡シーン** | マイリストに複数のシリーズを登録 → 新着通知で更新をキャッチ → 各シリーズを並走で視聴 |
| **視聴シーン** | 作業しながらプレミテ上で再生 → 自動遷移で次の話を継続再生 |
| **記録シーン** | どこまで見たかが自動記録 → ライフログとして視聴歴が蓄積される |
| **評価シーン** | シリーズ完結後、または途中で、レビュー・スコアを投稿 |
| **共有シーン** | 推しのシリーズをまとめ機能で整理 → LINEで友人にシェア |

## 3. コアアーキテクチャ

### 3.1 初期フェーズで重視する機能

#### 3.1.1 YouTube埋め込みプレーヤー（自動遷移型）

プレミテ上に埋め込まれたYouTubeプレーヤーは、シリーズ全体を「継続的に視聴できる」設計：

- **再生リスト内の動画を埋め込み再生**
- **一話終了時に自動的に次の話に遷移**
- **ユーザーが操作しなくても再生を継続**
- **作業しながらのラジオ感覚視聴を実現**

技術的な利点として、YouTubeを開かずにプレミテ内で視聴が完結し、視聴進捗情報をプレミテ側で正確に取得できる。

記録仕様は「最後に開いた話数を記録」するシンプルな設計とする。細かい記録ロジックより「最後に開いた話数を信頼する」ことで複雑性を排除し、複数レビュワーの統計的集約により個別のノイズは自動的に吸収される。

#### 3.1.2 マイリスト機能

ユーザーが「追いかけている再生リスト」を一元管理する機能：

- **複数の再生リストをマイリストに登録**
- **マイリスト画面で全シリーズの進捗を一覧表示**
- **各シリーズの新着動画へすぐにアクセス可能**
- **複数シリーズの「並走視聴」を効率化**

初期フェーズでのマイリストは「視聴ライフログの管理ダッシュボード」として機能する。

#### 3.1.3 新着通知機能

マイリストに登録した再生リストに新しい動画が追加された際に通知する機能：

- 通知対象：マイリスト登録済みの再生リストに新しい動画がアップされた時
- 通知方法：プレミテ内の通知（ベルアイコン）およびNEWバッジ表示

この機能は、ユーザーが「プレミテを毎日開く理由」を作る初期フェーズの重要な役割を担う。

### 3.2 全フェーズ共通の基礎機能

#### レビュー・スコアリング機能

プレミテの将来的なコア価値であり、初期フェーズから実装する。ただし、レビューが少ない段階ではUI/メッセージングで前面には出さない。

| 要素 | 形式 | 詳細 |
|------|------|------|
| **スコア** | 5段階評価 | 1～5の数値評価 |
| **コメント** | テキスト | レビュー理由・感想（任意） |

スコアのみでも投稿可能だが、コメント付きが推奨（信頼度スコアに影響する）。

#### タグシステム（ニコニコ動画方式）

再生リストにタグを付与することで、関連する実況シリーズへの「回遊」を実現：

| 主体 | 権限 | 詳細 |
|------|------|------|
| **再生リスト登録者** | 初期タグを設定 | シリーズの意図を反映 |
| **視聴者コミュニティ** | タグの追加・上書き | 実際の視聴者による評価を反映 |
| **登録者の選択肢** | 編集ロック機能 | 重要なシリーズはロック可能 |

### 3.3 将来フェーズで段階的に充実させる機能

#### 信頼度スコアリングの本格運用

転換トリガー達成後、レビューの蓄積に応じて信頼度スコアリングを前面に出した訴求に移行する。詳細な仕様・計算式は「プレミテ 信頼度スコアリングシステム仕様書」を参照。

信頼度スコアは以下の4要素で構成される：

| 要素 | 概要 |
|------|------|
| **視聴深度** | どこまで視聴したか。完走ステータスも加味する |
| **参考になった数** | コミュニティから評価されているレビューを優遇する |
| **コメント有無** | 理由を書いたレビューを優遇する |
| **ジャンル専門性** | そのゲームのジャンルに詳しいレビュワーを優遇する |

シリーズスコアは全レビューの信頼度加重平均によって算出し、スケールは5点満点とする。

```
シリーズスコア = Σ(各レビューの星評価 × 信頼度スコア) / Σ(信頼度スコア)
```

#### 推し応援機能（将来フェーズで追加）

ユーザーが「推しの再生リスト」をサポートする機能。推しのランキングを上げるためにレビューを投稿して応援する仕組みを提供する。「推しをサポートしたい」という動機からレビュー投稿を促し、ランキングへの貢献を通じて達成感を提供する。

## 4. ユーザー動機づけ設計

### 4.1 問題設定

重要な問い：

> **「なぜユーザーはレビューやスコアを投稿するのか？」**

MyAnimeListが成功したのは、単に「評価システム」があるからではなく、ユーザーが「自分の好きなコンテンツを他者と共有したい」「コミュニティと繋がりたい」という心理が満たされるからである。

プレミテも同様に、**ユーザーの自然な動機を引き出す仕組み**が必須。初期フェーズでは特に、「レビューを書く」という行為のハードルを下げ、視聴記録の延長として自然な評価投稿を促す設計が重要となる。

### 4.2 まとめ機能（友人シェア型）——初期フェーズより実装

ユーザーが**複数の再生リストを「まとめ」として組織し、自分のレビューコメントと一緒に友人にシェアできる機能**。

```
ユーザーが「推しゲーム実況まとめ」を作成：

【推しゲーム実況まとめ】
  ├─ ゲームA「〇〇実況」
  │  └─ [自分のコメント] 「本当に面白い。実況者の反応が最高」
  │
  ├─ ゲームB「△△実況」
  │  └─ [自分のコメント] 「ゲームの難易度が高いけど戦略の面白さが伝わってくる」
  │
  └─ ゲームC「□□実況」
     └─ [自分のコメント] 「この実況者は本当にゲームが上手。見てて惚れ惚れする」

↓ LINEで友人にシェア
```

**なぜこれが効果的か**：趣味嗜好がよく出るまとめを「身近な友人に教える」という自然な行動がトリガーとなり、「友人に推すのに、自分のレビューコメントは必須」という流れでレビュー投稿への動機が生まれる。

> **「好きな実況を友人に勧めたい」→ 自分のコメントを付けたい → レビュー投稿へ**

### 4.3 推し応援機能——将来フェーズで追加

将来フェーズで追加する。「推しをサポートしたい」という動機でレビューを投稿し、ランキングが上がることで達成感が生まれる仕組み。詳細は将来フェーズへの移行時に仕様を策定する。

## 5. コンテンツ発見メカニズム

### 5.1 発見フロー

```
ユーザーのカスタマージャーニー：

1. Google検索で「ゲーム名 + 実況」を検索
   ↓
2. プレミテでそのゲームの再生リストを発見
   ↓
3. 「まずはこれを見てみよう」と選択、マイリストに追加
   ↓
4. プレミテ上で再生、自動遷移で継続視聴
   ↓
5. シリーズを見ている途中で、タグ経由で関連する別実況を発見
   ↓
6. その別実況もマイリストに追加、並走視聴開始
```

### 5.2 タグの威力

- **YouTube内で埋もれたコンテンツを可視化**
- **同じジャンル・テイストの実況への自然な回遊**
- **ユーザーの「続きを見たい」という欲求を満たす**

## 6. スケーリング戦略

### 6.1 基本方針：日本市場への特化

プレミテは**日本の利用者に特化した、日本語統一のプラットフォーム**を目指す。

**なぜ日本市場か**：日本の実況文化の成熟度、YouTube日本版での「ゲーム名 + 実況」検索の多さ、MyAnimeListの成功事例、そしてグローバル大手プラットフォームが対応していないニッチという競争環境が揃っている。

日本語統一により、ユーザーインターフェイスからタグ・レビューコメントまで品質の高い情報が蓄積され、コミュニティの一体感が生まれやすい。

### 6.2 長期的なビジョン

**段階1（初期フェーズ）**：日本市場での基盤構築

- 「視聴記録＆新着通知サービス」として日本ユーザーの信頼を獲得
- 高品質な実況シリーズのカタログ化
- レビュー・コメントデータの蓄積

**段階2（転換フェーズ）**：格付けサイトへのシフト

- 転換トリガー達成を確認次第、UI/メッセージングを「格付けサイト」に軸を移す
- 信頼度スコアリングを全面訴求
- コミュニティの評価力を前面に出したプロモーション

**段階3（将来フェーズ）**：プラットフォームとしての確立

- 推し応援機能など、コミュニティ機能の拡充
- 将来的な多言語対応の検討（当面は日本語に集中）

**段階4（モバイルフェーズ）**：iOS / Androidネイティブアプリへの展開

- iOS・Android向けネイティブアプリのリリース
- モバイルアプリの利用には**会員登録が必須**（ゲスト利用不可）
- ゲストユーザーはWebのみ利用可能。アプリはログイン済みユーザー専用の体験として設計する
- プレミテは動画プレーヤーとしての側面が強く、スマートフォンでの「ながら視聴」ニーズに応えるため、モバイルアプリへの展開は自然な進化の到達点と位置づける

**重要な方針**：「急いで拡大するより、まず日本で『本物』を作る」

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

### 実現されるユーザー体験

| 段階 | 体験 |
|------|------|
| **1. 発見** | Google検索で「ゲーム名 + 実況」を検索 → プレミテで再生リストを発見 |
| **2. 登録** | 気に入った再生リストをマイリストに追加。視聴進捗が自動記録される |
| **3. 追跡** | 新着通知でシリーズの更新をキャッチ。見逃しゼロで追い続ける |
| **4. 視聴** | プレミテ上で自動遷移プレーヤーで継続視聴。作業しながらラジオ感覚で楽しむ |
| **5. 並走** | タグ経由で関連する別の実況を発見 → 複数シリーズを並走で追いかける |
| **6. 評価** | シリーズ完結後、レビュー・スコアを投稿 → コミュニティの一部になる |
| **7. 共有** | 推しの実況をまとめ機能で整理 → LINEで友人にシェア → 友人も参加 |

### 成功指標

| 指標 | 意味 |
|------|------|
| **マイリスト登録数** | ユーザーが「追いかけたい」と思った実況シリーズの数 |
| **新着通知の開封率** | 初期フェーズの重要指標。ユーザーがプレミテを日常的に開いているか |
| **レビュー投稿数** | ユーザーが「共有したい」「応援したい」と思った回数 |
| **コメント付きレビュー数** | 格付けサイトへの転換判断に直結する最重要指標 |
| **リピートユーザー率** | 複数回マイリスト登録・レビュー投稿するユーザーの割合 |
| **レビュー投稿率** | 再生リスト登録数に対するレビュー投稿数の割合。転換トリガーの補助指標 |

## 7. 運営体制

### 7.1 基本方針

プレミテは**AIを主軸とした運営体制**を採用する。

技術の進化を背景に、当初想定していた人間のアルバイト・ボランティア中心の運営から方針を転換した。AI運営者を配置し、日常的な運営業務をAIが自律的に担う。将来的に志を同じくする人間が加わる可能性は排除しないが、基本はAIが運営の主体となる。

運営者は**プラットフォーム全体で1名**を配置する。初期状態では全ジャンル（RPG・シミュレーション・格闘アクション・アドベンチャー・シューティング・スポーツレース・テーブルパズル・教育学習・その他）の審査を、この1名のAI運営者が担当する。

| 役割 | 担当 |
|------|------|
| **オーナー** | プラットフォーム全体の最終責任者。STEP 2審査・方針決定を担う |
| **AI運営者** | プラットフォーム全体の日常運営を自律的に担うAI。初期は全ジャンルを担当 |
| **人間の運営者**（任意） | 将来的に加わる可能性がある。担当ジャンルを持ち、AI運営者と並走する |

### 7.3 AI運営者の理想的な役割

AI運営者が自律的に担うことを理想とする業務は以下の通り。

**審査業務**

ユーザーから提出された再生リスト登録提案・ゲームタイトル編集提案や通報（再生リスト・レビューコメント）に対して一次スクリーニング（STEP 1）を実施する。スパム・荒らしの検出、更新停止の判定、完結判定などを担い、判定結果をコメントとして記録する。将来的にはSTEP 1の承認判断そのものもAIが自律的に完結できるよう、審査権限の拡張を予定している。

**コンテンツ追加業務（理想）**

担当ジャンルの再生リストおよびゲームタイトルを自律的に発見・登録する。ユーザーからの提案を待つだけでなく、AI運営者自身がコンテンツを能動的に発掘することで、プラットフォームのコンテンツ充実を加速させる。

### 7.4 オーナーの関与

オーナーはすべての審査ワークフローの最終承認者（STEP 2）として機能する。AI運営者が自律的に運営できる範囲が広がるほど、オーナーの関与は最終判断に集中できる設計を目指す。

## 終わりに

プレミテは、まず「視聴記録＆新着通知サービス」として多くのゲーム実況ファンの日常に入り込む。マイリストに登録し、進捗を記録し、新着を追いかける——その積み重ねが、ゲームコメントの文化を育てる土台となる。

データが成熟した先に、プレミテは「格付けサイト」へと進化する。YouTubeの「囲い込み」から解放され、人間が実際に視聴して形成した信頼できるレビューが、本当に面白い実況シリーズとの出会いを生む。

AI生成コンテンツが氾濫する近い未来において、**「人間による本当の評価」が最後の砦**となる時代が必ず来る。その時に、プレミテが「信頼できるガイド」として機能することを確信している。
