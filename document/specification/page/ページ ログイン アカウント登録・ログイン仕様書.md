## 1. 概要

ゲムコメはアカウントなしでも利用できるが、アカウントを作成することで視聴管理・レビュー投稿などの機能が利用可能になる。アカウント作成は無料。

認証基盤にはFirebase Authenticationを使用する。

---

## 2. アカウントなし／ありでできること

| 機能 | アカウントなし | アカウントあり |
|------|:---:|:---:|
| 再生リストの閲覧・検索 | ○ | ○ |
| ゲームタイトル詳細ページの閲覧 | ○ | ○ |
| チャンネル詳細ページの閲覧 | ○ | ○ |
| 動画プレーヤーの視聴 | ○ | ○ |
| 視聴履歴の記録 | ○（LocalStorage・端末内のみ） | ○（DB保存・端末横断） |
| マイリストの管理 | × | ○ |
| 視聴ステータスの設定 | × | ○ |
| レビュー（星評価・コメント）の投稿 | × | ○ |
| 再生リストの提案 | × | ○ |
| プロフィールページの公開 | × | ○ |

## 3. ログイン要求モーダル

未ログインユーザーがアカウントと紐づく操作を行おうとした場合、ログイン要求モーダルを表示してアカウント作成またはログインへ誘導する。

### 3.1 表示トリガー

| トリガー操作 | 備考 |
|------------|------|
| 星評価をクリックしようとした | レビュー投稿機能仕様書に準ずる |
| 視聴ステータスを選択しようとした | レビュー投稿機能仕様書に準ずる |
| 「レビューを投稿する」ボタンを押した | レビュー投稿機能仕様書に準ずる |
| マイリストに追加しようとした | マイリスト機能仕様書に準ずる |
| 再生リストの提案ページにアクセスした | ログインページへリダイレクト |

### 3.2 モーダルの構成要素

| 項目 | 仕様 |
|------|------|
| タイトル | 操作に応じて変化する（例：「レビューを投稿するにはアカウントが必要です」） |
| 本文 | 「ゲムコメのアカウントを作成すると、視聴記録の管理やレビューの投稿ができます。」 |
| CTA（主） | 「アカウントを作成する」ボタン → アカウント作成フォームへ遷移 |
| CTA（副） | 「ログイン」リンク → ログインページへ遷移 |
| 閉じる | モーダル外クリックまたは「×」ボタンで閉じる |

## 4. ログイン・アカウント作成への導線

### 4.1 ヘッダーメニューからの導線

ヘッダーに「ログイン」ボタンを常時表示する。クリックするとログインページへ遷移する。ログインページにはアカウント作成フォームへの切り替えリンクを設ける。

### 4.2 操作起点からの導線

以下の操作を未ログインユーザーが行おうとした場合、[第3章](#3-ログイン要求モーダル)のモーダルを表示してアカウント作成またはログインへ誘導する。

- 星レビューをつけようとした
- 視聴ステータスを入力しようとした
- マイリストに追加しようとした

### 4.3 招待リンクからの導線

既存ユーザーが発行した招待リンクをクリックすると、アカウント作成フォームに直接遷移する（[第11章](#11-招待リンク)参照）。

## 5. ログインフォーム仕様

### 5.1 フォーム構成要素

| 要素 | 仕様 |
|------|------|
| メールアドレス入力フィールド | テキスト入力。メールアドレス形式のバリデーションあり |
| パスワード入力フィールド | マスク表示。表示／非表示トグルボタンを設置 |
| ログイン状態を維持するチェックボックス | デフォルトOFF（詳細は5.2節参照） |
| ログインボタン | 「ログインする」 |
| パスワードを忘れた方はこちら | クリックでパスワード再発行フローへ遷移（[第9章](#9-パスワード再発行)参照） |
| アカウント作成リンク | 「アカウントをお持ちでない方はこちら」→ アカウント作成フォームへ遷移 |

### 5.2 ログイン状態を維持するチェックボックス

| 項目 | 仕様 |
|------|------|
| デフォルト | OFF |
| OFF時 | セッション有効期限：24時間 |
| ON時 | セッション有効期限：7日間 |

### 5.3 ログインエラー

| エラーケース | 表示メッセージ |
|------------|-------------|
| メールアドレス・パスワードの組み合わせが不正 | 「メールアドレスまたはパスワードが正しくありません」 |
| 入力フィールドが空 | 各フィールドに「入力してください」のインラインエラーを表示 |
| メールアドレスの形式が不正 | 「正しいメールアドレスの形式で入力してください」 |

### 5.4 ログイン成功後の遷移

ログイン成功後のリダイレクト先は、Firebase Authenticationで付与されたロールによって分岐する。

| ロール | 遷移先 |
|--------|--------|
| 一般ユーザー | TOPページ（`/`） |
| オーナー・運営者（管理者） | 管理画面ダッシュボード（`/admin`） |

> ログインフォームは一般ユーザーと管理者で共通（`/login`）。ロール判定はFirebase Authのカスタムクレームをもとに行う。

## 6. ソーシャルログイン

ログインページおよびアカウント作成フォームに、以下のソーシャルログインボタンを設置する。

| プロバイダ | 選定理由 |
|-----------|---------|
| Google | YouTubeとの親和性が高く、ゲームコンテンツの視聴者層との相性が良い |
| X（旧Twitter） | 国内ゲーマー層の利用率が高く、ゲームコミュニティとの接続性が高い |

### 6.1 ソーシャルログインのフロー

```
ソーシャルログインボタンをクリック
    ↓
各プロバイダの認証画面へ遷移
    ↓
認証成功
    ↓
    ├─ 初回ログインの場合 → アカウント新規作成 → 初期設定フローへ（第10章参照）
    └─ 2回目以降の場合 → ログイン完了 → 元のページへ戻る
```

### 6.2 注意事項

- ソーシャルログインで作成したアカウントには初期状態でパスワードが存在しない。後から設定ページでパスワードを設定できる。
- 同一メールアドレスで異なるプロバイダを使用した場合の挙動はFirebase Authenticationの仕様に準ずる。

## 7. パスキーログイン

### 7.1 概要

パスキーは、デバイスの生体認証（指紋・顔認証）や画面ロック（PIN等）を使ってパスワードなしでログインできる仕組みである。メールアドレス＋パスワードでアカウントを作成した後、設定ページから追加で登録できる。

### 7.2 登録方法

設定ページ（`/settings`）にパスキー登録セクションを設け、ユーザーが任意のタイミングで登録できる。初期設定フローには案内を含めない。

| 項目 | 仕様 |
|------|------|
| 登録場所 | 設定ページ（`/settings`）> セキュリティセクション |
| 登録条件 | メールアドレス＋パスワードでログイン済みのユーザー |
| 複数登録 | 複数デバイス分のパスキーを登録可能 |
| 削除 | 設定ページから登録済みパスキーを個別に削除可能 |

### 7.3 パスキーでのログインフロー

```
ログインページで「パスキーでログイン」をクリック
    ↓
デバイスの生体認証または画面ロックで認証
    ↓
認証成功 → ログイン完了 → 元のページへ戻る
```

## 8. アカウント作成フロー（メール認証）

### 8.0 ステップインジケーター

アカウント作成から初期設定完了までの全画面に、現在の工程位置を示すステップインジケーターを表示する。

#### 工程構成

メール認証経由とソーシャルログイン経由で工程数が異なるため、それぞれ別のインジケーターを表示する。

**メール認証経由（全6工程）**

```
① アカウント情報入力 → ② メール確認 → ③ プロフィール設定 → ④ ジャンル選択 → ⑤ タイトル選択 → ⑥ マイリスト追加
```

**ソーシャルログイン経由（全5工程）**

```
① 連携確認 → ② プロフィール設定 → ③ ジャンル選択 → ④ タイトル選択 → ⑤ マイリスト追加
```

#### 表示仕様

| 項目 | 仕様 |
|------|------|
| 表示位置 | 各画面の上部に固定表示 |
| 表示形式 | 丸数字またはドットを横並びにし、完了済みステップ・現在のステップ・未完了ステップを色で区別する |
| 現在のステップ | ブランドカラー（#E53935）でハイライト表示 |
| 完了済みステップ | チェックマーク付きでグレー表示 |
| 未完了ステップ | グレー表示 |
| スキップ時の挙動 | Step 2（ジャンル選択）以降をスキップした場合、スキップしたステップはグレーのまま完了画面（Step 6 または Step 5）へ遷移する |

### 8.1 フォーム構成要素

| 要素 | 仕様 |
|------|------|
| メールアドレス入力フィールド | テキスト入力。メールアドレス形式のバリデーションあり |
| パスワード入力フィールド | マスク表示。表示／非表示トグルボタンを設置 |
| パスワード確認入力フィールド | 同上。パスワードフィールドと一致しない場合はエラー表示 |
| 年齢確認チェックボックス | 「私は13歳以上です」。必須。未チェックの場合は送信ボタンを非活性にする |
| アカウントを作成するボタン | 「アカウントを作成する」 |
| ログインリンク | 「すでにアカウントをお持ちの方はこちら」→ ログインページへ遷移 |

※ ユーザー名はこの時点では入力しない。アカウント作成後の初期設定フローで入力する（[第10章](#10-初期設定フロー)参照）。
※ 生年月日は収集しない。年齢確認はユーザーの自己申告チェックボックスのみとする。

### 8.2 バリデーション

| チェック内容 | エラー時の表示 |
|------------|-------------|
| メールアドレスの形式が不正 | 「正しいメールアドレスの形式で入力してください」 |
| メールアドレスがすでに登録済み | 「このメールアドレスはすでに登録されています」 |
| パスワードが8文字未満 | 「パスワードは8文字以上で入力してください」 |
| パスワード確認が一致しない | 「パスワードが一致しません」 |
| 年齢確認チェックボックスが未チェック | 送信ボタンを非活性のまま維持する（エラーメッセージは表示しない） |

### 8.3 アカウント作成フロー

```
メールアドレス・パスワードを入力して「アカウントを作成する」をクリック
    ↓
バリデーション
    ↓
仮登録メールを送信
    ↓
「確認メールを送信しました」画面を表示
    ↓
ユーザーがメール内のリンクをクリック
    ↓
本登録完了
    ↓
初期設定フローへ（第10章参照）
```

### 8.4 仮登録メール

| 項目 | 仕様 |
|------|------|
| 送信タイミング | アカウント作成フォーム送信直後 |
| リンクの有効期限 | 24時間 |
| 有効期限切れ時 | リンクをクリックした際に「リンクの有効期限が切れています」の旨を表示し、再送信を促す |

#### サンプル

```
件名：【ゲムコメ】メールアドレスの確認をお願いします

ゲムコメへのご登録ありがとうございます。

以下のボタンをクリックして、メールアドレスの確認を完了してください。

[ メールアドレスを確認する ]
https://gamecome.jp/verify?token=xxxxxx

このリンクの有効期限は24時間です。

心当たりのない場合は、このメールを無視していただいてかまいません。
あなたのアカウントは引き続き安全に保護されています。

─────────────────────────
ゲムコメ https://gamecome.jp
このメールは送信専用です。返信はできません。
─────────────────────────
```

### 8.5 仮登録メールの再送信

「確認メールを送信しました」画面に「メールが届かない場合は再送信する」リンクを設置する。クリックすると同一メールアドレス宛に仮登録メールを再送信する。再送信後はボタンを非活性にし、一定時間経過後に再度押せるようにする（連続送信防止）。

### 8.6 「確認メールを送信しました」画面の構成要素

| 要素 | 内容 |
|------|------|
| メッセージ | 「[入力したメールアドレス] に確認メールを送信しました。メール内のリンクをクリックして登録を完了してください。」 |
| 補足 | 「メールが届かない場合は、迷惑メールフォルダもご確認ください。」 |
| 再送信リンク | 「メールが届かない場合は再送信する」 |

## 9. パスワード再発行

### 9.1 フロー

```
ログインページの「パスワードをお忘れの方はこちら」をクリック
    ↓
パスワード再発行フォームへ遷移
    ↓
登録済みメールアドレスを入力して送信
    ↓
パスワード再設定メールを送信
    ↓
「パスワード再設定メールを送信しました」画面を表示
    ↓
ユーザーがメール内のリンクをクリック
    ↓
新しいパスワードを入力する画面へ遷移
    ↓
パスワード再設定完了 → ログインページへ遷移
```

### 9.2 パスワード再発行フォームの仕様

| 要素 | 仕様 |
|------|------|
| メールアドレス入力フィールド | テキスト入力 |
| 送信ボタン | 「パスワード再設定メールを送信する」 |

※ 入力したメールアドレスがゲムコメに登録されていない場合も、セキュリティ上の理由から「パスワード再設定メールを送信しました」と同様の画面を表示し、登録有無を明かさない。

### 9.3 パスワード再設定メールのサンプル

```
パスワードの再設定を申請していない場合は、このメールを無視してください。
あなたのアカウントは引き続き安全に保護されています。

─────────────────────────
ゲムコメ https://gamecome.jp
このメールは送信専用です。返信はできません。
─────────────────────────
```

## 10. 初期設定フロー

アカウント作成完了（本登録完了）直後、または初回ソーシャルログイン直後に初期設定フローを開始する。

### 10.1 フロー概要

```
Step 1：プロフィール設定（ユーザー名・アイコン・公開設定）
    ↓
Step 2：好きなゲームジャンルを選択　← ここからスキップ可能
    ↓
Step 3：好きなゲームタイトルを選択
    ↓
Step 4：再生リストをマイリストに追加
    ↓
ホーム画面へ遷移
```

### 10.2 Step 1：プロフィール設定

| 項目 | 仕様 |
|------|------|
| ユーザー名入力フィールド | テキスト入力。1〜30文字。空欄不可 |
| アイコン画像アップロード | 任意。対応形式：JPG / PNG / GIF。ファイルサイズ上限：5MB。未設定の場合はデフォルトアイコンを使用 |
| マイリストを公開するトグル | ON/OFFのトグルスイッチ。デフォルトON。ここで選択した値が設定ページ（`/settings`）のプライバシー設定の初期値となる |
| レビュー履歴を公開するトグル | ON/OFFのトグルスイッチ。デフォルトON。同上 |
| 次へボタン | 「次へ」。ユーザー名が入力されている場合のみ活性 |
| スキップボタン | 「スキップして始める」。クリックするとStep 2以降を省略してホーム画面へ遷移 |

※ スキップした場合、ユーザー名・公開設定は後からプロフィールページの編集機能・設定ページでそれぞれ変更できる。スキップ時の公開設定はデフォルト値（ON）が適用される。

### 10.3 Step 2：好きなゲームジャンルを選択

| 項目 | 仕様 |
|------|------|
| 表示するジャンル | DBのジャンルマスタ（楽天ブックスのカテゴリ情報をもとに定義）から全件表示 |
| 選択方式 | 複数選択可 |
| 次へボタン | 「次へ」。1つ以上選択されている場合のみ活性 |
| スキップボタン | 「スキップ」。クリックするとStep 3以降を省略してホーム画面へ遷移 |

### 10.4 Step 3：好きなゲームタイトルを選択

Step 2で選択したジャンルに紐づくゲームタイトルの一覧を表示する。

| 項目 | 仕様 |
|------|------|
| 表示内容 | ゲームタイトル名・楽天ブックスのパッケージ画像 |
| 表示形式 | フラットなリスト形式（ジャンルによるグルーピングなし） |
| 選択方式 | 複数選択可 |
| 次へボタン | 「次へ」。1つ以上選択されている場合のみ活性 |
| スキップボタン | 「スキップ」。クリックするとStep 4を省略してホーム画面へ遷移 |

### 10.5 Step 4：再生リストをマイリストに追加

Step 3で選択したゲームタイトルに紐づく再生リストをフラットな一覧で表示する。

| 項目 | 仕様 |
|------|------|
| 表示内容 | 再生リストのサムネイル・タイトル・チャンネル名 |
| 操作 | 各再生リストに「マイリストに追加」ボタンを設置。クリックでマイリストに追加する |
| 追加時の視聴ステータス | 「見たい」固定 |
| 完了ボタン | 「ゲムコメを始める」。クリックするとホーム画面へ遷移 |

※ マイリストに追加しなかった再生リストの情報は記録しない。

### 10.6 初期設定完了後の遷移先

ホーム画面（`/`）へ遷移する。

## 11. 招待リンク

### 11.1 概要

ログイン済みユーザーは招待リンクを発行し、知人に共有できる。招待リンクをクリックしたユーザーはアカウント作成フォームへ直接遷移する。招待した側のユーザーへの特典は設けない。

### 11.2 招待リンクの発行場所

設定ページ（`/settings`）に「招待リンクを発行する」セクションを設ける。

| 項目 | 仕様 |
|------|------|
| ボタン | 「招待リンクをコピーする」。クリックでリンクをクリップボードにコピー |
| リンクの形式 | `https://gamecome.jp/signup?ref=[userId]` |
| リンクの有効期限 | なし（無期限） |

### 11.3 招待リンク経由のアカウント作成フロー

招待リンクをクリックすると、通常のアカウント作成フォームへ遷移する。アカウント作成後の初期設定フローは通常と同一（[第10章](#10-初期設定フロー)参照）。
