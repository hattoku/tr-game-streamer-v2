## 1. 概要

### 1.1 機能概要

設定ページは、ログイン済みユーザーが自分のアカウントに関する各種設定を管理するページである。

メールアドレス・パスワード・パスキーなどの認証情報の変更、プロフィールの公開設定、ログアウト・退会といった操作を1ページに集約して提供する。

### 1.2 非スコープ

- プロフィール情報（ユーザー名・アイコン・自己紹介文）の編集（プロフィールページのインライン編集機能を使用する）

## 2. アクセス条件・URL

### 2.1 アクセス権限

| ユーザー状態 | アクセス可否 |
|------------|------------|
| ログイン済み | ✅ アクセス可能 |
| 未ログイン | ❌ アクセス不可。ログインページへリダイレクト |

### 2.2 ページURL

| 項目 | 内容 |
|------|------|
| URL | `/settings` |
| ヘッダーナビゲーション | 表示する |

### 2.3 流入元

| 流入元 | 導線 |
|--------|------|
| プロフィールページ | 「公開設定 ⚙」ボタン（自分のページのみ表示） |
| ヘッダーナビゲーション | ユーザーメニュー内の「設定」リンク（実装者に委ねる） |

## 4. アカウント情報セクション

### 4.1 概要

認証に関する情報（メールアドレス・パスワード・パスキー）を管理するセクション。各項目は現在の設定状態を表示し、変更操作はモーダルで行う。

### 4.2 セクションレイアウト

```
┌─────────────────────────────────────────────────┐
│ アカウント情報                                    │
│                                                 │
│  メールアドレス                                   │
│  example@email.com          [変更する]           │
│ ─────────────────────────────────────────────── │
│  パスワード                                      │
│  ••••••••                   [変更する]           │
│ ─────────────────────────────────────────────── │
│  パスキー                                        │
│  未登録                     [設定する]           │
│  ※登録済みの場合：登録済み  [削除する]           │
└─────────────────────────────────────────────────┘
```

### 4.3 メールアドレス変更

#### 表示

| 項目 | 仕様 |
|------|------|
| 現在のメールアドレス | マスクせず全文を表示する |
| ボタン文言 | 「変更する」 |

#### モーダル仕様

| 項目 | 仕様 |
|------|------|
| モーダルタイトル | 「メールアドレスを変更する」 |
| 入力フィールド | 新しいメールアドレス（テキスト入力） |
| 確認フィールド | 新しいメールアドレス（確認用・再入力） |
| 送信ボタン | 「変更メールを送信する」 |
| キャンセルボタン | 「キャンセル」。クリックでモーダルを閉じる |

#### 変更フロー

```
[変更する] をクリック
    ↓
モーダルを表示
    ↓
新しいメールアドレスを入力・送信
    ↓
新しいメールアドレス宛に確認メールを送信
    ↓
モーダルに「確認メールを送信しました」と表示
    ↓
ユーザーがメール内のリンクをクリック
    ↓
メールアドレスの変更が完了
```

#### バリデーション

| チェック内容 | エラーメッセージ |
|------------|---------------|
| 入力値がメールアドレスの形式であること | 「正しいメールアドレスを入力してください」 |
| 確認フィールドと一致すること | 「メールアドレスが一致しません」 |
| 現在のメールアドレスと異なること | 「現在のメールアドレスと同じです」 |
| すでに他のアカウントで使用されていないこと | 「このメールアドレスはすでに使用されています」 |
| 空欄でないこと | 「メールアドレスを入力してください」 |

### 4.4 パスワード変更

#### 表示

| 項目 | 仕様 |
|------|------|
| パスワード表示 | 「••••••••」（固定マスク表示）|
| ボタン文言 | 「変更する」 |
| 表示条件 | パスキーのみで登録しているユーザー（パスワード未設定）にも表示する。パスワードを新たに設定するための入口として機能する |

#### モーダル仕様

| 項目 | 仕様 |
|------|------|
| モーダルタイトル | 「パスワードを変更する」 |
| 入力フィールド① | 現在のパスワード（パスワード未設定ユーザーの場合は非表示） |
| 入力フィールド② | 新しいパスワード |
| 入力フィールド③ | 新しいパスワード（確認用・再入力） |
| 保存ボタン | 「変更する」 |
| キャンセルボタン | 「キャンセル」。クリックでモーダルを閉じる |

#### バリデーション

| チェック内容 | エラーメッセージ |
|------------|---------------|
| 現在のパスワードが正しいこと（パスワード設定済みの場合） | 「現在のパスワードが正しくありません」 |
| 新しいパスワードが8文字以上であること | 「パスワードは8文字以上で入力してください」 |
| 確認フィールドと一致すること | 「パスワードが一致しません」 |
| 空欄でないこと | 「パスワードを入力してください」 |

### 4.5 パスキー設定

パスキーはメールアドレス＋パスワードの代替認証手段として機能する。

#### 表示

| 状態 | 表示内容 | ボタン文言 |
|------|---------|----------|
| 未登録 | 「未登録」 | 「設定する」 |
| 登録済み | 「登録済み」 | 「削除する」 |

#### 登録モーダル仕様

| 項目 | 仕様 |
|------|------|
| モーダルタイトル | 「パスキーを設定する」 |
| 説明文 | 「パスキーを設定すると、パスワードの代わりに顔認証・指紋認証・PINなどでログインできます。」 |
| 登録ボタン | 「パスキーを登録する」。クリックでブラウザのパスキー登録フローを起動する |
| キャンセルボタン | 「キャンセル」。クリックでモーダルを閉じる |

#### 削除モーダル仕様

| 項目 | 仕様 |
|------|------|
| モーダルタイトル | 「パスキーを削除する」 |
| 注意書き | パスワードが未設定の場合：「パスキーを削除すると、ログインできなくなります。削除前にパスワードを設定してください。」を表示し、削除ボタンを非活性にする |
| 削除ボタン | 「削除する」（パスワード設定済みの場合のみ活性） |
| キャンセルボタン | 「キャンセル」。クリックでモーダルを閉じる |

> **設計思想**: パスキーとパスワードの両方が未設定になるとログイン不能になるため、パスキー削除はパスワード設定済みの場合のみ許可する。

## 5. プライバシー設定セクション

### 5.1 概要

プロフィールページで他のユーザーに公開する情報の範囲を設定するセクション。設定はトグルスイッチで操作する。

各設定の初期値はアカウント作成時の初期設定フロー（プロフィール設定ステップ）でユーザーが選択した値が反映される。変更はリアルタイムで保存され、保存ボタンは不要とする。

### 5.2 セクションレイアウト

```
┌─────────────────────────────────────────────────┐
│ プライバシー設定                                  │
│                                                 │
│  マイリストを公開する                 [●────]ON  │
│  他のユーザーにマイリストを公開します             │
│ ─────────────────────────────────────────────── │
│  レビュー履歴を公開する               [●────]ON  │
│  他のユーザーにレビュー履歴を公開します           │
└─────────────────────────────────────────────────┘
```

### 5.3 設定項目

| 設定項目 | デフォルト | 説明文 | OFF時の挙動 |
|---------|----------|--------|-----------|
| マイリストを公開する | アカウント作成時にユーザーが選択した値 | 「他のユーザーにマイリストを公開します」 | 自分のプロフィールページのマイリストセクションが他のユーザーから非表示になる |
| レビュー履歴を公開する | アカウント作成時にユーザーが選択した値 | 「他のユーザーにレビュー履歴を公開します」 | 自分のプロフィールページのレビュー履歴セクションが他のユーザーから非表示になる |

> 公開設定の詳細な挙動はプロフィールページ仕様書 第2.3節を参照。

### 5.4 保存仕様

| 項目 | 仕様 |
|------|------|
| 保存タイミング | トグル操作と同時にリアルタイム保存 |
| 保存先 | Firestore の `users/{userId}` ドキュメント |
| フィードバック | 保存完了時にトースト通知「設定を保存しました」を表示 |

## 6. 通知設定セクション

### 6.1 概要

プッシュ通知の許可状態と通知種別のON/OFFを管理するセクション。
ログイン済みユーザーのみが対象であり、このページ自体がログイン必須のため全ユーザーに表示される。

### 6.2 セクションレイアウト

以下の3パターンをASCIIアートで表現すること。

**パターンA：未許可（default）状態**

```
┌─────────────────────────────────────────────────┐
│ 通知設定                                        │
│                                                 │
│  プッシュ通知を許可すると、マイリスト登録中の         │
│  シリーズに新着動画が追加されたときなどに           │
│  お知らせします。                                 │
│                                                 │
│  [通知を許可する]                                │
└─────────────────────────────────────────────────┘
```

**パターンB：許可済み（granted）状態**

```
┌─────────────────────────────────────────────────┐
│ 通知設定                                        │
│                                                 │
│  ✅ 通知が有効です                               │
│ ─────────────────────────────────────────────── │
│  シリーズ新着通知                     [●────]ON  │
│  マイリスト登録中のシリーズに新着動画が             │
│  追加されたとき                                   │
│ ─────────────────────────────────────────────── │
│  お知らせ通知                         [●────]ON  │
│  プレミテからのお知らせがあるとき                 │
└─────────────────────────────────────────────────┘
```

**パターンC：拒否済み（denied）状態**

```
┌─────────────────────────────────────────────────┐
│ 通知設定                                        │
│                                                 │
│  🚫 通知がブロックされています                   │
│                                                 │
│  ブラウザの設定から、このサイトの通知をONに         │
│  してください。                                   │
└─────────────────────────────────────────────────┘
```

### 6.3 許可状態の種別と表示

| 状態 | 条件 | 表示内容 |
|------|------|---------|
| 未許可（default） | まだ許可を求めていない | 説明文＋「通知を許可する」ボタン |
| 許可済み（granted） | ブラウザで通知を許可している | 許可済みバッジ＋通知種別トグル |
| 拒否済み（denied） | ブラウザで通知を拒否している | 拒否中バッジ＋ブラウザ設定変更の案内テキスト（ボタンなし） |

拒否済みの場合、アプリ側から再度許可を求めることはできない。ブラウザの設定からONにするよう案内するテキストを表示するにとどめる。

### 6.4 通知種別トグル

許可済み（granted）状態のときのみ表示する。

| 通知種別 | デフォルト | 説明文 |
|---------|----------|-------|
| シリーズ新着通知 | ON | マイリスト登録中のシリーズに新着動画が追加されたとき |
| お知らせ通知 | ON | プレミテからのお知らせがあるとき |

- トグルの変更は即時反映する（保存ボタン不要）
- 設定はFirestoreのユーザードキュメントに保存する

### 6.5 「通知を許可する」ボタンの動作フロー

以下の流れをASCIIフローチャートで記述すること。

```
「通知を許可する」ボタンをクリック
    ↓
ブラウザの許可ダイアログを表示
    ↓
├── 許可の場合 ── FCMトークンを取得しFirestoreに保存 ──> 許可済み状態の表示に切り替え
└── 拒否の場合 ──> 拒否済み状態の表示に切り替え
```

詳細仕様は/document/specification/management/プレミテ FCMプッシュ通知基盤仕様書.mdの第4章を参照すること。

## 8. 退会

### 8.1 表示

```
┌─────────────────────────────────────────────────┐
│ 退会                                            │
│                                                 │
│  退会するとすべてのデータが削除され、              │
│  元に戻すことはできません。                       │
│                                                 │
│  [退会する]                                      │
└─────────────────────────────────────────────────┘
```

| 項目 | 仕様 |
|------|------|
| ボタン文言 | 「退会する」 |
| ボタンスタイル | 危険な操作であることを示す赤系のスタイルを使用する |

### 8.2 確認モーダル仕様

| 項目 | 仕様 |
|------|------|
| モーダルタイトル | 「退会しますか？」 |
| 本文 | 「退会すると、投稿したレビュー・マイリスト・視聴履歴などすべてのデータが削除されます。この操作は取り消せません。」 |
| 実行ボタン | 「退会する」（赤系スタイル） |
| キャンセルボタン | 「キャンセル」。クリックでモーダルを閉じる |

### 8.3 退会処理

退会実行時に削除するデータは以下のとおり。

| 削除対象 | 詳細 |
|---------|------|
| ユーザーアカウント | Firebase Auth のアカウントを削除する |
| ユーザードキュメント | Firestore の `users/{userId}` ドキュメントを削除する |
| レビュー | `reviews` コレクション内の当該ユーザーの投稿をすべて削除する |
| マイリスト | `mylist` コレクション内の当該ユーザーのデータをすべて削除する |
| 視聴履歴 | 当該ユーザーの視聴履歴データをすべて削除する |

> **補足**: 退会によるレビュー削除は、他のユーザーのレビューへの「参考になった」投票（helpfulCount）にも影響する。退会ユーザーが押した「参考になった」は削除し、カウントからも除外する。

### 8.4 退会後の遷移

| 項目 | 仕様 |
|------|------|
| 遷移先 | トップページ（`/`） |
