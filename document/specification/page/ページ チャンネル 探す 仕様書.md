## 1. 概要

### 1.1 機能概要

「チャンネル名から探す」は、ゲムコメに登録されている再生リストと紐づくチャンネルを検索・発見するためのページである。キーワードで絞り込み、ソートした一覧からユーザーが興味のあるチャンネルを見つけることができる。

「再生リストを探す」が再生リスト単位での発見を目的とするのに対し、本ページは**チャンネル単位での発見**を起点とする。チャンネルを起点に、そのチャンネルが持つ実況シリーズへ到達するための導線を提供する。

チャンネルが持つメタ情報は再生リストやゲームタイトルと比べて少ないため、シンプルな構成を重視する。

### 1.2 アクセス条件

| 条件 | 内容 |
|------|------|
| ログイン済みユーザー | ✅ アクセス可能 |
| 未ログインユーザー | ✅ アクセス可能（検索・閲覧は可能。マイリスト登録はログイン必須） |

### 1.3 ページURL

| 項目 | 内容 |
|------|------|
| URL | `/channels` |
| ヘッダーナビゲーション | 常に表示（ログイン状態を問わず） |

│ チャンネル名から探す                                           │

│ ┌───────────────────────────────┬─────────────────────────┐ │
│ │ 左カラム                       │ 右カラム                 │ │
│ │ カード一覧                     │ 検索条件パネル           │ │
│ │                               │                         │ │
│ │ ソート: [再生リスト登録数順 ▼]  │ [ キーワード検索 ]        │ │

│ │ ┌─────────────────────────┐  │                         │ │

│ │ ┌─────────────────────────┐  │                         │ │

## 3. 左カラム：カード一覧

### 3.1 一覧ヘッダー

カード一覧の上部に以下を表示する。

| 表示要素 | 詳細 |
|---------|------|
| 検索結果件数 | 「XX件」の形式で表示。検索・フィルター適用後もリアルタイムに更新 |
| ソートUI | 検索結果件数の右側にドロップダウンを設置（ソート仕様は [第5章](#5-ソート仕様) に定義） |

### 3.2 カードのデザイン方針

1枚のカードで、ユーザーが「このチャンネルを追いかけたいかどうか」を判断できる情報を過不足なく提供する。チャンネルの基本情報と代表的な再生リストをコンパクトにまとめ、視認性を重視したシンプルなレイアウトを採用する。

```
┌────────────────────────────────────────────────────────────┐
│ ┌──────────┐  チャンネル名                                  │
│ │          │                                               │
│ │ チャンネル│  □ 3 再生リスト  ▶ 45 動画                   │
│ │サムネイル │                                               │
│ │ （正方形）│  ─────────────────────────────               │
│ │          │  ┌────┐ タイトルA  ★ 4.8                      │
│ └──────────┘  ├────┤ タイトルB  ★ 4.5                      │
│               └────┘ タイトルC  評価なし                    │
└────────────────────────────────────────────────────────────┘
```

### 3.3 カードに含める表示要素

#### チャンネル情報エリア

| 表示要素 | 詳細 |
|---------|------|
| チャンネルサムネイル | YouTubeチャンネルのアイコン画像（正方形） |
| チャンネル名 | YouTubeチャンネル名（最大1行、超過時は省略記号…） |
| 再生リスト登録数 | 再生リストを示すアイコン +「再生リスト」テキスト + 数値（例：□ 3 再生リスト）。このチャンネルに紐づく再生リストの登録数合計 |
| 動画数 | 動画再生を示すアイコン +「動画」テキスト + 数値（例：▶ 45 動画）。このチャンネルに紐づく全再生リストの動画数合計 |

#### ピックアップ再生リストエリア

チャンネル情報エリアと視覚的に区切り（境界線または背景色の差異）を設け、「このチャンネルの代表的な再生リスト」であることを明示する。

スコアが高い順で上位3件の再生リストを表示する。

| 表示要素 | 詳細 |
|---------|------|
| 再生リストサムネイル | YouTube再生リストのサムネイル画像（小サイズ） |
| 再生リストタイトル | 最大1行、超過時は省略記号（…） |
| スコア | ★アイコン + 数値（小数点第1位、例：★ 4.8）。レビューが0件の場合は「評価なし」と表示 |

**ピックアップ件数が3件未満の場合**:

チャンネルに紐づく再生リストが3件未満の場合は、存在する件数のみ表示する（空行は設けない）。

### 3.4 カードのクリック動作

#### チャンネル情報エリアのクリック

チャンネル情報エリア（サムネイル・チャンネル名・再生リスト数・動画数）をクリックした場合の遷移先は**スコープ外**とする（チャンネル詳細ページは別途仕様定義）。

> ⚠️ **実装上の注意**: チャンネル詳細ページが未実装の間は、クリック操作を無効化するか、遷移先をTBDとして設計上の余地を残すこと。

#### ピックアップ再生リストのクリック

ピックアップエリアに表示された各再生リスト（サムネイル・タイトル・スコアを含む行全体）をクリックすると、該当する再生リストの**動画プレーヤーページ**へ遷移する。

- ログイン済みユーザーかつマイリスト登録済みの場合：最後に開いた話数から再生を開始できる（動画プレーヤー仕様書に準ずる）
- 未ログインユーザー・初回訪問の場合：第1話から再生を開始する

### 3.5 ページネーション

| 項目 | 仕様 |
|------|------|
| 1ページあたりの表示件数 | 20件 |
| 方式 | ページネーション（ページ番号リンク形式） |
| 表示位置 | カード一覧の下部 |

## 4. 右カラム：検索条件パネル

右カラムはページとともにスクロールする（スクロール追従なし）。

### 4.1 パネル構成

```
┌─────────────────────────┐
│ 検索条件                 │
│                         │
│ [ キーワード検索 ]        │
│                         │
│ [ 検索条件をリセット ]    │
└─────────────────────────┘
```

### 4.2 キーワード検索

| 項目 | 仕様 |
|------|------|
| 入力形式 | テキスト入力フィールド |
| 検索対象 | **チャンネル名** |
| 検索タイミング | 入力のたびにリアルタイムでフィルタリング（デバウンス処理を適用、遅延目安：300ms） |
| 部分一致 | 対応（入力文字列を含む結果を表示） |
| 大文字小文字 | 区別しない |
| プレースホルダー | 「チャンネル名で検索」 |
| クリアボタン | 入力値がある場合に「×」ボタンを表示。クリックで入力内容をクリアし、全件表示に戻す |

### 4.3 検索条件のリセット

| 項目 | 仕様 |
|------|------|
| ボタン表示条件 | キーワード入力がある場合に活性化。何も設定されていない場合はグレーアウト |
| クリック時の動作 | キーワードをリセットし、全件表示に戻す |
| ボタン文言 | 「検索条件をリセット」 |

## 5. ソート仕様

### 5.1 ソート条件

| ソート | 説明 | デフォルト |
|--------|------|-----------|
| 再生リスト登録数順 | そのチャンネルに紐づく再生リストの登録数が多い順に表示 | ✅ デフォルト |
| チャンネル名順 | チャンネル名のあいうえお順（昇順）に表示 | — |
| 動画数順 | そのチャンネルに紐づく全再生リストの動画数合計が多い順に表示 | — |

> **設計思想**: デフォルトを「再生リスト登録数順」とすることで、初めてページを訪れたユーザーが「多くの再生リストを持つ実績あるチャンネル」を最初に目にできる導線を設計する。

### 5.2 登録数・動画数が同数の場合の並び順

同じ数値のチャンネルが複数存在する場合の並び順は、チャンネル名のあいうえお順（昇順）とする。

### 5.3 ソートUIの仕様

| 項目 | 仕様 |
|------|------|
| UI形式 | ドロップダウン（セレクトボックス） |
| 配置位置 | カード一覧ヘッダーの右側（件数表示の右） |
| 表示文言 | 選択中のソート名を表示（例：「再生リスト登録数順」） |

## 6. 空状態の表示

検索条件を適用した結果、0件になった場合は以下を表示する。

| 要素 | 内容 |
|------|------|
| アイコン | 検索（虫眼鏡）アイコン |
| メッセージ | 「該当するチャンネルが見つかりませんでした」 |
| サブメッセージ | 「検索条件を変えてお試しください」 |
| リセットボタン | 「検索条件をリセット」ボタンを表示し、クリックで全件表示に戻す |

## 7. スマートフォン対応

### 7.1 レイアウト変更

スマートフォン（画面幅が一定以下）では、2カラム構成を **1カラム構成** に切り替える。検索条件パネルをカード一覧の上部に配置する。

検索条件の項目数が少ないため（キーワード検索のみ）、アコーディオン（じゃばら）は使用せず、常に展開した状態で表示する。

```
┌───────────────────────────────┐
│ ヘッダー                       │
├───────────────────────────────┤
│ チャンネル名から探す            │
│                               │
│ ┌─────────────────────────┐  │
│ │ [ チャンネル名で検索  × ] │  │  ← 常時表示（じゃばらなし）
│ └─────────────────────────┘  │
│                               │
│ ソート: [再生リスト登録数順 ▼]  │
│ XX件                          │
│                               │
│ ┌─────────────────────────┐  │
│ │ カード                   │  │
│ └─────────────────────────┘  │
│  ...                          │
└───────────────────────────────┘
```

### 7.2 キーワード入力中の状態表示

キーワードが入力されている場合、入力フィールド内に「×」ボタンを表示し、タップするとキーワードをクリアして全件表示に戻す。
