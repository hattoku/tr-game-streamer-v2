## 1. 概要

### 1.1 機能概要

プロフィールページは、ゲムコメに登録したユーザーの公開情報を表示するページである。

レビュー一覧に表示される投稿者名リンクをクリックすることで遷移する。閲覧者が「このレビュワーはどんな人物か」「どのジャンルに詳しいか」「どんな作品を追っているか」を把握できるよう設計し、レビューの信頼性判断を補助する役割を担う。

### 1.2 ページの目的

レビュー投稿者の専門性・活動実績・視聴傾向を可視化することで、以下を実現する。

- レビュワーへの信頼感を形成する
- 閲覧者がレビューを参考にするかどうかの判断材料を提供する
- ゲムコメのコミュニティ感を醸成する

### 1.3 非スコープ

- 視聴履歴の公開表示（本人のみ参照可能な視聴履歴ページは別仕様書に定義済み。プロフィールページには掲載しない）
- フォロー・フォロワー機能
- ユーザー間のメッセージ機能

## 2. アクセス条件・URL

### 2.1 アクセス権限

| ユーザー状態 | アクセス可否 | 表示内容 |
|------------|------------|--------|
| 未ログイン | ✅ アクセス可能 | プロフィール情報・統計・ジャンル分布を表示。マイリスト・レビュー履歴はユーザーの公開設定に従う |
| ログイン済み（他人のページ） | ✅ アクセス可能 | 同上 |
| ログイン済み（自分のページ） | ✅ アクセス可能 | 公開設定にかかわらず全セクションを表示 ＋ プロフィール編集ボタンを表示 |

### 2.3 マイリスト・レビュー履歴の公開設定

マイリストとレビュー履歴はそれぞれ独立してON/OFFを設定できる。設定は `/settings`（設定ページ）で管理する（設定ページの詳細仕様は別途定義）。

| 設定 | デフォルト | 他人のプロフィールページでの挙動 |
|------|----------|-------------------------------|
| マイリストを公開する | ON | OFF時はマイリストセクション自体を非表示にし、本人のページでのみ表示する |
| レビュー履歴を公開する | ON | OFF時はレビュー履歴セクション自体を非表示にし、本人のページでのみ表示する |

> **設計思想**: 自分のページでは公開設定に関わらず常に全セクションを表示することで、設定変更後の見た目を本人が確認できるようにする。非公開時はセクションごと消すシンプルな実装とし、「非公開です」などのメッセージは表示しない。

### 2.2 ページURL

| 項目 | 内容 |
|------|------|
| URL | `/profile/[userId]` |
| ヘッダーナビゲーション | 常に表示（ログイン状態を問わず） |

## 3. ページレイアウト

### 3.1 全体構成

ページは **1カラム構成** で配置される。各セクションを縦に積み上げる形式とする。

```
┌─────────────────────────────────────────────────┐
│ ヘッダー                                         │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │ ① プロフィールヘッダー                     │  │
│  │   アイコン / ユーザー名 / 自己紹介 / 登録日 │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │ ② 統計情報                                 │  │
│  │   投稿レビュー数 / 参考になった数 / ステータス│  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │ ③ ジャンル分布                              │  │
│  │   よくレビューするジャンル Top3              │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │ ④ マイリスト                               │  │
│  │   登録中の再生リスト一覧（じゃばら展開）     │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
│  ┌───────────────────────────────────────────┐  │
│  │ ⑤ レビュー履歴                              │  │
│  │   投稿済みレビューの一覧（じゃばら展開）     │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
└─────────────────────────────────────────────────┘
```

※ マイリスト・レビュー履歴セクションはユーザーの公開設定によって他人のページでは非表示になる場合がある（第2.3節参照）。自分のページでは常に表示する。

### 3.2 じゃばら展開の共通仕様

マイリストセクション・レビュー履歴セクションは、共通のじゃばら（アコーディオン）展開パターンを採用する。

| 状態 | 表示内容 |
|------|---------|
| 折りたたみ時（デフォルト） | 上位3件のカードを表示。下部に「すべて見る（XX件）▼」ボタンを表示 |
| 展開時 | 20件単位でページネーション表示。上部に「折りたたむ ▲」ボタンを表示 |

| 項目 | 仕様 |
|------|------|
| 展開ボタン文言 | 「すべて見る（XX件）▼」（XXは総件数） |
| 折りたたみボタン文言 | 「折りたたむ ▲」 |
| 展開後のページネーション | 20件/ページ、ページ番号リンク形式 |
| 3件以下の場合 | 「すべて見る」ボタンを表示しない（全件をそのまま表示） |

## 5. 統計情報セクション

### 5.1 概要

ユーザーのゲムコメ上での活動量を数値で示すセクション。レビュワーとしての実績を一目で把握できるよう設計する。

### 5.2 表示レイアウト

数値カードを横並びで表示する。

```
┌──────────┐  ┌──────────┐  ┌────────────────────┐
│ レビュー  │  │ 参考に   │  │ 視聴ステータス      │
│ 投稿数   │  │ なった数 │  │ 内訳               │
│          │  │          │  │                    │
│   42     │  │   138    │  │ 視聴中  12         │
│   件     │  │   回     │  │ 完走    31         │
│          │  │          │  │ 見たい  8          │
│          │  │          │  │ 一時中断 2         │
│          │  │          │  │ 断念    3          │
└──────────┘  └──────────┘  └────────────────────┘
```

### 5.3 各項目の仕様

| 項目 | 集計内容 | 表示形式 |
|------|---------|---------|
| レビュー投稿数 | 星評価・視聴ステータス・コメントのいずれかが存在する投稿の総数 | 「XX件」 |
| 「参考になった」獲得数 | 自分の全レビューが受け取った「参考になった」の合計数 | 「XX回」 |
| 視聴ステータス内訳 | マイリストに登録された全再生リストのステータス別件数 | ステータス名 + 件数を全種表示。件数が0のステータスは非表示 |

### 5.4 視聴ステータスの表示対象

| ステータス | 表示条件 |
|-----------|---------|
| 視聴中 | 件数が1以上の場合に表示 |
| 完走 | 件数が1以上の場合に表示 |
| 見たい | 件数が1以上の場合に表示 |
| 一時中断 | 件数が1以上の場合に表示 |
| 断念 | 件数が1以上の場合に表示 |

## 6. ジャンル分布セクション

### 6.1 概要

このユーザーがレビューを投稿したゲームタイトルのジャンル分布を表示するセクション。どのジャンルに詳しいかを示す「専門性の指標」として機能する。

> **注記**: 本セクションのジャンル分布は「レビュー履歴」を基に集計する。信頼度スコアリングシステムのジャンル専門性（G）は「マイリスト登録」を基に算出しており、データソースが異なる。

### 6.2 データソースと集計方法

レビュー履歴に紐づくゲームタイトルの**楽天ブックスジャンルデータ**（DBのジャンルマスタに登録されているもの）を集計する。

集計ロジック：
- ユーザーが投稿したレビューに紐づく再生リスト → 対応するゲームタイトル → そのゲームタイトルのジャンルを取得
- 同一ゲームタイトルへのレビューが複数ある場合でも、ジャンルのカウントは1回のみとする（ゲームタイトル単位で集計）
- 件数の多い順に上位3件を表示する

### 6.3 表示レイアウト

```
┌─────────────────────────────────────────────────┐
│ よくレビューするジャンル                          │
│                                                 │
│  [RPG 12件]  [アクション 8件]  [アドベンチャー 5件] │
└─────────────────────────────────────────────────┘
```

### 6.4 表示仕様

| 項目 | 仕様 |
|------|------|
| 表示形式 | バッジ形式。ジャンル名 + レビュー件数を表示（例：[RPG 12件]） |
| 表示件数 | 上位3件 |
| 件数の意味 | そのジャンルに属するゲームタイトルをレビューした件数 |
| データが0件の場合 | セクション自体を非表示にする |
| 件数が同数の場合の並び順 | 50音順（昇順） |

## 7. マイリストセクション

### 7.1 概要

このユーザーがマイリストに登録している再生リストを一覧表示するセクション。視聴傾向の把握や、同じ作品を追っているユーザーの発見に役立てる。

公開範囲はユーザーの設定に従う（第2.3節参照）。デフォルトは公開。

### 7.2 表示レイアウト

じゃばら展開の共通仕様（第3.2節）に準じる。

```
┌─────────────────────────────────────────────────┐
│ マイリスト（56件）                                │
│                                                 │
│  ┌─────────────────────────────────────────┐   │
│  │ カード 1                                  │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ カード 2                                  │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ カード 3                                  │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
│           [ すべて見る（56件）▼ ]               │
└─────────────────────────────────────────────────┘
```

### 7.3 カードの表示要素

```
┌──────────────────────────────────────────────────┐
│ ┌──────────┐  再生リストタイトル                   │
│ │          │  ゲームタイトル名                     │
│ │サムネイル │                                     │
│ │          │  [視聴ステータスバッジ]  ★ 4.2        │
│ └──────────┘                                     │
└──────────────────────────────────────────────────┘
```

| 要素 | 詳細 |
|------|------|
| 再生リストのサムネイル | YouTube再生リストのサムネイル画像。クリックで動画プレーヤーページへ遷移 |
| 再生リストタイトル | クリックで動画プレーヤーページへ遷移。最大2行、超過時は省略記号（…） |
| ゲームタイトル名 | 対象ゲームのタイトル名。クリックでゲームタイトル詳細ページへ遷移 |
| 視聴ステータスバッジ | 登録時に設定したステータスをバッジ形式で表示 |
| スコア（★評価） | ゲムコメ上の再生リストスコア（加重平均）。★アイコン + 小数点第1位（例：★ 4.2）。レビュー0件の場合は「評価なし」と表示 |

### 7.4 ソート

| 項目 | 仕様 |
|------|------|
| デフォルトソート | マイリストへの登録日時の降順（新しく登録した順） |

### 7.5 空状態の表示

マイリストが0件の場合は以下を表示する。

| 要素 | 内容 |
|------|------|
| アイコン | リスト（または再生リスト）アイコン |
| メッセージ | 「まだマイリストに登録している再生リストはありません」 |

## 8. レビュー履歴セクション

### 8.1 概要

このユーザーが投稿したレビューを新着順で一覧表示するセクション。閲覧者がレビュワーの評価傾向や過去の投稿内容を確認できる。

公開範囲はユーザーの設定に従う（第2.3節参照）。デフォルトは公開。

### 8.2 ネタバレフィルター

レビュー履歴にはネタバレフィルター機能を設ける。

| 項目 | 仕様 |
|------|------|
| デフォルト | ON（ネタバレコメントを折りたたみ表示） |
| ON時の挙動 | ネタバレフラグのついたレビューのコメント本文を「ネタバレを含みます。表示する」として折りたたみ表示。クリックで個別展開可能 |
| OFF時の挙動 | すべてのコメントをそのまま表示 |
| フィルターUI | セクションヘッダー付近にトグルを設置（例：「ネタバレを隠す」） |
| 設定の保存（未ログイン） | LocalStorageに保存し、次回訪問時にも状態を維持 |
| 設定の保存（ログイン済み） | DBに保存し、端末をまたいで状態を維持 |

> **補足**: ネタバレフィルターの設定はレビュー投稿機能仕様書のネタバレ非表示フィルターと同仕様。閲覧者側の設定として機能し、プロフィールページの表示に影響する。

### 8.3 表示レイアウト

```
│  │ カード 2                                  │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ カード 3                                  │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
│           [ すべて見る（42件）▼ ]               │
└─────────────────────────────────────────────────┘
```

### 8.4 カードの表示要素

```

│ ┌──────────┐  再生リストタイトル                       │
│ │          │  ゲームタイトル名                         │

└──────────────────────────────────────────────────────┘
```

| 要素 | 詳細 |
|------|------|
| 再生リストのサムネイル | YouTube再生リストのサムネイル画像。クリックで動画プレーヤーページへ遷移 |
| 再生リストタイトル | クリックで動画プレーヤーページへ遷移。最大2行、超過時は省略記号（…） |
| ゲームタイトル名 | 対象ゲームのタイトル名。クリックでゲームタイトル詳細ページへ遷移 |
| 星評価 | ★表示（例：★★★★☆ 4.0）。未評価の場合は非表示 |
| 視聴ステータス | 投稿時の視聴ステータス。設定されている場合のみ表示 |

| 投稿日 | 「YYYY年M月D日」の形式で表示。更新がある場合は「（YYYY年M月D日 更新）」を付記 |

### 8.5 ソート

| 項目 | 仕様 |
|------|------|
| デフォルトソート | 投稿日時の降順（新着順） |

### 8.6 空状態の表示

レビューが1件もない場合は以下を表示する。

| 要素 | 内容 |
|------|------|
| アイコン | レビュー（星）アイコン |
| メッセージ | 「まだレビューの投稿はありません」 |

## 9. プロフィール編集機能

### 9.1 編集対象

自分のプロフィールページにのみ「編集する」ボタンを表示する。クリックするとプロフィールヘッダーセクションが編集モードに切り替わる。

### 9.2 編集可能項目

| 項目 | 仕様 |
|------|------|
| プロフィール画像 | 画像ファイルをアップロードして変更可能。対応形式：JPG / PNG / GIF。推奨サイズ：200×200px以上の正方形 |
| ユーザー名 | テキスト入力。1〜30文字。空欄不可 |
| 自己紹介文 | テキストエリア。最大200文字。文字数カウンターを表示。任意入力（空欄可） |

### 9.3 編集UIの仕様

| 項目 | 仕様 |
|------|------|
| 編集モードへの移行 | 「編集する」ボタンクリックで、プロフィールヘッダーセクションがインライン編集フォームに切り替わる |
| 保存ボタン | 「保存する」ボタン。クリックでFirestoreに保存し、表示モードに戻る |
| キャンセルボタン | 「キャンセル」ボタン。クリックで変更を破棄し、表示モードに戻る |

### 9.4 バリデーション

| 項目 | バリデーション内容 |
|------|----------------|
| ユーザー名 | 1文字以上30文字以内。空欄の場合は保存不可（エラーメッセージを表示） |
| 自己紹介文 | 200文字以内。超過時はカウンターを赤色表示し保存不可 |
| プロフィール画像 | 対応形式外のファイルはアップロード不可（エラーメッセージを表示）。ファイルサイズ上限：5MB |

## 10. データ仕様

### 10.1 Firestoreデータ構造

プロフィールページで参照・更新するFirestoreのフィールドを以下に定義する。

#### usersコレクション（既存 + 追加）

```
users/{userId}
  ├── displayName: string          # ユーザー名
  ├── email: string                # メールアドレス（非公開）
  ├── profileImageUrl: string      # プロフィール画像URL（新規追加）
  ├── bio: string                  # 自己紹介文（新規追加）
  ├── accountCreatedAt: timestamp  # アカウント登録日
  ├── reviewCount: number          # レビュー投稿数（既存）
  ├── isMylistPublic: boolean      # マイリストの公開設定（新規追加、デフォルト: true）
  └── isReviewHistoryPublic: boolean  # レビュー履歴の公開設定（新規追加、デフォルト: true）
```

#### プロフィールページで集計・参照するコレクション

| データ | 参照元 | 集計方法 |
|--------|--------|---------|
| レビュー投稿数 | `reviews` コレクション | `userId` でフィルタして件数カウント |
| 「参考になった」獲得数合計 | `reviews` コレクション | `userId` でフィルタして `helpfulCount` フィールドを合計 |
| 視聴ステータス内訳 | `mylist`（マイリスト相当のコレクション） | `userId` でフィルタして `watchStatus` ごとに件数カウント |
| ジャンル分布 | `reviews` → `playlists` → `games` | レビューに紐づくゲームのジャンルをゲームタイトル単位で集計 |
| マイリスト一覧 | `mylist` コレクション | `userId` でフィルタして登録日時降順で取得 |
| レビュー履歴一覧 | `reviews` コレクション | `userId` でフィルタして投稿日時降順で取得 |

### 10.2 「参考になった」獲得数フィールドの追加

既存の `reviews` コレクションに `helpfulCount` フィールドが未定義の場合は追加する。レビュー投稿機能仕様書の「参考になったボタン」仕様に準じて、ボタン押下時にインクリメント・再クリック時にデクリメントする。
