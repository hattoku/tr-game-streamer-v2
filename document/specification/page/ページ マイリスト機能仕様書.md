## 1. 概要

### 1.1 機能概要

マイリストは、ログイン済みユーザーが自分の「追いかけている再生リスト」を一元管理するための機能である。複数の実況シリーズの視聴ステータスや視聴進捗を一覧で確認でき、効率的な「並走視聴」を実現する。

なお、マイリストへの登録データは、信頼度スコアリングシステムのジャンル専門性（G）の算出にも使用される。詳細は「信頼度スコアリングシステム仕様書」を参照すること。

### 1.2 アクセス条件

| 条件 | 内容 |
|------|------|
| ログイン状態 | ✅ マイリストページにアクセス可能 |
| 未ログイン状態 | ❌ アクセス不可（ログインページへリダイレクト） |

### 1.3 マイリスト登録のトリガー

再生リストページに設置された「マイリストに追加」ボタンを押すことで登録する。

**登録時の視聴ステータス選択**

マイリストへの追加時に、ユーザーが視聴ステータスを選択するモーダルまたはインラインフォームを表示する。

| ステータス | アイコン | 説明 |
|-----------|---------|------|
| 見たい（Plan to Watch） | 📋 | これから視聴する予定 |
| 視聴中（Watching） | ▶️ | 現在視聴中 |
| 完走（Completed） | ✅ | 最終話まで視聴済み |
| 一時中断（On Hold） | ⏸️ | 一旦中断中 |
| 断念（Dropped） | ❌ | 途中で視聴をやめた |

> **設計思想**: 「とりあえず登録」する場合は「見たい」、「続きから見る」場合は「視聴中」を選ぶことが多いと想定されるため、ユーザー自身が意図を持って選択できる設計にする。

## 3. フィルタリング仕様

### 3.1 フィルタUIレイアウト

使用頻度に応じてタブの優先度を設計する。

```
[すべて] [視聴中] [見たい] [完走] [その他 ▼]
                                   └ 一時中断
                                   └ 断念
```

| タブ | 表示条件 | 優先度 |
|------|---------|--------|
| すべて | 常に表示（全ステータス） | 最高（デフォルト選択） |
| 視聴中 | 常に表示 | 高 |
| 見たい | 常に表示 | 高 |
| 完走 | 常に表示 | 中 |
| その他 ▼ | ドロップダウン（一時中断・断念） | 低 |

**「その他」ドロップダウンの挙動**

- クリックでドロップダウンを開き、「一時中断」「断念」を選択できる
- 「一時中断」または「断念」が選択されている場合、タブ部分に「その他：一時中断」のように選択中のステータスを表示する

### 3.2 各タブの件数表示

各タブには件数バッジを表示し、ユーザーが全体像を把握しやすくする。

```
[すべて 24] [視聴中 12] [見たい 5] [完走 6] [その他 ▼ 1]
```

## 4. ソート仕様

### 4.1 ソート条件

| ソート | 説明 | デフォルト |
|--------|------|-----------|
| 最後に再生した動画（新しい順） | 直近に動画を開いた再生リストが上に表示される | ✅ デフォルト |
| 再生リスト名（昇順） | 再生リストのタイトルのあいうえお順 | — |

> **設計思想**: 「続きを見たい」というユーザーの自然な動線に沿って、直近に触れたシリーズを上位に出すことで、再生への摩擦を最小化する。

## 5. カード仕様

### 5.1 カードのデザイン方針

一枚のカードの中に「再生リストについての情報」と「最後に再生した動画」を親子関係がわかるように表示する。

```
┌────────────────────────────────────────────────────────┐
│ ┌──────────┐  「◯◯実況」                               │
│ │ 再生リスト│  配信者サムネイル  配信者名               │
│ │サムネイル │                                          │
│ │          │  [ステータスバッジ] ▶️ 視聴中              │
│ └──────────┘                                          │
│ ─────────────────────────────────────────────────      │
│  最後に再生した動画                                     │
│  ┌────────────┐  動画タイトル（Part 8「〇〇編」）       │
│  │ 動画       │  進捗バー                              │
│  │ サムネイル │                                        │
│  └────────────┘                                       │
└────────────────────────────────────────────────────────┘
```

### 5.2 再生リスト情報エリア（親）

| 表示要素 | 詳細 |
|---------|------|
| 再生リストのサムネイル | YouTube再生リストのサムネイル画像 |
| 再生リストのタイトル | 最大1〜2行、超過時は省略記号（…） |
| 配信者のサムネイル | 小さな円形のチャンネルアイコン |
| 配信者名 | チャンネル名 |
| 視聴ステータスバッジ | 現在のステータスをアイコン＋テキストで表示（例: ▶️ 視聴中） |

### 5.3 最後に再生した動画エリア（子）

親エリアと視覚的に区切り（境界線または背景色の差異）を設け、「この再生リストの中の動画」であることを明示する。

| 表示要素 | 詳細 |
|---------|------|
| 動画サムネイル | 最後に開いた動画のサムネイル画像 |
| 進捗バー | サムネイル下部に視聴進捗%をバー形式で表示（詳細仕様は「ゲムコメ動画プレーヤー仕様書」の「右カラム＞動画リスト＞進捗バー」に準ずる） |
| 動画タイトル | 最後に開いた動画のタイトル |

### 5.4 最後の動画を視聴済みの場合の表示

再生リストの最終動画を視聴済みの場合、子エリア（最後に再生した動画）は非表示になる。

**新着動画がある場合の通知**

YouTubeの再生リストに新しい動画が追加されている場合、子エリアの代わりに新着通知を表示する。

```
┌────────────────────────────────────────────────────────┐
│ ┌──────────┐  「◯◯実況」                               │
│ │ 再生リスト│  配信者サムネイル  配信者名               │
│ │サムネイル │                                          │
│ └──────────┘  ✅ 完走                                  │
│ ─────────────────────────────────────────────────      │
│  🔔 NEW  新着動画があります                             │
└────────────────────────────────────────────────────────┘
```

| 表示要素 | 詳細 |
|---------|------|
| アイコン | 🔔（ベルアイコン） |
| テキスト | 「NEW」の文字 + 「新着動画があります」 |
| 話数 | **非表示**（話数は表示しない） |

**新着通知の表示制御**

新着通知の表示はユーザーがトグルでON/OFFを制御できる。

| 設定 | 動作 |
|------|------|
| ON（デフォルト） | 新着動画がある場合に🔔 NEW を表示する |
| OFF | 新着通知を非表示にする（完走後は子エリアが空のまま） |

設定の保存先：
- ログイン済みユーザー：Firestoreのユーザードキュメントに保存
- 未ログインユーザー：（マイリストページはログイン必須のため対象外）

### 5.5 逆順トグル

再生リストによっては動画の並びが逆順（新しい動画がPart 1に近い位置に登録されている）の場合があるため、各カードに逆順トグルを設置する。

| 設定 | 動作 |
|------|------|
| OFF（デフォルト） | 再生リスト本来の順序で表示・再生 |
| ON | 再生リストの順序を反転して表示・再生 |

**保存範囲**: ユーザーごと、かつ再生リスト単位で保存（動画プレーヤーページの逆順トグルと設定を共有する）

### 5.6 カードのクリック動作

カードをクリックすると、その再生リストのページ（動画プレーヤーページ）に遷移する。

- 遷移先では「最後に開いた話数」から再生を開始できる（動画プレーヤー仕様書に準ずる）

---

### 5.7 ステータスの自動更新

視聴進捗に応じた自動ステータス変更は**行わない**。ステータスはユーザーが手動で変更する。

> **設計思想**: 自動変更はユーザーの意図と乖離する場合があるため、シンプルに手動管理とする。

### 5.8 ステータス変更のUI

各カード上に「ステータス変更」のUIを設ける（例: バッジをクリックするとステータス選択ドロップダウンが開く）。

## 6. TOPページとの連携

TOPページには、マイリストの情報をもとにした以下のセクションを設ける。詳細な仕様は別途定義する。

| セクション | 内容 |
|-----------|------|
| 視聴中の動画 | マイリストの「視聴中」ステータスの再生リストのうち、最後に再生した動画を表示 |
| 新着動画 | マイリストに登録した再生リストに新着動画がある場合に表示 |
