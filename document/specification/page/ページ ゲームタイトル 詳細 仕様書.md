## 1. 概要

### 1.1 機能概要

「ゲームタイトル詳細ページ」は、「ゲームタイトルから探す」ページでユーザーが選択したゲームタイトルの詳細情報と、そのゲームに紐づく再生リスト一覧を掲載するページである。

ユーザーはこのページを通じて、特定のゲームタイトルに関する基本情報（パッケージ画像・ジャンル・タグ）を確認しつつ、そのゲームに関連する再生リストへ到達できる。また、未登録の再生リストの登録を促す導線も設けることで、コンテンツ充実を図る。

### 1.2 アクセス条件

|条件        |内容                                 |
|----------|-----------------------------------|
|ログイン済みユーザー|✅ アクセス可能                           |
|未ログインユーザー |✅ アクセス可能（閲覧は可能。タグ編集・再生リスト追加はログイン必須）|

### 1.3 ページURL

|項目         |内容                      |
|-----------|------------------------|
| URL | `/games/[gameId]` |
|ヘッダーナビゲーション|常に表示（ログイン状態を問わず）        |

## 3. ゲームタイトルセクション

### 3.1 レイアウト方針

PCとスマートフォンでレイアウト構造が異なる。

|デバイス   |構造                                                                                         |
|-------|-------------------------------------------------------------------------------------------|
|PC     |ゲームタイトルを全幅で表示した後、パッケージエリア（左）と基本情報エリア（右）を2カラムで横並び表示                                         |
|スマートフォン|ゲームタイトル・ジャンル・テーマ・再生リスト数・総動画数・タグ・編集ボタンを上部に縦積みで表示した後、パッケージ画像・楽天ブックスで見るボタン・楽天商品名をまとめたカードを下部に配置|

### 3.2 PC表示レイアウト

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  実況パワフルプロ野球（全幅でタイトル表示）                       │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  ┌──────────────────┐  │  ジャンル：スポーツ・レース             │
│  │                  │  │  テーマ：野球                          │
│  │   パッケージ画像  │  │                                       │
│  │     （縦長）      │  │  □ 3 再生リスト  ▶ 45 動画           │
│  │                  │  │                                       │
│  │                  │  │  [野球] [スポーツ] [ストーリー] [✏️]  │
│  │ [楽天ブックスで   │  │                                       │
│  │   見る ↗]        │  │  コナミが贈る野球シミュレーションの…   │
│  └──────────────────┘  │  [AIによる生成]                        │
│                         │                                       │
│                         │  [ゲームタイトル情報の編集を提案する]   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

左カラム（パッケージエリア）にはパッケージ画像と楽天ブックスで見るボタンのみを配置する。右カラム（基本情報エリア）にはジャンル・テーマ・再生リスト数・総動画数・タグ・説明文・情報編集ボタンを配置する。ゲームタイトルは2カラムの上部に全幅で独立して表示する。

### 3.3 表示要素の仕様

#### ゲームタイトルのサムネイル

|項目    |仕様                              |
|------|--------------------------------|
|取得元   |楽天ブックスAPIから取得したパッケージ画像          |
|アスペクト比|約3:4（縦長）                        |
|未取得時  |デフォルトのプレースホルダー画像を表示             |
|クリック動作|楽天ブックス商品詳細ページ（アフィリエイトリンク）へ別タブで遷移|

#### 楽天ブックスで見るボタン

|項目       |仕様                       |
|---------|-------------------------|
|ボタン文言    |「楽天ブックスで見る ↗」            |
|遷移先      |楽天ブックス商品詳細ページ（アフィリエイトリンク）|
|開き方      |別タブ（`target="_blank"`）   |
|アフィリエイトキー|別途定義                     |


> **楽天ブックスを使用する理由**: 家庭用ゲーム機の日本語ゲームパッケージ画像を取得できること、アフィリエイトリンクによる収益ルートを設けられること、商品カテゴリの親子構造が単一のルールで定義されていること、および日本人向けサービスとして最適な国内データベースであることによる。海外データベースは英語パッケージのため使用しない。なお、取得情報の利用は楽天アフィリエイトの規定に準拠する。

#### ゲームタイトル

|項目  |仕様             |
|----|---------------|
|表示内容|ゲームタイトル名（DB登録値）|
|最大行数|制限なし（改行して全文表示） |

#### ジャンル

|項目    |仕様                                        |
|------|------------------------------------------|
|表示形式  |テキスト。複数の場合は「/」区切りで表示（例：「スポーツ・レース / アクション」）|
|データ取得元|DBのジャンルマスタ                                |

#### テーマ

|項目    |仕様                                  |
|------|------------------------------------|
|表示位置  |ジャンルの直下                             |
|表示形式  |テキスト。複数の場合は「/」区切りで表示（例：「野球 / eスポーツ」）|
|データ取得元|DBのテーママスタ                           |
|未登録時  |表示しない                               |

#### 再生リスト数・総動画数

|項目         |仕様                                         |
|-----------|-------------------------------------------|
|表示位置       |ジャンルの下、タグの上                                |
|再生リスト数の表示形式|再生リストを示すアイコン +「再生リスト」テキスト + 数値（例：□ 3 再生リスト）|
|総動画数の表示形式  |動画再生を示すアイコン +「動画」テキスト + 数値（例：▶ 45 動画）      |
|再生リスト数の定義  |このゲームタイトルに紐づく再生リストの登録数合計                   |
|総動画数の定義    |このゲームタイトルに紐づく全再生リストの動画数合計                  |
|アイコン       |実装者に委ねる（「ゲームタイトルから探す」のカード表示と統一すること）        |

#### ゲームに付けられたタグ

|項目      |仕様                                               |
|--------|-------------------------------------------------|
|表示形式    |バッジ形式で全件表示（件数制限なし）                               |
|表示優先順位  |1. 編集不可タグ（管理者（オーナー・運営者）が設定したロックタグ）、2. 編集可能タグ（登録日が古い順）      |
|クリック動作  |「ゲームタイトルから探す」ページへ遷移し、クリックしたタグが検索条件として適用された状態で表示する|
|ログイン必須操作|なし（閲覧・クリックは未ログインでも可能）                            |

#### ゲームタイトルの説明文

|項目   |仕様               |
|-----|-----------------|
|表示位置 |タグ一覧の下・情報編集ボタンの上 |
|文字数目安|200文字程度          |
|表示条件 |説明文が登録されている場合のみ表示|

**AIによる生成バッジの表示条件**

|状態                    |表示                    |
|----------------------|----------------------|
|説明文がAI生成かつバッジ表示トグルがON |説明文の直下に「AIによる生成」バッジを表示|
|説明文がAI生成かつバッジ表示トグルがOFF|説明文のみ表示。バッジは非表示       |
|説明文が手動入力              |バッジは表示しない             |

**AIによる生成バッジのマウスオーバー**

|項目      |仕様                               |
|--------|---------------------------------|
|ツールチップ文言|「AIによる生成のため、誤りを含む可能性があります」       |
|対象      |説明文テキストのみ。パッケージ画像・ジャンル・テーマには適用しない|


> **バッジの適用範囲について**: パッケージ画像・ジャンル・テーマは楽天ブックスAPIから取得した情報であるため、AIによる生成バッジの対象外とする。バッジは説明文にのみ付与する。

#### タグ編集アイコン

|項目  |仕様                                                 |
|----|---------------------------------------------------|
|表示位置|タグ一覧の末尾に隣接して表示                                     |
|表示条件|ログイン済みユーザーのみ表示                                     |
|動作  |クリックするとタグ編集モーダルを開く（詳細は [第8章](#8-一般ユーザーによるタグ編集) に定義）|
|アイコン|鉛筆アイコン（✏️）等、編集を示すアイコン（実装者に委ねる）                      |

#### 情報を編集ボタン

ログイン済みのユーザー種別によってボタン文言が変わる。

|ユーザー種別 |ボタン文言              |
|-------|-------------------|
|未ログイン  |表示しない              |
|一般ユーザー |「ゲームタイトル情報の編集を提案する」|
|管理者（オーナー・運営者）|「ゲームタイトル情報を編集する」   |

- 一般ユーザーが「編集を提案する」を押した場合は審査フローへ進む（本仕様書では定義しない。別途仕様書を作成予定）。
- 管理者（オーナー・運営者）が「情報を編集する」を押した場合は [第7章](#7-管理者（オーナー・運営者）による情報編集) の編集フローへ進む。

### 4.2 カードの仕様

カードのデザインおよび表示要素は「再生リストを探す」仕様書（第3章）で定義したカードと同一とする。

### 4.3 ソート仕様

|ソート       |説明                     |デフォルト  |
|----------|-----------------------|-------|
|スコアが高い順   |加重平均スコアの高い順に表示         |✅ デフォルト|
|マイリスト登録者数順|マイリストへの登録者数が多い順に表示     |—      |
|新着順       |再生リスト内の最新動画の投稿日が新しい順に表示|—      |

### 4.4 再生リストを追加するボタン

ログイン済みのユーザー種別によってクリック後の動作が変わる。遷移先は共通だが、「再生リストを追加する」ページ側でユーザー種別を判定し、管理者（オーナー・運営者）は登録フロー・一般ユーザーは提案フローへ分岐する（「再生リストを追加する」仕様書 第1章・第5章参照）。

|ユーザー種別 |表示・動作                                                          |
|-------|---------------------------------------------------------------|
|未ログイン  |ボタンを表示。クリック時はログインを促すダイアログを表示、またはログインページへ遷移                     |
|一般ユーザー |ボタンを表示。クリック時は「再生リストを追加する」ページへ遷移し、ゲームタイトルがあらかじめ入力された状態で開く（提案フロー）|
|管理者（オーナー・運営者）|ボタンを表示。クリック時は「再生リストを追加する」ページへ遷移し、ゲームタイトルがあらかじめ入力された状態で開く（登録フロー）|

|項目   |仕様                        |
|-----|--------------------------|
|ボタン文言|「再生リストを追加する」（ユーザー種別によらず共通）|
|表示位置 |セクション上部、ソートUIの右側（または下部）   |
|表示条件 |常時表示                      |

## 7. 管理者（オーナー・運営者）による情報編集

### 7.1 編集フローの概要

管理者（オーナー・運営者）が「ゲームタイトル情報を編集する」ボタンをクリックすると、インライン編集フォームまたは編集モーダルが開く（実装方式は実装者に委ねる）。

編集フローは以下のとおり。

```
[情報を編集する] クリック
       ↓
   編集フォームを表示
       ↓
   JANコードを入力
       ↓
   楽天ブックスAPIでリアルタイム取得（結果を画面に表示）
       ↓
   取得情報をもとにAIが説明文を自動生成（ライブ表示）
       ↓
   ジャンル・テーマをマスタと照合（詳細は 7.5節 参照）
       ↓
   管理者（オーナー・運営者）が各項目を確認・修正
       ↓
   [保存] ボタンで一括保存
```

### 7.2 編集可能な項目

|項目           |説明                                                     |入力可能条件       |
|-------------|-------------------------------------------------------|-------------|
|楽天ブックス JANコード|商品のJANコード（13桁）。このコードをもとにAPIで商品情報を自動取得                  |常時入力可        |
|楽天ブックス 商品名   |APIから自動取得した商品名。手動での上書き修正も可                             |JANコード入力後に入力可|
|楽天ブックス URL   |APIから自動取得した商品ページURL                                    |JANコード入力後に入力可|
|ジャンル（複数選択可）  |DBのジャンルマスタで定義されたリストから選択。楽天ブックスのカテゴリ情報と照合できた場合は自動入力     |常時入力可        |
|テーマ（複数選択可）   |DBのテーママスタで定義されたリストから選択。楽天ブックスのカテゴリ情報と照合できた場合は自動入力      |常時入力可        |
|タグ（複数選択可）    |編集不可の固定タグと、編集可能なタグの両方を登録できる                            |常時入力可        |
|ゲームタイトルの説明文  |200文字程度のゲーム紹介テキスト。JANコード入力後にAIが自動生成する。手動でのフリー入力・上書き修正も可|JANコード入力後に入力可|

> **JANコードを使用する理由**: 楽天ブックスAPIの仕様上、URLからでは在庫がない商品の情報が正しく参照できない問題があるため、JANコードをキーとしてAPI経由で情報を取得する。

|発売日       |商品の発売年月日                               |2024年07月18日                                |
|関連作品      |関連する再生リスト・作品名                           |実況パワフルプロ野球                                 |
|販売元       |販売会社名                                  |コナミデジタルエンタテインメント                           |
|対応機種等     |対応するゲーム機                               |Nintendo Switch                            |
|プレーヤー人数   |対応プレーヤー人数                              |1人～4人                                      |

### 7.4 AI説明文の自動生成

#### 生成タイミング

JANコード入力後、楽天ブックスAPIからの情報取得が完了した直後に説明文の自動生成を開始し、生成テキストをリアルタイムでフォームに表示（ストリーミング表示）する。

#### AIへのインプット

楽天ブックスAPIから取得できた以下の情報をすべてAIへのインプットとして使用する。

|項目   |内容                  |
|-----|--------------------|
|商品名  |楽天ブックスから取得したゲームタイトル名|
|ジャンル |カテゴリパスから抽出したジャンル    |
|テーマ  |カテゴリパスから抽出したテーマ     |
|対応機種等|対応ゲーム機の情報（取得できた場合）  |
|関連作品 |再生リスト・関連作品名（取得できた場合） |
|販売元  |販売会社名（取得できた場合）      |
|発売日  |発売年月日（取得できた場合）      |


> **インプット方針**: 取得できた情報をすべてAIに渡し、AI側で説明文として適切な内容に整形させる。取得できなかった項目はインプットから省略する。

#### 生成テキストの仕様

|項目   |仕様                                                        |
|-----|----------------------------------------------------------|
|目標文字数|200文字程度                                                   |
|言語   |日本語                                                       |
|内容方針 |ゲームの特徴・ジャンル・再生リスト背景等をもとにした紹介文。断定的な誤情報を避け、楽天情報から読み取れる範囲で記述する|

#### 生成後の操作

管理者（オーナー・運営者）は生成されたテキストをそのまま保存するか、テキストエリアを直接編集して修正・上書きできる。手動で編集した場合もAI生成フラグの状態（ON/OFF）は変わらないため、管理者（オーナー・運営者）が必要に応じて手動でトグルを切り替える。

#### AI生成バッジ表示トグルの仕様

|項目         |仕様                                                    |
|-----------|------------------------------------------------------|
|表示条件       |管理者（オーナー・運営者）が編集フォームを開いた場合のみ表示。一般ユーザーおよび未ログインユーザーには非表示|
|デフォルト状態    |AI生成直後はON（バッジ表示）                                      |
|トグルON      |ユーザー向けページに「AIによる生成」バッジを表示する                           |
|トグルOFF     |バッジを非表示にする（説明文自体は引き続き表示される）                           |
|手動入力時のデフォルト|JANコード入力前から説明文を手動入力した場合はOFF（バッジ非表示）をデフォルトとする          |

### 7.5 ジャンル・テーマのマスタ照合とバリデーション

JANコード入力後、楽天ブックスAPIから取得したカテゴリ情報（ジャンル・テーマ）をDBのジャンルマスタ・テーママスタと照合する。

#### 照合結果に応じた挙動

|照合結果              |ジャンルフィールド        |テーマフィールド         |保存ボタン   |
|------------------|-----------------|-----------------|--------|
|ジャンル・テーマともにマスタと一致 |一致したジャンルを自動選択    |一致したテーマを自動選択     |活性（保存可能）|
|ジャンルは一致・テーマはマスタにない|一致したジャンルを自動選択    |空欄（自動入力なし）+ エラー表示|非活性     |
|ジャンルがマスタにない       |空欄（自動入力なし）+ エラー表示|—                |非活性     |
|ジャンル・テーマ情報が取得できない |自動入力なし（管理者（オーナー・運営者）が手動選択） |自動入力なし（管理者（オーナー・運営者）が手動選択） |活性（保存可能）|


> **カテゴリ情報が取得できない場合（最終行）について**: 楽天ブックスAPIからカテゴリ情報自体が返ってこない商品は、そもそも照合対象がないためバリデーションエラーとしない。管理者（オーナー・運営者）がジャンル・テーマを手動で選択して保存する。

#### エラー表示の仕様

|項目           |仕様                                             |
|-------------|-----------------------------------------------|
|エラー表示位置      |ジャンルまたはテーマの入力フィールド直下                           |
|ジャンルのエラーメッセージ|「取得したジャンル「{ジャンル名}」はマスタに登録されていません。マスタを確認してください。」|
|テーマのエラーメッセージ |「取得したテーマ「{テーマ名}」はマスタに登録されていません。マスタを確認してください。」  |
|保存ボタンの状態     |エラーが1件でも存在する間は非活性（`disabled`）とする               |
|エラーの解消条件     |管理者（オーナー・運営者）がジャンル・テーマをマスタ内の値に手動修正したとき                   |


> **エラーを保存ブロックとする理由**: マスタ外の値がDBに登録されると、ジャンルフィルターや表示の整合性が崩れるため、マスタとの一致を登録の必須条件とする。

### 7.6 タグの固定タグ設定

管理者（オーナー・運営者）がタグを登録する際、各タグに対して「固定タグ（編集不可）」か「通常タグ（編集可能）」かを設定できる。固定タグは一般ユーザーによる編集・削除ができない。

### 7.7 保存時の処理

「保存」ボタンクリック時に以下の処理を行う。

- 編集内容をFirestoreのゲームタイトルドキュメントへ保存する
- 楽天ブックスAPIから取得したすべての情報（取得できた項目）を別フィールドに記録する
- 説明文テキストおよびAI生成バッジ表示フラグ（ON/OFF）を保存する
- 保存成功時はトーストメッセージ等で完了を通知し、編集フォームを閉じる
- 保存失敗時はエラーメッセージを表示する

### 8.4 タグ入力サジェストの仕様

#### 概要

タグ乱立を抑制しつつ、マスタにない新しいタグも受け入れるための補助機能。タグマスタに登録された定番タグを候補として提示し、既存タグとの表記ゆれによる重複登録を防ぐ。

#### サジェストの動作

```
[ ホ               ]   ← 1文字以上入力
┌─────────────────┐
│ ホラー           │   ← マスタの前方一致候補（最大5件）
│ ...             │
└─────────────────┘
```

|項目         |仕様                                           |
|-----------|---------------------------------------------|
|表示タイミング    |入力フィールドに1文字以上入力した時点でサジェストを表示する               |
|マッチング方式    |前方一致（入力文字列で始まるタグのみ候補とする）                     |
|候補ソース      |DBのタグマスタに登録されたタグ                             |
|最大表示件数     |5件                                           |
|候補の並び順     |タグ名の五十音順（昇順）                                 |
|候補の選択      |クリック（タップ）で即時追加。選択と同時に入力フィールドをクリアしサジェストを閉じる   |
|サジェストの非表示条件|入力が0文字になったとき、または候補が0件のとき                     |
|マスタ外のタグ入力  |候補が0件または候補以外の値でも、Enterキーまたは追加ボタンで新規タグとして追加できる|

#### 既登録タグの除外

このゲームタイトルに既に付与されているタグはサジェスト候補から除外する。追加済みのタグを誤って重複選択することを防ぐ。

#### サジェストと新規タグの関係

サジェストはあくまで補助であり、入力を制限するものではない。マスタに存在しないタグも自由に追加できる。新規追加されたタグはDBに登録され、以降のサジェスト候補には含まれない（サジェスト候補はタグマスタのみを参照する）。
