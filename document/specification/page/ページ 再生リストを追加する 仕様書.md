## 1. 概要

### 1.1 機能概要

「再生リストを追加する」は、ゲムコメにゲーム実況の再生リストを登録するための機能である。YouTubeの再生リストURLとゲームタイトルを入力することで、ゲムコメ上に新しい実況シリーズを追加できる。

ユーザー種別によって操作の結果が異なる。

| ユーザー種別 | 操作の結果 |
|------------|-----------|
| 管理者（オーナー・運営者） | 入力内容がそのまま本番DBに登録される |
| 一般ユーザー | 入力内容が「提案」としてFirestoreの専用提案コレクションに保存される。管理者（オーナー・運営者）が承認するまでサイト上には公開されない |

### 1.2 アクセス条件

| 条件 | 内容 |
|------|------|
| 管理者（オーナー・運営者）アカウント（Firebase Auth）でログイン済み | ✅ アクセス可能（登録フロー） |
| 一般ユーザーでログイン済み | ✅ アクセス可能（提案フロー） |
| 未ログインユーザー | ❌ アクセス不可（ログインページへリダイレクト） |

> **補足**: アクセス制御はFirebase Authのロール判定による。未ログインユーザーがURLに直接アクセスした場合もリダイレクトする。

### 1.3 ページURL

| 項目 | 内容 |
|------|------|
| URL | `/playlists/new` |
| ヘッダーナビゲーション | 表示する |

### 1.4 流入元

本ページへの導線は以下の3箇所から提供される。管理者（オーナー・運営者）・一般ユーザーともに同じ導線を使用する。

| 流入元 | 導線 | 備考 |
|--------|------|------|
| ゲームタイトル詳細ページ | 「再生リストを追加する」ボタン | ゲームタイトルがデフォルト指定された状態で遷移 |
| チャンネル詳細ページ | 「再生リストを追加する」ボタン | チャンネルが固定指定された状態で遷移 |
| 再生リストを探すページ | 「再生リストを追加する」ボタン | 特になし |

### 2.1 全体構成

ページは **1カラム構成** で配置される。管理者（オーナー・運営者）と一般ユーザーでレイアウトの差異はない。

```

│                                                             │
│  再生リストを追加する                                         │
│                                                             │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ 【案内エリア】                                         │  │

│  └───────────────────────────────────────────────────────┘  │

│  ┌───────────────────────────────────────────────────────┐  │

└─────────────────────────────────────────────────────────────┘
```

## 3. 流入元による表示の違い

### 3.1 通常流入（再生リストを探すページから）

案内エリアは表示しない。フォームのみ表示する。

### 3.2 ゲームタイトル詳細ページから流入した場合

ページ上部にゲームタイトルを示す案内エリアを表示する。ゲームタイトル選択フォームはデフォルトで選択済みの状態になる（変更可能）。

```
┌───────────────────────────────────────────────────────┐
│ 🎮  「ゼルダの伝説 ブレス オブ ザ ワイルド」の          │
│      再生リストを追加する                               │
└───────────────────────────────────────────────────────┘
```

| 要素 | 内容 |
|------|------|
| アイコン | 🎮（ゲームコントローラー） |
| テキスト | 「{ゲームタイトル名}」の再生リストを追加する |
| ゲームタイトル選択フォーム | 対象ゲームタイトルがデフォルト選択済み（変更可能） |

### 3.3 チャンネル詳細ページから流入した場合

ページ上部にチャンネルを示す案内エリアを表示する。フォームはチャンネルに関する項目を持たないが、登録・提案処理時にチャンネルの一致確認を行う（第5章参照）。

```
┌───────────────────────────────────────────────────────┐
│ ⚬ チャンネルアイコン  「◯◯ゲームズ」の                 │
│                       再生リストを追加する              │
└───────────────────────────────────────────────────────┘
```

| 要素 | 内容 |
|------|------|
| チャンネルアイコン | 円形のチャンネルサムネイル |
| テキスト | 「{チャンネル名}」の再生リストを追加する |
| フォームへの影響 | フォーム自体の変更なし。ただし処理時にチャンネル一致確認を実施 |

## 4. フォーム仕様

### 4.1 再生リストURL入力

| 項目 | 仕様 |
|------|------|
| 入力形式 | テキスト入力フィールド |
| プレースホルダー | 「https://www.youtube.com/playlist?list=...」 |
| 取得タイミング | フォーカスが外れた（blur）タイミングで自動的にYouTube APIへ情報取得リクエストを送信 |
| 有効なURL形式 | YouTubeの再生リストURL（`playlist?list=` を含むURL）のみ |

#### 4.1.1 再生リスト確認エリア

URLが有効な場合、入力フィールドの下に再生リストの情報を自動表示する。

```
┌───────────────────────────────────────────────────────────┐
│ ┌──────────────┐  再生リストタイトル                       │
│ │              │                                          │
│ │ 再生リスト   │  ⚬ チャンネルアイコン  チャンネル名       │
│ │ サムネイル   │                                          │
│ │ （16:9）     │  動画数: 42本                            │
│ │              │                                          │
│ └──────────────┘                                          │
└───────────────────────────────────────────────────────────┘
```

| 表示要素 | 詳細 |
|---------|------|
| 再生リストのサムネイル | YouTube APIから取得したサムネイル画像（アスペクト比 16:9） |
| 再生リストのタイトル | YouTube APIから取得した再生リストのタイトル |
| チャンネルアイコン | 小さな円形のチャンネルサムネイル |
| チャンネル名 | YouTubeチャンネル名 |
| 動画数 | 再生リストに登録されている動画数（例：「動画数: 42本」） |

#### 4.1.2 取得中の表示

YouTube APIへのリクエスト中は、確認エリアにローディングインジケーターを表示する。

#### 4.1.3 取得後のチャンネル状態表示

YouTube APIからチャンネル情報を取得した後、そのチャンネルがゲムコメに登録済みかどうかを確認し、確認エリア内に状態を表示する。

| 状態 | 表示内容 |
|------|---------|
| 登録済みチャンネル | 「✅ ゲムコメ登録済みチャンネルです。再生リストと紐づけます。」 |
| 未登録チャンネル | 「ℹ️ 未登録のチャンネルです。再生リスト登録と同時に新規チャンネルとして登録します。」 |

#### 4.1.4 未登録チャンネルの説明文自動生成（管理者（オーナー・運営者）のみ）

チャンネルが未登録の場合、チャンネル状態表示の直後にAIによるチャンネル説明文の自動生成を開始する。生成テキストは確認エリア内にリアルタイムでストリーミング表示する。

**生成タイミング**

YouTube APIからのチャンネル情報取得が完了し、かつそのチャンネルがゲムコメ未登録と判定された直後に自動生成を開始する。登録済みチャンネルの場合は生成しない。

**AIへのインプット**

| 項目 | 内容 |
|------|------|
| チャンネル名 | YouTube APIから取得したチャンネル名 |
| チャンネルの説明文 | YouTube APIから取得したチャンネル説明文（YouTubeに登録されているもの） |

> **インプット方針**: 取得できた情報をすべてAIに渡し、AI側でゲームコメンタリーチャンネルとして適切な紹介文に整形させる。

**生成テキストの仕様**

| 項目 | 仕様 |
|------|------|
| 目標文字数 | 200文字程度 |
| 言語 | 日本語 |
| 内容方針 | チャンネルの特徴・実況スタイル・扱うゲームジャンル等をもとにした紹介文。YouTubeの説明文から読み取れる範囲で記述し、断定的な誤情報を避ける |

**確認エリアへの表示レイアウト（未登録チャンネル時）**

```
┌───────────────────────────────────────────────────────────┐
│ ┌──────────────┐  再生リストタイトル                       │
│ │              │                                          │
│ │ 再生リスト   │  ⚬ チャンネルアイコン  チャンネル名       │
│ │ サムネイル   │                                          │
│ │ （16:9）     │  動画数: 42本                            │
│ │              │                                          │
│ └──────────────┘                                          │
│                                                           │
│ ℹ️ 未登録のチャンネルです。再生リスト登録と同時に新規      │
│    チャンネルとして登録します。                            │
│                                                           │
│ チャンネルの説明文                                         │
│ ┌───────────────────────────────────────────────────────┐ │
│ │ （ストリーミングで生成テキストが表示される）             │ │
│ └───────────────────────────────────────────────────────┘ │
│ [AIによる生成]  AI生成バッジ表示: [ ON / OFF ]             │
└───────────────────────────────────────────────────────────┘
```

**生成後の操作**

管理者（オーナー・運営者）は生成されたテキストをそのまま登録するか、テキストエリアを直接編集して修正・上書きできる。手動で編集した場合もAI生成フラグの状態（ON/OFF）は変わらないため、管理者（オーナー・運営者）が必要に応じて手動でトグルを切り替える。

**AI生成バッジ表示トグルの仕様**

| 項目 | 仕様 |
|------|------|
| 表示条件 | 管理者（オーナー・運営者）の確認エリアにのみ表示。一般ユーザーには表示しない |
| デフォルト状態 | AI生成直後はON（バッジ表示） |
| トグルON | チャンネル詳細ページに「AIによる生成」バッジを表示する |
| トグルOFF | バッジを非表示にする（説明文自体は引き続き表示される） |

### 4.2 ゲームタイトル選択

| 項目 | 仕様 |
|------|------|
| 入力形式 | ボタンクリックでモーダルを開く |
| ボタン文言（未選択時） | 「ゲームタイトルを選択する」 |
| ボタン文言（選択済み時） | 選択したゲームタイトル名を表示（例：「ゼルダの伝説 ブレス オブ ザ ワイルド」） |
| 選択解除 | 選択済み状態のボタン横に「×」を表示。クリックで選択を解除 |

#### 4.2.1 ゲームタイトル選択モーダル

「ゲームタイトルを選択する」ボタンをクリックすると、モーダルが開く。

```
┌─────────────────────────────────────────────────────┐
│ ゲームタイトルを選択                            [ × ] │
├─────────────────────────────────────────────────────┤
│ [ ゲームタイトルを検索...                        × ] │
├─────────────────────────────────────────────────────┤
│                                                     │
│ ┌──────────────────────────────────────────────┐    │
│ │ ┌────┐  ゼルダの伝説 ブレス オブ ザ ワイルド   │    │
│ │ │ 画像│                                      │    │
│ │ └────┘                                      │    │
│ └──────────────────────────────────────────────┘    │
│ ┌──────────────────────────────────────────────┐    │
│ │ ┌────┐  モンスターハンターワールド              │    │
│ │ │ 画像│                                      │    │
│ │ └────┘                                      │    │
│ └──────────────────────────────────────────────┘    │
│  ...                                               │
│                                                     │
├─────────────────────────────────────────────────────┤
│ 探しているゲームタイトルが見つかりませんか？           │
│ [ ゲームタイトルの追加を提案する ]                    │
└─────────────────────────────────────────────────────┘
```

| 項目 | 仕様 |
|------|------|
| レイアウト | リスト形式（1行1件） |
| 表示要素（1件） | 楽天ブックスAPIのパッケージ画像（小さめ）＋ゲームタイトル名 |
| 検索フィールド | モーダル上部に配置。部分一致でリアルタイムフィルタリング（デバウンス: 300ms） |
| 検索対象 | ゲームタイトル名 |
| クリア | 入力値がある場合に「×」ボタンを表示。クリックで全件表示に戻す |
| 選択動作 | 行をクリックするとモーダルが閉じ、選択したゲームタイトルがボタンに反映される |
| 閉じる | 右上の「×」ボタンまたはモーダル外のクリックで閉じる |
| フッターエリア | モーダル最下部に「探しているゲームタイトルが見つかりませんか？」というテキストと「ゲームタイトルの追加を提案する」ボタンを常時表示する。クリックするとゲームタイトル追加提案モーダルへ切り替わる（4.2.2参照） |

> **フッターを常時表示する理由**: 検索結果が0件のときのみ表示する方式では、見つからないユーザーが検索キーワードを変えながら悩んでいる状態では目に留まりにくい。常時表示することで、詰まったタイミングでいつでも提案できる導線を確保する。

#### 4.2.2 ゲームタイトル追加提案モーダル（一般ユーザーのみ）

「ゲームタイトルの追加を提案する」ボタンをクリックすると、同一モーダル内の表示をゲームタイトル選択からゲームタイトル追加提案フォームへ切り替える。

```
┌─────────────────────────────────────────────────────┐
│ ゲームタイトルの追加を提案する                  [ × ] │
├─────────────────────────────────────────────────────┤
│                                                     │
│ 追加してほしいゲームタイトルの情報を入力してください。 │
│ 確認後、ゲムコメへの登録を検討します。               │
│                                                     │
│ 追加したい再生リストのURL *                          │
│ [ https://www.youtube.com/playlist?list=...     ]   │
│                                                     │
│ ゲームタイトル名 *                                   │
│ [ テキスト入力                                  ]   │
│                                                     │
│ 備考（任意）                                         │
│ [ テキストエリア                                ]   │
│                                                     │
│ [ 戻る ]           [ 提案を送信する ]                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**フォーム項目**

| 項目 | 必須 | 種別 | 備考 |
|------|:---:|------|------|
| 追加したい再生リストのURL | ✅ | テキスト入力 | YouTubeの再生リストURL形式のみ有効。再生リスト追加フォームで入力済みのURLがある場合は自動入力する |
| ゲームタイトル名 | ✅ | テキスト入力 | 最大100文字 |
| 備考 | — | テキストエリア | 最大500文字。発売機種・再生リスト情報など補足があれば記入 |

**バリデーション**

| チェック内容 | エラーメッセージ |
|------------|--------------|
| 再生リストURLが未入力 | 「再生リストのURLを入力してください」 |
| 再生リストURLの形式が不正 | 「YouTubeの再生リストURLを入力してください」 |
| ゲームタイトル名が未入力 | 「ゲームタイトル名を入力してください」 |
| ゲームタイトル名が文字数超過 | 「100文字以内で入力してください」 |
| 備考が文字数超過 | 「500文字以内で入力してください」 |

**送信後の動作**

提案データをFirestoreの専用コレクションに保存し、モーダル内に以下の完了メッセージを表示する。その後モーダルを閉じても再生リスト追加フォームの状態はリセットされない。

```
┌─────────────────────────────────────────────────────┐
│ ゲームタイトルの追加を提案する                  [ × ] │
├─────────────────────────────────────────────────────┤
│                                                     │
│  📨 提案を受け付けました                             │
│                                                     │
│  ありがとうございます！                              │
│  内容を確認後、ゲームタイトルの登録を検討します。     │
│  登録後は再生リストの提案が可能になります。           │
│                                                     │
│                          [ 閉じる ]                 │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**保存内容**

| 項目 | 内容 |
|------|------|
| 保存先 | Firestoreの専用コレクション（コレクション名は実装者に委ねる） |
| 保存データ | 再生リストURL・ゲームタイトル名・備考・提案ユーザーID・提案日時・ステータス（「審査中」） |
| 承認フロー | 管理者（オーナー・運営者）による審査・登録処理は管理_審査ワークフロー仕様書に準ずる |

> **対象ユーザー**: 本機能は一般ユーザーのみ対象。管理者（オーナー・運営者）はゲームタイトルをマスタ管理画面から直接登録できるため、本モーダルは表示しない。

### 4.3 送信ボタン

ユーザー種別によってボタン文言が異なる。

| ユーザー種別 | ボタン文言 |
|------------|----------|
| 管理者（オーナー・運営者） | 「登録する」 |
| 一般ユーザー | 「提案する」 |

| 項目 | 仕様 |
|------|------|
| 活性条件 | 再生リスト確認エリアに情報が表示済み（URLが有効）、かつゲームタイトルが選択済みの場合のみ活性 |
| 非活性時 | グレーアウトし、クリック不可 |
| クリック時 | ユーザー種別に応じた処理を実行（第5章参照） |

## 5. 登録・提案処理

### 5.1 処理フローの分岐

送信ボタンクリック後の処理は、ユーザー種別によって分岐する。

```
送信ボタンをクリック
    ↓
バリデーション（5.2参照）
    ↓
    ├─ 管理者（オーナー・運営者）の場合 ──────────────────────────────┐
    │   チャンネルの処理（5.3参照）                       │
    │       ↓                                           │
    │   再生リストをFirestoreに登録（本番コレクション）    │
    │       ↓                                           │
    │   動画リストをFirestoreに登録                       │
    │       ↓                                           │
    │   管理者（オーナー・運営者）向け完了画面へ遷移（6.1参照）               │
    │                                                   │
    └─ 一般ユーザーの場合 ──────────────────────────────┐
        提案データをFirestoreに保存（提案コレクション）   │
            ↓                                          │
        一般ユーザー向け完了画面へ遷移（6.2参照）        │
```

### 5.2 バリデーション

管理者（オーナー・運営者）・一般ユーザー共通のバリデーション。

| チェック内容 | エラー時の動作 |
|------------|-------------|
| 再生リストURLがYouTube再生リストURLの形式であること | インラインエラー＋トースト通知を表示 |
| YouTube APIからの情報取得が成功していること | インラインエラー＋トースト通知を表示 |
| ゲームタイトルが選択されていること | インラインエラーを表示 |
| チャンネル詳細ページ流入時：入力された再生リストURLのチャンネルが、流入元チャンネルと一致すること | インラインエラー＋トースト通知を表示 |

### 5.3 チャンネルの処理（管理者（オーナー・運営者）のみ）

| チャンネルの状態 | 処理内容 |
|---------------|--------|
| ゲムコメ登録済みチャンネル | 既存チャンネルドキュメントと再生リストを紐づける |
| ゲムコメ未登録チャンネル | YouTube APIから取得したチャンネル情報（チャンネル名・アイコン等）およびAI生成済みの説明文・AI生成バッジフラグを含めて新規チャンネルをFirestoreに登録し、再生リストと紐づける |

### 5.4 提案データの保存仕様（一般ユーザーのみ）

一般ユーザーが「提案する」をクリックした場合、再生リストの情報は本番の再生リストコレクションには保存されず、Firestoreの専用提案コレクションに保存される。

| 項目 | 内容 |
|------|------|
| 保存先 | Firestoreの専用提案コレクション（コレクション名は実装者に委ねる） |
| 保存内容 | 再生リストURL・YouTube APIから取得した再生リスト情報・選択したゲームタイトル・提案ユーザーID・提案日時・ステータス（「審査中」） |
| 公開状態 | 管理者（オーナー・運営者）が承認するまでサイト上には公開されない |
| 承認フロー | 管理者（オーナー・運営者）向けの審査UI・承認処理は別途定義する |

## 7. エラーハンドリング

### 7.1 エラー表示の方針

エラーは以下の2つを組み合わせて表示する。

| 表示方法 | 用途 |
|---------|------|
| インラインエラー | 該当するフォームフィールドの下にエラーメッセージを赤テキストで表示 |
| トースト通知 | 画面上部に一定時間（3秒程度）表示後、自動で消える |

### 7.2 エラーケース一覧

#### 共通エラー（管理者（オーナー・運営者）・一般ユーザー共通）

| エラーケース | インラインメッセージ | トーストメッセージ |
|------------|------------------|----------------|
| 入力URLがYouTube再生リストURLではない | 「YouTubeの再生リストURLを入力してください」 | 「URLの形式が正しくありません」 |
| YouTube APIからの情報取得失敗（存在しないURL等） | 「再生リストが見つかりませんでした。URLを確認してください」 | 「再生リストの取得に失敗しました」 |
| チャンネル詳細から流入時のチャンネル不一致 | 「このチャンネルの再生リストではありません。URLを確認してください」 | 「チャンネルが一致しません」 |
| ゲームタイトル未選択での送信ボタンクリック（※ボタンは非活性のため通常発生しないが念のため） | 「ゲームタイトルを選択してください」 | — |
| ネットワークエラー等 | — | 「送信に失敗しました。時間をおいて再試行してください」 |

#### 管理者（オーナー・運営者）固有エラー

| エラーケース | インラインメッセージ | トーストメッセージ |
|------------|------------------|----------------|
| すでにゲムコメに登録済みの再生リスト | 「この再生リストはすでに登録されています」 | 「登録済みの再生リストです」 |
| AI説明文の生成失敗 | — | 「説明文の自動生成に失敗しました。手動で入力してください」 |

#### 一般ユーザー固有エラー

| エラーケース | インラインメッセージ | トーストメッセージ |
|------------|------------------|----------------|
| すでに同じ再生リストの提案が存在する（審査中または登録済み） | 「この再生リストはすでに提案済み、または登録済みです」 | 「提案済みの再生リストです」 |
