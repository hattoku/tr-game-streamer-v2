## 1. 概要

### 1.1 機能概要

視聴履歴は、ユーザーがゲムコメ上で視聴した動画の履歴を追跡・一覧表示するための機能である。ユーザーの視聴体験を継続的にサポートし、「どこまで見たか」を素早く把握できる場を提供する。

| 状態 | データ取得元 |
|------|------------|
| ログイン済み | Firestore（DB） |
| 未ログイン | LocalStorage（ブラウザ） |

### 1.2 機能のスコープ

- 視聴した動画の履歴表示（日付別グルーピング）
- キーワード検索・日付フィルタリング
- 個別削除・全件削除
- ログイン時のLocalStorageからDBへのマージ

### 1.3 非スコープ

- 視聴履歴をもとにしたレコメンデーション機能（将来検討）
- 視聴履歴のエクスポート機能

## 2. アクセス条件・URL

### 2.1 アクセス権限

| ユーザー状態 | アクセス可否 | 表示内容 |
|------------|------------|--------|
| ログイン済み | ✅ アクセス可能 | DBから取得した視聴履歴を表示 |
| 未ログイン | ✅ アクセス可能 | LocalStorageから取得した視聴履歴を表示 |

> **設計思想**: マイリスト機能（ログイン必須）とは異なり、視聴履歴はログイン前の段階から蓄積・表示されることでユーザー体験を損なわない設計とする。

### 2.2 ページURL

| 項目 | 内容 |
|------|------|
| URL | `/history` |
| ヘッダーナビゲーション | 常に表示（ログイン済み・未ログインを問わず） |

### 2.3 ヘッダーナビゲーションの表示

ヘッダーナビゲーションに「視聴履歴」リンクを常時表示する。ログイン状態によって表示内容が切り替わるが、ページ自体には常にアクセスできる。

### 3.1 全体構成

```

│ 視聴履歴                                                      │

│ ┌───────────────────────────────┬─────────────────────────┐ │

│ │ 視聴履歴一覧                   │ 操作パネル               │ │
│ │                               │                         │ │
│ │ ── 2026年3月5日 ──            │ [ キーワード検索 ]        │ │
│ │ ┌─────────────────────────┐  │                         │ │
│ │ │ カード                   │  │ [ カレンダーフィルター ]  │ │
│ │ └─────────────────────────┘  │                         │ │

```

### 3.2 空状態の表示

視聴履歴が1件もない場合（初回訪問時・全件削除後）は、以下を表示する。

| 要素 | 内容 |
|------|------|
| アイコン | 時計または再生アイコン |
| メッセージ | 「まだ視聴履歴がありません」 |
| サブメッセージ | 「再生リストを開いて動画を視聴すると、ここに記録されます」 |

## 4. 左カラム：視聴履歴一覧

### 4.1 日付グルーピング

視聴履歴は**視聴日ごとにグルーピング**して表示される。

**並び順**: 視聴した日時の新しい順（降順）

**日付ラベルの表示形式**:

| 条件 | 表示例 |
|------|--------|
| 当日 | 「今日」 |
| 前日 | 「昨日」 |
| それ以外 | 「2026年3月3日（火）」 |

同一日付内でのカードの並び順は、視聴した時刻の新しい順（降順）。

### 4.2 カード仕様

視聴履歴の各エントリは1枚のカードとして表示される。

```
┌───────────────────────────────────────────────────────┐
│ ┌──────────────┐  動画タイトル（最大2行、超過時は…）   ⋮ │
│ │  動画         │                                      │
│ │  サムネイル   │  ⚬ 配信者サムネイル  配信者名         │
│ │  （16:9）     │                                      │
│ │ [█████░░░░]  │                                      │
│ └──────────────┘                                      │
└───────────────────────────────────────────────────────┘
```

#### カードに含める表示要素

| 表示要素 | 詳細 |
|---------|------|
| 動画サムネイル | 最後に開いた動画のサムネイル画像（アスペクト比 16:9） |
| 進捗バー | サムネイル下部に視聴進捗%をバー形式で表示（詳細仕様は「ゲムコメ動画プレーヤー仕様書」の「右カラム＞動画リスト＞進捗バー」に準ずる） |
| 動画タイトル | 最後に開いた動画のタイトル（最大2行、超過時は省略記号） |
| 配信者サムネイル | 小さな円形のチャンネルアイコン |
| 配信者名 | YouTubeチャンネル名 |
| 三点リーダー（⋮） | カード右上に配置。クリックで操作メニューを表示 |

#### カードのクリック動作

カード本体（三点リーダー以外）をクリックすると、該当する再生リストの動画プレーヤーページへ遷移する。遷移先では「最後に開いた話数」から再生を開始できる（動画プレーヤー仕様書に準ずる）。

### 4.3 三点リーダーメニュー

各カードの右上に三点リーダー（⋮）を表示する。クリックするとドロップダウンメニューが開く。

| メニュー項目 | 動作 |
|------------|------|
| この動画の視聴履歴を削除 | 確認ダイアログを表示後、該当エントリを削除 |

## 5. 右カラム：操作パネル

右カラムには以下の3つの操作UIを縦並びで配置する。

### 5.1 キーワード検索

| 項目 | 仕様 |
|------|------|
| 入力形式 | テキスト入力フィールド |
| 検索対象 | 動画タイトル、配信者名 |
| 検索タイミング | 入力のたびにリアルタイムでフィルタリング（デバウンス処理を適用） |
| プレースホルダー | 「動画タイトルや配信者名で検索」 |
| クリアボタン | 入力値がある場合に表示。クリックで入力内容をクリアし、全件表示に戻す |

### 5.2 カレンダーフィルター

特定の日付以前に視聴した動画だけを表示するフィルター。

| 項目 | 仕様 |
|------|------|
| 選択方式 | 1つの日付のみ選択（その日を含む、それ以前に視聴した動画を表示） |
| UI形式 | カレンダーピッカー |
| デフォルト状態 | 未選択（全件表示） |
| フィルター解除 | クリアボタンで解除。解除後は全件表示（デフォルト状態）に戻る |
| 選択可能範囲 | 視聴履歴が存在する最古の日付〜今日 |

**表示例（2026年3月3日を選択した場合）**:

```
[カレンダー] 2026年3月3日以前  [×]

→ 2026年3月3日（火）以前に視聴した動画のみ表示
```

### 5.3 すべての履歴を削除

| 項目 | 仕様 |
|------|------|
| ボタン表示 | 常に表示（履歴が0件の場合はグレーアウト） |
| クリック時 | 確認ダイアログを表示 |
| 確認ダイアログの内容 | 「視聴履歴をすべて削除しますか？この操作は取り消せません。」 |
| ダイアログのボタン | 「削除する」（破壊的アクション）／「キャンセル」 |
| 削除後 | 空状態の表示に切り替わる |

### 5.4 フィルターの複合適用

キーワード検索とカレンダーフィルターは**同時に適用可能**。両方が指定されている場合は AND 条件でフィルタリングされる。

## 6. 視聴履歴の記録仕様

### 6.1 記録タイミング

再生リストにある動画の**再生を開始したタイミング**（再生ボタンを押した時点）で視聴履歴に記録される。

具体的には、動画プレーヤー仕様書に定める以下の操作が記録トリガーとなる。

| 操作 | 記録トリガー |
|------|------------|
| 再生ボタンを押す | ✅ 記録 |
| 「次へ」ボタンで遷移 | ✅ 記録 |
| 連続再生による自動遷移 | ✅ 記録 |
| 「前へ」ボタンで遷移 | ✅ 記録 |

### 6.2 記録内容

| データ | 内容 |
|--------|------|
| 動画ID | 視聴した動画の識別子 |
| 再生リストID | 動画が属する再生リストの識別子 |
| 視聴日時 | 記録した時刻（タイムスタンプ） |
| 視聴進捗% | 各動画の視聴進捗（動画プレーヤー仕様書に準ずる） |

### 6.3 同一動画の重複記録

同じ動画を複数回視聴した場合の挙動：

| 項目 | 仕様 |
|------|------|
| 記録方式 | 最新の視聴日時で上書きする（重複エントリは作成しない） |
| 履歴一覧での表示 | 最後に視聴した日付のグループに1件のみ表示される |

## 7. データ保存仕様

### 7.1 ログイン済みユーザー

```
ユーザーが再生ボタンを押す
    ↓
フロントエンド：視聴履歴エントリを生成
    ↓
API呼び出し（Firestore保存）
    ↓
Firestore：視聴履歴コレクションに保存
```

### 7.2 未ログインユーザー

```
ユーザーが再生ボタンを押す
    ↓
フロントエンド：視聴履歴エントリを生成
    ↓
LocalStorage保存
    ↓
ブラウザ内に永続化
```

### 7.3 ログイン時のマージ処理

未ログイン状態でブラウザに蓄積した視聴履歴は、**ログイン後にDBへ自動的にマージ**される。

**マージのタイミング**: ログイン処理完了直後

**マージのルール**:

| 条件 | 処理 |
|------|------|
| LocalStorageに履歴あり、DBに同一動画の履歴なし | LocalStorageの履歴をDBに追加 |
| LocalStorageに履歴あり、DBに同一動画の履歴あり | より新しい視聴日時のデータを優先して保持 |
| LocalStorageに履歴なし | 何もしない |

**マージ後の処理**: マージ完了後、LocalStorage内の視聴履歴データを削除する。

### 7.4 LocalStorageのキー設計

```
viewing_history: [
  {
    videoId: string,
    playlistId: string,
    watchedAt: ISO8601文字列,
    progressPercent: number
  },
  ...
]
```

## 8. 削除仕様

### 8.1 個別削除（三点リーダーから）

**操作フロー**:

```
三点リーダー（⋮）をクリック
    ↓
「この動画の視聴履歴を削除」をクリック
    ↓
確認ダイアログを表示
    ↓
「削除する」をクリック
    ↓
該当エントリを削除
    ↓
一覧からカードが消える
（その日付グループが空になった場合、日付ラベルごと非表示）
```

**確認ダイアログの内容**:

| 項目 | 内容 |
|------|------|
| タイトル | 「視聴履歴を削除しますか？」 |
| メッセージ | 「この動画の視聴履歴を削除します。この操作は取り消せません。」 |
| ボタン | 「削除する」（破壊的アクション）／「キャンセル」 |

### 8.2 全件削除（右カラムから）

**操作フロー**:

```
「すべての履歴を削除」をクリック
    ↓
確認ダイアログを表示
    ↓
「削除する」をクリック
    ↓
全エントリを削除
    ↓
空状態の表示に切り替わる
```

**確認ダイアログの内容**:

| 項目 | 内容 |
|------|------|
| タイトル | 「すべての視聴履歴を削除しますか？」 |
| メッセージ | 「視聴履歴をすべて削除します。この操作は取り消せません。」 |
| ボタン | 「すべて削除する」（破壊的アクション）／「キャンセル」 |

### 8.3 削除対象データ

| ユーザー状態 | 削除対象 |
|------------|--------|
| ログイン済み | Firestore内の該当ユーザーの視聴履歴データ |
| 未ログイン | LocalStorage内の視聴履歴データ |
