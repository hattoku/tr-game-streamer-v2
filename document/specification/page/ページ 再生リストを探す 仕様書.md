## 1. 概要

### 1.1 機能概要

「再生リストを探す」は、ゲムコメに登録されているゲーム実況シリーズの再生リストを検索・発見するためのページである。キーワードやタグを組み合わせて絞り込み、スコアや登録数などでソートした一覧からユーザーが自分に合った再生リストを見つけることができる。

### 1.2 アクセス条件

| 条件 | 内容 |
|------|------|
| ログイン済みユーザー | ✅ アクセス可能 |
| 未ログインユーザー | ✅ アクセス可能（検索・閲覧は可能。マイリスト登録はログイン必須） |

### 1.3 ページURL

| 項目 | 内容 |
|------|------|
| URL | `/playlists` |
| ヘッダーナビゲーション | 常に表示（ログイン状態を問わず） |

│ │ 左カラム                       │ 右カラム                 │ │
│ │ カード一覧                     │ 検索条件パネル           │ │

## 3. 左カラム：カード一覧

### 3.1 一覧ヘッダー

カード一覧の上部に以下を表示する。

| 表示要素 | 詳細 |
|---------|------|
| 検索結果件数 | 「XX件」の形式で表示。検索・フィルター適用後もリアルタイムに更新 |
| ソートUI | 検索結果件数の右側にドロップダウンを設置（ソート仕様は [第5章](#5-ソート仕様) に定義） |

### 3.2 カードのデザイン方針

1枚のカードで、ユーザーが「この再生リストを追いかけたいかどうか」を判断できる情報を過不足なく提供する。

```
┌────────────────────────────────────────────────────────────┐
│ ┌──────────────┐  再生リストのタイトル（最大2行）           │
│ │              │                                          │
│ │  再生リスト  │  ⚬ チャンネルアイコン  チャンネル名       │
│ │  サムネイル  │                                          │
│ │  （16:9）    │  ★ 4.8  👥 1,234  💬 56               │
│ │              │                                          │
│ └──────────────┘  [タグ] [タグ] [タグ]                    │
└────────────────────────────────────────────────────────────┘
```

### 3.3 カードに含める表示要素

| 表示要素 | 詳細 |
|---------|------|
| 再生リストのサムネイル | YouTube再生リストのサムネイル画像（アスペクト比 16:9） |
| 再生リストのタイトル | 最大2行。超過時は省略記号（…）で切り捨て |
| チャンネルアイコン | 小さな円形のチャンネルアイコン |
| チャンネル名 | YouTubeチャンネル名 |
| スコア | ★アイコン + 数値（小数点第1位、例：★ 4.8）。レビューが0件の場合は「評価なし」と表示 |
| マイリスト数 | 👥アイコン + 数値（例：👥 1,234） |
| レビュー数 | 💬アイコン + 数値（例：💬 56） |
| タグ | バッジ形式で最大3件まで表示。4件以上は「+N」の形式で省略（例：+2）。表示順・強調表示の仕様は [3.5節](#35-カード内タグの表示仕様) に定義 |

### 3.4 カードのクリック動作

カードをクリックすると、該当する再生リストの**動画プレーヤーページ**へ遷移する。

- ログイン済みユーザーかつマイリスト登録済みの場合：最後に開いた話数から再生を開始できる（動画プレーヤー仕様書に準ずる）
- 未ログインユーザー・初回訪問の場合：第1話から再生を開始する

### 3.5 カード内タグの表示仕様

#### 表示優先順位

カードに表示するタグは以下の優先順位で並べる。

| 優先度 | 対象 | 説明 |
|--------|------|------|
| 1（最優先） | 編集不可タグ | 再生リストの登録者が設定した、ロック（編集不可）状態のタグ |
| 2 | 編集可能タグ | タグが登録された日付が古い順 |

#### 検索時の強調表示

タグ絞り込みで選択されているタグがカード内に表示されている場合、そのタグを**太字**にして目立つよう表示する。これにより、ユーザーが「なぜこのカードが絞り込み結果に表示されているのか」を視覚的に確認できる。

**例**（タグ「RPG」で絞り込み中の場合）:

```
[アクション] [**RPG**] [協力プレイ]
             ↑ 太字で強調
```

### 3.6 ページネーション

| 項目 | 仕様 |
|------|------|
| 1ページあたりの表示件数 | 20件 |
| 方式 | ページネーション（ページ番号リンク形式） |
| 表示位置 | カード一覧の下部 |

## 4. 右カラム：検索条件パネル

右カラムはページとともにスクロールする（スクロール追従なし）。

### 4.1 パネル構成

```
┌─────────────────────────┐
│ 検索条件                 │
│                         │
│ [ キーワード検索 ]        │
│                         │
│ ──────────────────────  │
│                         │
│ タグで絞り込む            │
│ [タグA] [タグB] [タグC]  │
│ ...                     │
│                         │
│ [ 検索条件をリセット ]    │
└─────────────────────────┘
```

### 4.2 キーワード検索

| 項目 | 仕様 |
|------|------|
| 入力形式 | テキスト入力フィールド |
| 検索対象 | **ゲームタイトル**、**チャンネル名** |
| 検索タイミング | 入力のたびにリアルタイムでフィルタリング（デバウンス処理を適用、遅延目安：300ms） |
| 部分一致 | 対応（入力文字列を含む結果を表示） |
| 大文字小文字 | 区別しない |
| プレースホルダー | 「ゲームタイトル・チャンネル名で検索」 |
| クリアボタン | 入力値がある場合に「×」ボタンを表示。クリックで入力内容をクリアし、全件表示に戻す |

### 4.3 タグ絞り込み

タグの詳細仕様は別途定義する。本仕様書ではUIと動作のみを規定する。

| 項目 | 仕様 |
|------|------|
| 表示形式 | バッジ形式でタグを一覧表示 |
| 選択方式 | 複数選択可能 |
| 選択時の動作 | 選択したタグがすべて付与されている再生リストのみを表示（AND条件） |
| 選択済みタグの表示 | ハイライト（背景色の変化など）で選択状態を明示 |
| 選択解除 | タグを再クリックすることで選択を解除 |
| 表示件数 | 「実況スタイル系」は全件表示。その他のカテゴリ（プレイスタイル系、編集スタイル系）は初期表示上位5件とし、「もっと見る」で展開する |
| カテゴリ分け | [再生リストタグ初期登録データ一覧.md](/document/master/再生リストタグ初期登録データ一覧.md) の構成に準ずる |

### 4.4 検索条件のリセット

| 項目 | 仕様 |
|------|------|
| ボタン表示条件 | キーワード入力またはタグ選択が1件以上ある場合に活性化。何も設定されていない場合はグレーアウト |
| クリック時の動作 | キーワード・タグの両方をリセットし、全件表示に戻す |
| ボタン文言 | 「検索条件をリセット」 |

### 4.5 検索条件の複合適用

キーワード検索とタグ絞り込みは **同時に適用可能**。両方が指定されている場合は AND 条件でフィルタリングされる。

## 5. ソート仕様

### 5.1 ソート条件

| ソート | 説明 | デフォルト |
|--------|------|-----------|
| スコアが高い順 | 加重平均スコアの高い順に表示 | ✅ デフォルト |
| マイリスト登録者数順 | マイリストへの登録者数が多い順に表示 | — |
| 新着順 | 再生リストに含まれる動画のうち、最新の動画の投稿日が新しい順に表示 | — |

> **設計思想**: デフォルトを「スコアが高い順」とすることで、初めてページを訪れたユーザーが「評判の良い再生リスト」を最初に目にできる導線を設計する。

### 5.2 「新着順」の定義とデータ要件

「新着順」は**再生リスト内の動画の最新投稿日**を基準とする。

```
例：
  再生リストA → 最新動画の投稿日：2026年3月4日
  再生リストB → 最新動画の投稿日：2026年3月1日
  再生リストC → 最新動画の投稿日：2026年2月20日

  → 新着順：A → B → C の順で表示
```

**必要なデータ保存仕様**:

YouTube Data API等での動画情報取得時に、再生リストごとに「最新動画の投稿日（`latestVideoPublishedAt`）」をFirestoreの再生リストドキュメントへ記録・更新する。動画情報の定期取得処理（Cloud Functions）と連動して最新状態を維持する。

| フィールド名 | 型 | 説明 |
|------------|-----|------|
| `latestVideoPublishedAt` | timestamp | 再生リスト内の動画のうち最新の投稿日時 |

### 5.3 ソートUIの仕様

| 項目 | 仕様 |
|------|------|
| UI形式 | ドロップダウン（セレクトボックス） |
| 配置位置 | カード一覧ヘッダーの右側（件数表示の右） |
| 表示文言 | 選択中のソート名を表示（例：「スコアが高い順」） |

### 5.4 スコアなし・登録数0の扱い

| 条件 | 表示位置 |
|------|---------|
| スコアなし（レビュー0件） | スコアありの再生リストの後ろに表示 |
| マイリスト登録者数0 | 登録者数ありの再生リストの後ろに表示 |

## 6. 空状態の表示

検索条件・フィルターを適用した結果、0件になった場合は以下を表示する。

| 要素 | 内容 |
|------|------|
| アイコン | 検索（虫眼鏡）アイコン |
| メッセージ | 「該当する再生リストが見つかりませんでした」 |
| サブメッセージ | 「検索条件を変えてお試しください」 |
| リセットボタン | 「検索条件をリセット」ボタンを表示し、クリックで全件表示に戻す |

## 7. スマートフォン対応

### 7.1 レイアウト変更

スマートフォン（画面幅が一定以下）では、2カラム構成を **1カラム構成** に切り替える。検索条件パネルをカード一覧の上部に配置し、アコーディオン（じゃばら）形式で開閉できる。

```
┌───────────────────────────────┐
│ ヘッダー                       │
├───────────────────────────────┤
│ 再生リストを探す               │
│                               │
│ ┌─────────────────────────┐  │
│ │ 🔍 検索条件（3件適用中）▼│  │  ← アコーディオンヘッダー
│ │                         │  │
│ │ [ゼルダ ×] [RPG ×]      │  │  ← 適用中フィルター（閉じたまま表示）
│ │ [アクション ×]           │  │
│ └─────────────────────────┘  │
│                               │
│ ソート: [スコアが高い順 ▼]     │
│ XX件                          │
│                               │
│ ┌─────────────────────────┐  │
│ │ カード                   │  │
│ └─────────────────────────┘  │
│  ...                          │
└───────────────────────────────┘
```

### 7.2 検索条件パネルのアコーディオン仕様

| 項目 | 仕様 |
|------|------|
| デフォルト状態 | **折りたたみ（閉じた状態）** |
| 開閉操作 | パネルヘッダー（「検索条件」エリア）をタップで開閉トグル |
| 開閉アイコン | 閉じているとき「▼」、開いているとき「▲」 |
| 開いたときの表示内容 | キーワード検索・タグ絞り込み・リセットボタン（PCと同一内容） |

### 7.3 折りたたみ時の適用中フィルター表示

アコーディオンを閉じた状態でも、現在適用されている検索条件が視認できるよう、パネルヘッダー内に適用中のフィルターをバッジ形式で表示する。

#### ヘッダーの状態別表示

| 状態 | 表示 |
|------|------|
| 条件なし | 「検索条件 ▼」のみ表示 |
| 条件あり | 「検索条件（X件適用中）▼」と表示し、適用中フィルタータグを下段に並べて表示 |

#### 適用中フィルタータグの仕様

| 項目 | 仕様 |
|------|------|
| 表示形式 | バッジ形式（例：`ゼルダ ×`、`RPG ×`） |
| 表示対象 | キーワード入力値（入力がある場合）、選択中のタグ（各1件ずつ） |
| ×ボタン | 各バッジの右端に「×」を表示 |
| ×ボタンの動作 | タップするとそのフィルター条件のみを解除し、一覧をリアルタイムで更新する |
| アコーディオンへの影響 | ×ボタンのタップはアコーディオンの開閉を引き起こさない（イベントの伝播を止める） |

**例**（キーワード「ゼルダ」＋タグ「RPG」「アクション」が適用中の場合）:

```
┌──────────────────────────────────────┐
│ 🔍 検索条件（3件適用中）             ▼ │
│                                      │
│  [ゼルダ ×]  [RPG ×]  [アクション ×] │
└──────────────────────────────────────┘
```
