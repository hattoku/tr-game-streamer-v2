## 1. 概要

### 1.1 機能概要

「ゲームタイトルから探す」は、ゲムコメに登録されている再生リスト内でコンテンツとして取り上げられているゲームタイトルを検索・発見するためのページである。キーワード・ゲームジャンル・タグを組み合わせて絞り込み、ソートした一覧からユーザーが興味のあるゲームタイトルを見つけることができる。

「再生リストを探す」がシリーズ単位での発見を目的とするのに対し、本ページは**ゲームタイトル単位での発見**を起点とする。ゲームタイトルを起点に、そのゲームに関連する実況シリーズへ到達するための導線を提供する。

### 1.2 アクセス条件

| 条件 | 内容 |
|------|------|
| ログイン済みユーザー | ✅ アクセス可能 |
| 未ログインユーザー | ✅ アクセス可能（検索・閲覧は可能。マイリスト登録はログイン必須） |

### 1.3 ページURL

| 項目 | 内容 |
|------|------|
| URL | TBD |
| ヘッダーナビゲーション | 常に表示（ログイン状態を問わず） |

## 2. ページレイアウト

### 2.1 全体構成（PC）

ページは **2カラム構成** で配置される。カラム幅はサイト全体のデザインに準ずる。

```
┌─────────────────────────────────────────────────────────────┐
│ ヘッダー                                                      │
├─────────────────────────────────────────────────────────────┤
│ ゲームタイトルから探す                                         │
│                                                             │
│ ┌───────────────────────────────┬─────────────────────────┐ │
│ │ 左カラム                       │ 右カラム                 │ │
│ │ カード一覧                     │ 検索条件パネル           │ │
│ │                               │                         │ │
│ │ ソート: [再生リスト登録数順 ▼]  │ [ キーワード検索 ]        │ │
│ │ XX件                          │                         │ │
│ │                               │ [ ゲームジャンル ]        │ │
│ │ ┌─────────────────────────┐  │                         │ │
│ │ │ カード                   │  │ [ タグ絞り込み ]         │ │
│ │ └─────────────────────────┘  │                         │ │
│ │ ┌─────────────────────────┐  │ [ 検索条件をリセット ]    │ │
│ │ │ カード                   │  │                         │ │
│ │ └─────────────────────────┘  │                         │ │
│ │  ...                          │                         │ │
│ └───────────────────────────────┴─────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

## 3. 左カラム：カード一覧

### 3.1 一覧ヘッダー

カード一覧の上部に以下を表示する。

| 表示要素 | 詳細 |
|---------|------|
| 検索結果件数 | 「XX件」の形式で表示。検索・フィルター適用後もリアルタイムに更新 |
| ソートUI | 検索結果件数の右側にドロップダウンを設置（ソート仕様は [第5章](#5-ソート仕様) に定義） |

### 3.2 カードのデザイン方針

「再生リストを探す」のカードとの主な差異は、**ゲームタイトル画像が縦長（パッケージ画像）である**点にある。楽天ブックスAPIから取得するパッケージ画像のアスペクト比はおよそ **3:4**（縦長）となるため、カードレイアウトは画像を左側に縦長で配置し、情報エリアを右側に確保する横並びレイアウトを採用する。

```
┌────────────────────────────────────────────────────────────┐
│ ┌──────────┐  ゲームタイトル（最大2行）                      │
│ │          │                                               │
│ │ パッケージ│  ジャンル：RPG / アクション                    │
│ │  画像    │                                               │
│ │ （縦長）  │  □ 1 再生リスト  ▶ 11 動画                   │
│ │          │                                               │
│ └──────────┘  [タグ] [タグ] [タグ]                         │
└────────────────────────────────────────────────────────────┘
```

### 3.3 カードに含める表示要素

| 表示要素 | 詳細 |
|---------|------|
| ゲームタイトル画像 | 楽天ブックスAPIから取得したパッケージ画像。縦長（アスペクト比 約3:4）。画像未取得の場合はデフォルトのプレースホルダー画像を表示 |
| ゲームタイトル | 最大2行。超過時は省略記号（…）で切り捨て |
| ゲームジャンル | カンマ区切りまたは改行で複数表示（例：「RPG / アクション」） |
| 再生リスト登録数 | 再生リストを示すアイコン（実装者に委ねる）+「再生リスト」テキスト + 数値（例：□ 1 再生リスト）。このゲームに紐づく再生リストの登録数合計 |
| 動画数 | 動画再生を示すアイコン（実装者に委ねる）+「動画」テキスト + 数値（例：▶ 11 動画）。このゲームに紐づく全再生リストの動画数合計 |
| タグ | バッジ形式で最大3件まで表示。4件以上は「+N」の形式で省略（例：+2）。表示順・強調表示の仕様は [3.5節](#35-カード内タグの表示仕様) に定義 |

### 3.4 カードのクリック動作

カードをクリックすると、該当するゲームタイトルの**ゲームタイトル詳細ページ**へ遷移する。ゲームタイトル詳細ページでは、そのゲームに紐づく再生リストの一覧が表示される。

> **設計思想**: ゲームタイトルをクリックした後に再生リストの一覧が見えることで、ユーザーが「このゲームにはどんな実況シリーズがあるのか」を自然に把握できる導線を設計する。

### 3.5 カード内タグの表示仕様

#### 表示優先順位

カードに表示するタグは以下の優先順位で並べる。

| 優先度 | 対象 | 説明 |
|--------|------|------|
| 1（最優先） | 編集不可タグ | ゲームタイトルの登録者が設定した、ロック（編集不可）状態のタグ |
| 2 | 編集可能タグ | タグが登録された日付が古い順 |

#### 検索時の強調表示

タグ絞り込みで選択されているタグがカード内に表示されている場合、そのタグを**太字**にして目立つよう表示する。これにより、ユーザーが「なぜこのカードが絞り込み結果に表示されているのか」を視覚的に確認できる。

**例**（タグ「王道RPG」で絞り込み中の場合）:

```
[アクション] [**王道RPG**] [ストーリー重視]
             ↑ 太字で強調
```

### 3.6 ページネーション

| 項目 | 仕様 |
|------|------|
| 1ページあたりの表示件数 | 20件 |
| 方式 | ページネーション（ページ番号リンク形式） |
| 表示位置 | カード一覧の下部 |

## 4. 右カラム：検索条件パネル

右カラムはページとともにスクロールする（スクロール追従なし）。

### 4.1 パネル構成

```
┌─────────────────────────┐
│ 検索条件                 │
│                         │
│ [ キーワード検索 ]        │
│                         │
│ ──────────────────────  │
│                         │
│ ゲームジャンルで絞り込む  │
│ [RPG] [アクション] ...   │
│                         │
│ ──────────────────────  │
│                         │
│ タグで絞り込む            │
│ [タグA] [タグB] [タグC]  │
│ ...                     │
│                         │
│ [ 検索条件をリセット ]    │
└─────────────────────────┘
```

### 4.2 キーワード検索

| 項目 | 仕様 |
|------|------|
| 入力形式 | テキスト入力フィールド |
| 検索対象 | **ゲームタイトル名** |
| 検索タイミング | 入力のたびにリアルタイムでフィルタリング（デバウンス処理を適用、遅延目安：300ms） |
| 部分一致 | 対応（入力文字列を含む結果を表示） |
| 大文字小文字 | 区別しない |
| プレースホルダー | 「ゲームタイトル名で検索」 |
| クリアボタン | 入力値がある場合に「×」ボタンを表示。クリックで入力内容をクリアし、全件表示に戻す |

### 4.3 ゲームジャンルで絞り込む

| 項目 | 仕様 |
|------|------|
| 表示形式 | バッジ形式でジャンルを一覧表示 |
| 選択方式 | 複数選択可能 |
| 選択時の動作 | 選択したジャンルがいずれか一つでも一致するゲームタイトルを表示（OR条件） |
| 選択済みジャンルの表示 | ハイライト（背景色の変化など）で選択状態を明示 |
| 選択解除 | ジャンルを再クリックすることで選択を解除 |
| ジャンルデータの取得元 | DBのジャンルマスタから取得して表示する |

> **設計思想**: ジャンルは OR 条件とすることで、複数ジャンルにまたがるゲーム（例：「アクションRPG」）が「アクション」でも「RPG」でも発見できるようにする。

### 4.4 タグで絞り込む

タグの詳細仕様は別途定義する。本仕様書ではUIと動作のみを規定する。

| 項目 | 仕様 |
|------|------|
| 表示形式 | バッジ形式でタグを一覧表示 |
| 選択方式 | 複数選択可能 |
| 選択時の動作 | 選択したタグがすべて付与されているゲームタイトルのみを表示（AND条件） |
| 選択済みタグの表示 | ハイライト（背景色の変化など）で選択状態を明示 |
| 選択解除 | タグを再クリックすることで選択を解除 |
| 表示件数 | TBD（タグ仕様書に準ずる） |

### 4.5 検索条件のリセット

| 項目 | 仕様 |
|------|------|
| ボタン表示条件 | キーワード入力、ジャンル選択、タグ選択のいずれかが1件以上ある場合に活性化。何も設定されていない場合はグレーアウト |
| クリック時の動作 | キーワード・ジャンル・タグのすべてをリセットし、全件表示に戻す |
| ボタン文言 | 「検索条件をリセット」 |

### 4.6 検索条件の複合適用

キーワード検索・ゲームジャンル絞り込み・タグ絞り込みは**同時に適用可能**。複数の検索条件が指定されている場合は AND 条件でフィルタリングされる（ただし、ゲームジャンル内の複数選択は OR 条件。4.3節参照）。

## 5. ソート仕様

### 5.1 ソート条件

| ソート | 説明 | デフォルト |
|--------|------|-----------|
| 再生リスト登録数順 | そのゲームタイトルに紐づく再生リストの登録数が多い順に表示 | ✅ デフォルト |
| タイトル名順 | ゲームタイトル名のあいうえお順（昇順）に表示 | — |
| 動画数順 | そのゲームタイトルに紐づく全再生リストの動画数合計が多い順に表示 | — |

> **設計思想**: デフォルトを「再生リスト登録数順」とすることで、初めてページを訪れたユーザーが「多くの実況シリーズが存在する人気ゲームタイトル」を最初に目にできる導線を設計する。

### 5.2 登録数・動画数が同数の場合の並び順

同じ数値のゲームタイトルが複数存在する場合の並び順は、ゲームタイトル名のあいうえお順（昇順）とする。

### 5.3 ソートUIの仕様

| 項目 | 仕様 |
|------|------|
| UI形式 | ドロップダウン（セレクトボックス） |
| 配置位置 | カード一覧ヘッダーの右側（件数表示の右） |
| 表示文言 | 選択中のソート名を表示（例：「再生リスト登録数順」） |

## 6. 空状態の表示

検索条件・フィルターを適用した結果、0件になった場合は以下を表示する。

| 要素 | 内容 |
|------|------|
| アイコン | 検索（虫眼鏡）アイコン |
| メッセージ | 「該当するゲームタイトルが見つかりませんでした」 |
| サブメッセージ | 「検索条件を変えてお試しください」 |
| リセットボタン | 「検索条件をリセット」ボタンを表示し、クリックで全件表示に戻す |

## 7. スマートフォン対応

### 7.1 レイアウト変更

スマートフォン（画面幅が一定以下）では、2カラム構成を **1カラム構成** に切り替える。検索条件パネルをカード一覧の上部に配置し、アコーディオン（じゃばら）形式で開閉できる。

```
┌───────────────────────────────┐
│ ヘッダー                       │
├───────────────────────────────┤
│ ゲームタイトルから探す          │
│                               │
│ ┌─────────────────────────┐  │
│ │ 🔍 検索条件（2件適用中）▼│  │  ← アコーディオンヘッダー
│ │                         │  │
│ │ [ゼルダ ×] [RPG ×]      │  │  ← 適用中フィルター（閉じたまま表示）
│ └─────────────────────────┘  │
│                               │
│ ソート: [再生リスト登録数順 ▼]  │
│ XX件                          │
│                               │
│ ┌─────────────────────────┐  │
│ │ カード                   │  │
│ └─────────────────────────┘  │
│  ...                          │
└───────────────────────────────┘
```

### 7.2 検索条件パネルのアコーディオン仕様

| 項目 | 仕様 |
|------|------|
| デフォルト状態 | **折りたたみ（閉じた状態）** |
| 開閉操作 | パネルヘッダー（「検索条件」エリア）をタップで開閉トグル |
| 開閉アイコン | 閉じているとき「▼」、開いているとき「▲」 |
| 開いたときの表示内容 | キーワード検索・ゲームジャンル絞り込み・タグ絞り込み・リセットボタン（PCと同一内容） |

### 7.3 折りたたみ時の適用中フィルター表示

アコーディオンを閉じた状態でも、現在適用されている検索条件が視認できるよう、パネルヘッダー内に適用中のフィルターをバッジ形式で表示する。

#### ヘッダーの状態別表示

| 状態 | 表示 |
|------|------|
| 条件なし | 「検索条件 ▼」のみ表示 |
| 条件あり | 「検索条件（X件適用中）▼」と表示し、適用中フィルタータグを下段に並べて表示 |

#### 適用中フィルタータグの仕様

| 項目 | 仕様 |
|------|------|
| 表示形式 | バッジ形式（例：`ゼルダ ×`、`RPG ×`、`王道RPG ×`） |
| 表示対象 | キーワード入力値（入力がある場合）、選択中のゲームジャンル（各1件ずつ）、選択中のタグ（各1件ずつ） |
| ×ボタン | 各バッジの右端に「×」を表示 |
| ×ボタンの動作 | タップするとそのフィルター条件のみを解除し、一覧をリアルタイムで更新する |
| アコーディオンへの影響 | ×ボタンのタップはアコーディオンの開閉を引き起こさない（イベントの伝播を止める） |

**例**（キーワード「ゼルダ」＋ジャンル「RPG」＋タグ「王道RPG」が適用中の場合）:

```
┌──────────────────────────────────────┐
│ 🔍 検索条件（3件適用中）             ▼ │
│                                      │
│  [ゼルダ ×]  [RPG ×]  [王道RPG ×]   │
└──────────────────────────────────────┘
```
