## 1. 概要

TOPページはゲムコメの入口であり、ユーザーの状態に応じて2つの役割を果たす。

**未ログインユーザー向け**: ゲムコメが何であるかを伝えつつ、良質な再生リストとの出会いをすぐに提供する。マイリスト登録・視聴ステータス変更など、アクションが必要になったタイミングでアカウント作成を促す設計とする。

**ログイン済みユーザー向け**: 毎日使うダッシュボードとして機能する。「続きを見る」「新着を確認する」という反復利用の動線を最短化する。

## 3. ログイン状態による表示の違い

TOPページはログイン状態によってセクション構成が異なる。

### 未ログイン時のセクション構成

```
ヘッダー
  ↓
ヒーローセクション（サービス説明 + 検索バー）
  ↓
注目の再生リスト（スコア上位・タブ切り替え）
  ↓
タグピックアップセクション（人気タグ × 複数セクション）
  ↓
フッター
```

### ログイン済み時のセクション構成

```
ヘッダー
  ↓
マイリストセクション（視聴中・完走済みの再生リスト）
  ↓
注目の再生リスト（スコア上位・タブ切り替え）
  ↓
新着の再生リスト
  ↓
タグピックアップセクション（人気タグ × 複数セクション）
  ↓
フッター
```

## 4. ヒーローセクション（未ログイン時のみ）

### 4.1 概要

未ログインユーザーにのみ表示するセクション。コンパクトな帯として配置し、直下のコンテンツへ素早くたどり着けるよう縦の高さを抑える。

### 4.2 表示要素

| 要素 | 内容 |
|------|------|
| キャッチコピー | 「ゲーム実況動画の再生リスト視聴記録&新着通知サービス」 |
| サービス名 | 「ゲムコメ」 |
| 検索バー | 再生リストを探すページへつながるキーワード検索。プレースホルダー：「再生リスト・ゲームタイトル・チャンネル名で検索」 |
| 初めての方へリンク | 検索バーの下部に配置するテキストリンク。文言：「初めての方へ →」。クリックで初めての方へページへ遷移。目立たせすぎない小さめのスタイルとする（初めての方へ仕様書 3.1 に準ずる） |

### 4.3 検索バーの動作

| 項目 | 仕様 |
|------|------|
| 遷移先 | 「再生リストを探す」ページ（`/playlists`）。入力値をクエリパラメータとして渡す |
| 実行タイミング | Enterキー押下 or 検索ボタンクリック |

### 4.4 レイアウト（PC）

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   ゲーム実況動画の再生リスト視聴記録&新着通知サービス          │
│   ゲムコメ                                                 │
│                                                         │
│   [ 再生リスト・ゲームタイトル・チャンネル名で検索    🔍 ]   │
│                                                         │
│   初めての方へ →                                          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 5. マイリストセクション（ログイン済み時のみ）

### 5.1 概要

ログイン済みユーザーにのみ表示するセクション。TOPページの最上部に配置し、「今見るべき再生リスト」への導線を最短化する。

視聴中・完走済みを問わず、マイリストに登録している再生リストを1つのセクションにまとめて表示する。各カードは再生リストの状態に応じて表示内容を切り替える。

### 5.2 セクションヘッダー

| 項目 | 内容 |
|------|------|
| セクションタイトル | 「マイリスト」 |
| 一覧リンク | 「すべて見る →」。クリックでマイリストページ（`/mylist`）へ遷移 |

### 5.3 表示形式

横スクロール（カルーセル）形式。カードを横に並べ、スクロールで続きを確認できる。

| 項目 | 仕様 |
|------|------|
| 表示件数 | 画面幅に応じて自然に収まる件数を表示。件数上限は設けない |
| 並び順 | 最後に再生した動画が新しい順（マイリスト仕様書のデフォルトソートに準ずる） |

### 5.4 カードの表示内容

カードは再生リストの状態によって下部の表示が切り替わる。

**視聴中の場合**

```
┌────────────────────────────────────────────────┐
│ ┌──────────┐  再生リストタイトル                 │
│ │ 再生リスト│  チャンネル名                       │
│ │サムネイル │                                   │
│ └──────────┘  ▶️ 視聴中                         │
│ ──────────────────────────────────────         │
│  ┌──────────┐  動画タイトル（Part 8「〇〇編」）   │
│  │ 動画     │  進捗バー                          │
│  │サムネイル│                                   │
│  └──────────┘                                  │
└────────────────────────────────────────────────┘
```

**完走済み・新着あり の場合**

```
┌────────────────────────────────────────────────┐
│ ┌──────────┐  再生リストタイトル                 │
│ │ 再生リスト│  チャンネル名                       │
│ │サムネイル │                                   │
│ └──────────┘  ✅ 完走                           │
│ ──────────────────────────────────────         │
│  🔔 NEW  新着動画があります                      │
└────────────────────────────────────────────────┘
```

**完走済み・新着なし の場合**

完走済みかつ新着動画がない再生リストはカルーセルに表示しない。

### 5.5 カードのクリック動作

カードをクリックすると動画プレーヤーページへ遷移する。遷移後は「最後に開いた話数」から再生を開始できる（動画プレーヤー仕様書に準ずる）。

### 5.6 空状態の表示

マイリストが0件の場合、またはカルーセルに表示する対象が存在しない場合（全件が完走済み・新着なし）は以下を表示する。

| 要素 | 内容 |
|------|------|
| メッセージ | 「マイリストに再生リストを追加すると、ここに表示されます」 |
| リンク | 「再生リストを探す →」。クリックで `/playlists` へ遷移 |

## 6. 注目の再生リストセクション

### 6.1 概要

ゲムコメ内でスコアの高い再生リストを表示するセクション。未ログイン・ログイン済み問わず全ユーザーに表示する。初めて訪れたユーザーが「評判の良い再生リスト」を最初に目にできる導線として機能する。

### 6.2 セクションヘッダー

| 項目 | 内容 |
|------|------|
| セクションタイトル | 「注目の再生リスト」 |
| 一覧リンク | 「すべて見る →」。クリックで「再生リストを探す」ページ（`/playlists`）へ遷移 |

### 6.3 ジャンルタブ

セクションタイトルの下にタブを配置し、ジャンルで絞り込める。

| 項目 | 仕様 |
|------|------|
| デフォルト選択 | 「総合」 |
| タブの種類 | 総合 + 登録されているジャンル一覧（ジャンルマスタに準ずる） |
| 選択動作 | タブをクリックすると対象ジャンルの再生リストに絞り込んでカルーセルを更新する |

### 6.4 表示形式

横スクロール（カルーセル）形式。

| 項目 | 仕様 |
|------|------|
| 並び順 | 加重平均スコアの高い順（信頼度スコアリングシステム仕様書に準ずる） |
| スコアなし（レビュー0件）の再生リスト | カルーセルに表示しない |

### 6.5 カード仕様

「再生リストを探す」仕様書（第3章）で定義したカードと同一とする。

## 7. 新着の再生リストセクション（ログイン済み時のみ）

### 7.1 概要

ログイン済みユーザーにのみ表示するセクション。ゲムコメに新たに登録された再生リストを表示する。

### 7.2 セクションヘッダー

| 項目 | 内容 |
|------|------|
| セクションタイトル | 「新着の再生リスト」 |
| 一覧リンク | 「すべて見る →」。クリックで「再生リストを探す」ページ（`/playlists`）へ新着順で遷移 |

### 7.3 表示形式

横スクロール（カルーセル）形式。

| 項目 | 仕様 |
|------|------|
| 並び順 | ゲムコメへの登録日が新しい順 |

### 7.4 カード仕様

「再生リストを探す」仕様書（第3章）で定義したカードと同一とする。

## 8. タグピックアップセクション

### 8.1 概要

ゲムコメ内で人気の高いタグをピックアップし、タグごとにカルーセルセクションを複数表示する。未ログイン・ログイン済み問わず全ユーザーに表示する。タグを起点にした再生リスト発見の導線として機能する。

### 8.2 表示するタグの選定

| 項目 | 仕様 |
|------|------|
| 選定方法 | 自動。タグが付与されている再生リスト数の多い順（人気タグ上位）から表示する |
| 表示タグ数 | 6件 |
| 対象タグ | 通常タグのみ。固定タグ（管理者（オーナー・運営者）設定）も含む |

### 8.3 セクションヘッダー

タグごとのセクションタイトルは「# [タグ名] の再生リスト」とする。

| 項目 | 内容 |
|------|------|
| 一覧リンク | 「すべて見る →」。クリックで「再生リストを探す」ページ（`/playlists`）へ該当タグで絞り込んだ状態で遷移 |

### 8.4 表示形式

横スクロール（カルーセル）形式。

| 項目 | 仕様 |
|------|------|
| 並び順 | 加重平均スコアの高い順 |
| スコアなし（レビュー0件）の再生リスト | 表示する（スコアありの後ろに表示） |

### 8.5 カード仕様

「再生リストを探す」仕様書（第3章）で定義したカードと同一とする。

## 9. スマートフォン対応

### 9.1 ヒーローセクション

検索バーを全幅で表示する。キャッチコピー・説明文・検索バーを縦に積む1カラム構成とする。

### 9.2 カルーセル

全セクションのカルーセルはスマートフォンでも横スクロール形式を維持する。1画面に表示されるカード枚数はPCより少なくなる（カード幅はレスポンシブに調整）。
