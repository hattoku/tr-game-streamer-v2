## 概要

### 目的

プレミテの再生リスト詳細ページで、YouTubeの埋め込みプレーヤーを使用して、ゲーム実況シリーズを継続的に視聴できる環境を実現する。プレーヤーと動画リストが緊密に連携し、ユーザーが効率的に再生リストを視聴・進捗を管理できるよう設計。

### スコープ

- YouTube埋め込みプレーヤーの統合と制御
- 再生進捗の追跡と記録
- シアターモード機能
- 連続再生機能
- 動画リストの表示とプレーヤーとの連携
- ユーザーの視聴管理
- 「続きから再生する」機能による視聴再開

### 非スコープ

- ページ全体のUI/UXデザイン（別仕様書）
- レビュー投稿セクションの詳細（別仕様書）
- コメント機能
- ライブチャット統合

## ヒーローセクション構成

### 概要

再生リスト詳細ページの左カラムの上部に位置するセクション。YouTube埋め込みプレーヤー、再生・遷移制御ボタン、コントローラボタンで構成される。

### レイアウト

```
┌─────────────────────────────────┐
│  サムネイル表示エリア             │
│  （クリックでプレーヤーに切り替わり）│
│          ↓                     │
│  YouTubeプレーヤー（16:9）       │
│                                 │
│  [◀ 前へ] [再生/一時停止]        │
│  [次へ ▶] [シアターモード]        │
│  [連続再生]                      │
└─────────────────────────────────┘
```

### 高さ

動的に調整され、コンテンツに応じて拡張される。

## YouTube埋め込みプレーヤー

### プレーヤー仕様

| 項目 | 仕様 |
|------|------|
| サイズ | レスポンシブ対応（画面幅に応じてスケール） |
| アスペクト比 | 16:9 |
| YouTubeコントローラ | 表示（YouTubeの標準UI） |
| 自動再生 | デフォルトオフ（ユーザーが再生ボタンを押して開始） |
| ループ | サポートしない |

### プレーヤーの表示切り替え

**状態遷移**:

```
[初期状態：サムネイル表示]
    ↓ サムネイルをクリック
[プレーヤー表示]
    ↓ 再生ボタンをクリック
[再生中] → コントローラボタン表示
```

### 動画の切り替え

プレーヤーのsrc属性を動的に変更することで実現。YouTube Player API の `startSeconds` パラメータを使用して、前回停止した時刻から再開する場合がある。

```javascript
// 次へボタン時
const nextVideoId = playlistVideos[currentIndex + 1].id;
playerElement.src = `https://www.youtube.com/embed/${nextVideoId}?autoplay=1`;

// 前へボタン時（最初から再生）
const prevVideoId = playlistVideos[currentIndex - 1].id;
playerElement.src = `https://www.youtube.com/embed/${prevVideoId}?autoplay=1`;

// 「続きから再生する」時（前回停止した時刻から再開）
const lastPlayedSeconds = viewingData.episodes[episodeNumber].lastPlayedSeconds;
playerElement.src = `https://www.youtube.com/embed/${videoId}?autoplay=1&start=${lastPlayedSeconds}`;
```

## 再生制御機能

### 再生ボタンの状態

ページ読み込み時に、その再生リストの視聴データ有無によって、ボタンの表示状態が自動的に決まる。

#### 1. 「再生する」ボタン（初回アクセス時）

**表示条件**:

- その再生リストで一度も「再生ボタン」を押したことがない
- または、ログアウト状態で LocalStorage にそのリストの視聴データがない

**機能**:

- クリックすると、サムネイルをYouTubeプレーヤーに置き換え、動画の再生を開始

**動作**:

1. サムネイルが非表示になり、プレーヤーが表示される
2. 動画が自動再生開始される（最初の動画の最初から）
3. コントローラボタン（次へ、前へ、シアターモード、連続再生）が表示される
4. 「最後に開いた話数」をDBに記録（ユーザー側で記録処理）

#### 2. 「続きから再生する」ボタン＋進捗表示（リロード後）

**表示条件**:

- DBまたは LocalStorage に、その再生リストの視聴データが存在する
- ページ読み込み完了時に自動判定

**ボタン下の進捗表示**:

- **進捗バー**: ボタン下に直接付く形で表示。赤色（進捗済み）+ グレー（未視聴）で構成。幅はボタンと同じ幅、高さは2-3px
- **残り時間テキスト**: 進捗バーの右側に表示。「残り○分」の形式。グレー系の色で表示。進捗バーとテキスト間に8-16px程度の隙間

**残り時間の計算**:

- 残り時間（秒） = 動画総長（秒） - 前回停止した時刻（秒）
- 分への変換：切り下げ方式を採用
  - 例：残り300秒 → 「残り5分」、残り359秒 → 「残り5分」、残り360秒 → 「残り6分」

**進捗バーの表示値**:

- 進捗% = (前回停止した時刻（秒） / 動画総長（秒）) × 100

**機能**:

- クリックすると、最後に開いた動画が表示される
- その動画について、前回停止した時刻から自動再生が開始される
- コントローラボタン（次へ、前へ、シアターモード、連続再生）が表示される

**動作**:

1. 最後に開いた動画（例：Part 2）がプレーヤーに表示される
2. 前回停止した時刻（例：5分30秒）から自動再生開始
3. コントローラボタンが表示される
4. その動画が「最後に開いた話数」として記録される（既に記録済みだが、改めて確定）

### 次へボタン

**機能**:

- 再生リストの次の動画に遷移し、自動的に再生を開始

**表示条件**:

- 現在再生中の動画の次に別の動画が存在する場合に表示
- 最後の動画を再生している場合は非表示

**動作ロジック**:

```javascript
if (逆順トグル === OFF) {
  次の動画 = 再生リストの元々の順序での次の動画
} else {
  次の動画 = 逆順にした状態での次の動画
}

プレーヤーIDを切り替え → 最初から自動再生開始（startSecondsなし）
```

**記録処理**:

- 新しい動画を表示した時点で、その動画が「最後に開いた話数」として記録される

### 前へボタン

**機能**:

- 再生リストの前の動画に遷移し、**最初から再生**を開始

**表示条件**:

- 現在再生中の動画の前に別の動画が存在する場合に表示
- Part 1など、前の動画が存在しない場合は非表示

**動作ロジック**:

```javascript
if (逆順トグル === OFF) {
  前の動画 = 再生リストの元々の順序での前の動画
} else {
  前の動画 = 逆順にした状態での前の動画
}

プレーヤーIDを切り替え → 最初から再生開始（seekTo(0)ではなく、新しいiframeのsrc設定）
```

**記録処理**:

- 新しい動画を表示した時点で、その動画が「最後に開いた話数」として記録される

### コントローラボタン表示ルール

| 状態 | ボタン表示 |
|------|----------|
| 初期状態（「再生する」ボタン未クリック時） | 再生ボタンのみ |
| 「再生する」または「続きから再生する」クリック後 | 再生ボタン + コントローラボタン（次へ、前へ、シアターモード、連続再生） |
| 別の動画に遷移後 | コントローラボタンは表示されたままの状態を維持 |

**注記**: 「次へ」「前へ」ボタンは、動画の前後の存在有無に応じて個別に表示・非表示が切り替わる。再生状態とは無関係に、再生ボタン以外のコントローラボタンが表示されていれば、対応する前後の動画がない場合はそのボタンのみ非表示。

## 動画リスト

### 概要

右カラムに表示される、再生リストに含まれるすべての動画のサムネイル・タイトル・進捗情報。プレーヤーと密接に連携し、ユーザーが再生リスト全体を把握しながら視聴できるよう設計。

### 表示位置

右カラムの基本情報セクションの下

### 各動画に表示する情報

| 項目 | 詳細 |
|------|------|
| サムネイル | 動画のサムネイル画像（クリック可能） |
| 動画タイトル | サムネイル下に表示 |
| 進捗バー | ログイン中ユーザーの視聴進捗%（サムネイル下） |
| 動画の尺 | MM:SS形式で表示 |
| 現在再生中マーク | 背景色ハイライト + ▶アイコン |
| 動画件数 | 「全XX話」と表示 |

### 動画のクリック動作

ユーザーが動画リスト内の任意の動画をクリックすると：

1. プレーヤーがその動画に切り替わる
2. 動画が自動再生開始される（最初から）
3. 「最後に開いた話数」として記録される
4. リスト内の現在再生中マークが更新される

### 初期スクロール位置

**初回訪問時**:

- 再生リストの最初の動画（Part 1）からスクロール開始

**リピート訪問時**:

- ユーザーが最後に視聴した動画が最上位に来るようにスクロール（ページ読み込み完了時に自動実行）

### 現在再生中の表示方法

1. **背景色ハイライト**: 現在再生中の動画のサムネイル背景を薄いグレーまたは青で塗りつぶし
2. **▶アイコン**: サムネイル上に再生中を示す三角形（▶）アイコンを表示（白色で視認性を確保）

## 視聴進捗管理

### 記録対象データ

**1. 最後に開いた話数**

- **定義**: ユーザーが最後に「再生する」または「続きから再生する」ボタンを押した動画の話数
- **記録タイミング**: 再生ボタンを押した時点（その時点での視聴進捗%に関わらず）
- **用途**: 信頼度スコアリングシステム仕様書「視聴深度（D）」の計算に使用

**例**:

- ユーザーがPart 5を再生ボタンで開始した後、わずか5%しか視聴しなくても
- 「Part 5 = 話数5」として記録される
- 再生リストが全25話の場合、視聴深度 = 5/25 = 20%

**2. 再生時間（秒数）**

- **定義**: 各動画について、ユーザーが再生した時間を秒単位で記録
- **記録粒度**: 秒単位の再生時間
  - 例：動画再生中に300秒時点で停止した場合、「300秒」と記録
- **記録タイミング**: 定期的に（5秒ごと）に更新され、ページ離脱時に最終値が確定される
- **用途**: 「続きから再生する」時の再開位置特定、視聴進捗%の計算に使用

**3. 各動画の視聴進捗%**

- **定義**: 各動画について、ユーザーが視聴した時間の割合
- **計算方式**: 視聴進捗% = (再生時間（秒） / 動画総長（秒）) × 100
- **リアルタイム性**: 再生時間と同期して更新（5秒ごと）
- **用途**: 動画リストの進捗バー表示、視聴深度の補足情報

### 複数動画視聴時の記録ルール

同じセッション内で複数の動画を視聴した場合：

| データ | 記録方式 |
|------|--------|
| 各動画の再生時間（秒） | **個別に記録**（Part 3の180秒、Part 4の450秒、Part 5の120秒 など） |
| 各動画の視聴進捗% | **個別に記録**（Part 3の50%、Part 4の75%、Part 5の30% など） |
| 最後に開いた話数 | **一つのみ記録**（最後に再生ボタンを押した動画のみ） |

**例**:

```
ユーザーの操作順序：
1. Part 3を再生 → 50%視聴（180秒／360秒）
2. Part 4を再生 → 75%視聴（450秒／600秒）
3. Part 5を再生 → 30%視聴（120秒／400秒）→ 途中で中断

記録結果：
- 最後に開いた話数：5
- 再生時間：Part 3 = 180秒, Part 4 = 450秒, Part 5 = 120秒
- 視聴進捗：Part 3 = 50%, Part 4 = 75%, Part 5 = 30%
```

### 再生時間の記録タイミング

**記録方式**:

- 動画再生中：**5秒ごとに現在の再生時間を更新**
- ページ遷移時：その時点での再生時間を最終値として確定
- ページ離脱（別ページへの遷移、ブラウザを閉じるなど）：最終値を Firestore / LocalStorage に保存

**実装フロー**:

```
Part 1再生開始（0秒から）
  ↓ 5秒経過 → Part 1の再生時間: 5秒 を更新
  ↓ 10秒経過 → Part 1の再生時間: 10秒 を更新
  ↓ ...中略...
  ↓ 50秒経過 → Part 1の再生時間: 50秒 を更新
  ↓ 「次へ」クリック → Part 1の再生時間: 50秒（最終値）を確定
  ↓ Part 2再生開始（0秒から）
  ↓ 5秒経過 → Part 2の再生時間: 5秒 を更新
  ↓ ...中略...
  ↓ 75秒経過 → Part 2の再生時間: 75秒 を更新
  ↓ ページ離脱 → Part 2の再生時間: 75秒（最終値）を保存
```

### 右カラム進捗バー表示

**表示内容**:

- ログイン中のユーザー自身の視聴進捗%のみを表示
- 複数ユーザーの平均値ではない
- 割合テキストは表示しない

**表示位置**:

- 各動画のサムネイル下

**表示形式（YouTube 仕様準拠）**:

| 項目 | 仕様 |
|------|------|
| バー形状 | 水平の細い横線（高さ約 2-3px） |
| 進捗済み部分 | 赤色（RGB: 255, 0, 0 または #FF0000） |
| 未視聴部分 | グレー（薄いグレー、例：RGB: 200, 200, 200 または #C8C8C8） |
| サムネイルとの重なり | バーはサムネイル下端の直下に配置、または軽く重ねて表示 |
| 割合テキスト | 非表示 |

**視覚例**:

```
┌─────────────────────┐
│  サムネイル画像      │
└─────────────────────┘
 ████████░░░░░░░
```

**実装時の注意**:

- サムネイル画像の色を損なわないよう、バーは画像下部に配置
- スクリーンサイズによってバーの見え方が変わらないよう、相対サイズで実装

## シアターモード

### 概要

プレーヤーを全幅で大きく表示し、没入感のある視聴体験を提供する。

### レイアウト変更

詳細は「/document/specification/page/ページ 再生リスト詳細 仕様書.md」の「シアターモード時のレイアウト」を参照。

プレーヤーが全幅で表示され、下部に情報セクションが縮小表示される。

### ヘッダー表示

**シアターモード時のヘッダー仕様**:

| 項目 | 仕様 |
|------|------|
| 表示内容 | ロゴ、ナビゲーションメニュー、ユーザーメニュー（すべて表示） |
| コンパクト度 | 通常時の**50%サイズ**に縮小 |
| 背景色 | 黒で統一 |

**例**:

- 通常時：高さ 60px → シアターモード時：高さ 30px

### 背景色

**黒背景の適用範囲**:

1. **プレーヤーの左右余白**
   - 画面幅が広い場合、プレーヤーの左右に余白が生じる場合がある
   - この余白を黒（#000000）で埋める

2. **コントローラボタン部分**
   - 「次へ」「前へ」「シアターモード」「連続再生」のボタンセクション
   - 背景を黒で統一
   - テキスト・アイコンは白または明るい色で視認性を確保

### トグル状態の保存

**シアターモード状態は保持されない**:

- ユーザーが別のページに移動して再び戻った場合、シアターモードは OFF にリセットされる
- 同じセッション内での移動でもリセット対象

**理由**: シアターモードは一時的な表示モードとして設計

## 連続再生機能

### 概要

有効にすると、現在の動画が最後まで再生された時点で、自動的に次の動画に遷移し、再生を開始する。

### 初期状態

**デフォルト**: 有効（ON）

### 自動遷移ルール

**トリガー**:

- 現在再生中の動画が最後まで再生された時点（動画の最終フレームに到達）

**遷移先**:

```javascript
if (逆順トグル === OFF) {
  遷移先 = 再生リストの次の動画（元々の順序）
} else {
  遷移先 = 再生リストの次の動画（逆順の状態）
}
```

**動作**:

1. 次の動画が自動的に表示される
2. 再生が自動開始される
3. 遷移先の動画が「最後に開いた話数」として記録される

### 有効・無効時の「次へ」ボタン挙動

**連続再生 ON / OFF に関わらず、「次へ」ボタンの挙動は同じ**:

| 状態 | 動作 |
|------|------|
| 連続再生 ON | 次の動画に遷移して自動再生 |
| 連続再生 OFF | 次の動画に遷移して自動再生 |

**理由**: ユーザーが明示的に「次へ」を押した場合は、連続再生設定に関わらず遷移すべき

### 設定の保存範囲

**保存スコープ**: ユーザー全体の共通設定

- 一つの再生リストで連続再生を OFF にすると
- **全ての再生リストで連続再生が OFF になる**

**実装**:

- ログイン済みユーザー：Firestoreのユーザードキュメントに保存
- 未ログインユーザー：LocalStorageに保存

## データ保存仕様

### 視聴データの記録フロー

#### ログイン済みユーザー

```
ユーザーが「再生する」または「続きから再生する」ボタンを押す
    ↓
フロントエンド：最後に開いた話数を記録
    ↓
動画再生中：5秒ごとに再生時間と視聴進捗%を更新
    ↓
ページ離脱時：最終値を API呼び出し（Firestore保存）
    ↓
Firestore：視聴履歴データとして保存
```

#### 未ログインユーザー

```
ユーザーが「再生する」または「続きから再生する」ボタンを押す
    ↓
フロントエンド：最後に開いた話数を記録
    ↓
動画再生中：5秒ごとに再生時間と視聴進捗%を更新
    ↓
ページ離脱時：最終値を LocalStorage に保存
    ↓
ブラウザ内に永続化
```

### 記録対象データ

ログイン済みユーザー・未ログインユーザー共通：

| データ | 記録内容 |
|------|--------|
| 最後に開いた話数 | ユーザーが最後に再生ボタンを押した動画の話数 |
| 再生時間（秒数） | 各動画について、ユーザーが再生した時間を秒単位で記録 |
| 各動画の視聴進捗% | 各動画について、ユーザーが視聴した時間の割合 |
| 逆順トグル状態 | 各再生リストごとのON/OFF状態 |
| 連続再生設定 | ユーザー全体の共通設定（ON/OFF） |

## ユーザー状態管理

### 逆順トグル

#### 仕様

再生リストが逆順で登録されている場合、トグルをONにすることで、動画の順序を反転させて表示・再生できる。

#### 保存範囲

**ユーザーごと、かつ再生リスト単位で記憶**

- ユーザーAが「再生リスト1」で逆順トグルをONにしても、「再生リスト2」では影響しない
- ユーザーBの逆順トグル設定とは独立している

#### 保存先

- **ログイン済みユーザー**: Firestore に保存
- **未ログインユーザー**: LocalStorage に保存

#### 動作への反映

トグル状態が変更されると：

1. 「次へ」「前へ」ボタンの遷移先が即座に変わる
2. 連続再生の自動遷移が新しい順序に基づいて実行される
3. 右カラム動画リストの表示順序が反転する
4. 現在再生中の動画を最上位にスクロール

### 連続再生設定

#### 仕様

有効にすると、動画視聴完了時に自動的に次の動画に遷移し、再生を開始する。

#### 保存範囲

**ユーザー全体の共通設定**

一つの再生リストで設定を変更すると、全ての再生リストに反映される。

#### 初期状態

**デフォルト**: 有効（ON）

#### 保存先

- **ログイン済みユーザー**: Firestore に保存
- **未ログインユーザー**: LocalStorage に保存

### シアターモード状態

#### 仕様

プレーヤーを全幅で大きく表示するモード。

#### 状態保持

**保持されない**

別のページに移動して再び戻った場合、シアターモードは OFF にリセットされる。これは、シアターモードが一時的な表示モードとして設計されているため。

#### ローカル状態管理

Reactの`useState`で管理。ストレージには保存されない。
