## 1. 概要

本仕様書は、プレミテのフロント（一般ユーザー向け画面）において複数のページをまたいで共通的に使用するUIコンポーネントの仕様を定義する。

対象コンポーネントは以下のとおり。

| コンポーネント | 概要 |
|-------------|------|
| グローバルヘッダー | 全ページ共通のヘッダー。ロゴ・ナビゲーション・ユーザーメニューを含む |
| グローバルフッター | 全ページ共通のフッター。サービス案内リンク・法的リンクを含む |
| インフォメーションバー | ヘッダー直下に表示するお知らせバナー帯 |
| ログイン要求モーダル | 未ログインユーザーがログイン必須操作を行った際に表示するモーダル |
| トースト通知 | 操作結果をユーザーに通知する一時的なポップアップ |
| テストモードウィジェット | 開発者がログイン状態を即座に切り替えるための画面右下固定ウィジェット |
| 空状態 (Empty States) | 検索結果なしやデータ未登録時に表示するプレースホルダー表示 |
| ローディング | データ取得中のSkeleton Screenおよびアクション実行中のSpinner |

### 適用範囲

本仕様は `/` 配下のフロント全ページに適用する。ただし以下のページはヘッダー・フッターの表示形式が異なる（各仕様書を参照）。

| ページ | 備考 |
|--------|------|
| `/login` / `/signup` / `/password-reset` | 認証フロー用の簡易レイアウト（ロゴのみ）を使用 |
| 初期設定フロー | ロゴ＋ステップインジケーターのみのレイアウトを使用 |
| エラーページ | 404/403/500/503 ごとに表示形式が異なる（詳細は「共通 エラーページ 仕様書」を参照） |

## 2. グローバルヘッダー

### 2.1 概要

フロント全ページの最上部に固定表示されるコンポーネント。ロゴ・グローバルナビゲーション・ユーザーメニューで構成される。

### 2.2 全体レイアウト

#### PC版

```
┌────────────────────────────────────────────────────────────────┐
│  [プレミテ ロゴ]   再生リストを探す  ゲームから探す  チャンネルから探す  まとめ    [ログインボタン]  │
└────────────────────────────────────────────────────────────────┘
```

**ログイン済み時（右端部分）**

```
[マイリスト]  [視聴履歴]  [ ⚬ ユーザーアイコン ▼ ]
```

#### モバイル版

```
┌────────────────────────────────────────────────────────────────┐
│  [プレミテ ロゴ]                            [ハンバーガーアイコン ☰]  │
└────────────────────────────────────────────────────────────────┘
```

### 2.3 ロゴ

| 項目 | 仕様 |
|------|------|
| 表示内容 | サービスロゴ（テキストまたは画像） |
| クリック時の動作 | TOPページ（`/`）へ遷移 |
| 表示位置 | ヘッダー左端 |

### 2.4 グローバルナビゲーション（PC）

PC版ではヘッダー中央〜左寄りに水平並びで表示する。

| ナビ項目 | 遷移先 | 表示条件 |
|---------|--------|---------|
| 再生リストを探す | `/playlists` | 常時表示 |
| ゲームから探す | `/games` | 常時表示 |
| チャンネルから探す | `/channels` | 常時表示 |
| まとめ | `/collections` | 常時表示 |

- 現在表示中のページに対応するナビ項目をアクティブ状態（下線または色変化）で表示する。
- アクティブ判定はURLのプレフィクスで行う（例：`/games/[id]` 表示中は「ゲームから探す」がアクティブ）。

### 2.5 ユーザーエリア（PC・未ログイン時）

| 要素 | 仕様 |
|------|------|
| ログインボタン | 「ログイン」テキストボタン。クリックで `/login` へ遷移 |
| アカウント作成ボタン | 「アカウント作成」をブランドカラーのボタンで表示。クリックで `/signup` へ遷移 |

```
【PC・未ログイン時のヘッダー右端】

[ログイン]  [アカウント作成]
```

### 2.6 ユーザーエリア（PC・ログイン済み時）

| 要素 | 仕様 |
|------|------|
| マイリストリンク | 「マイリスト」テキストリンク。クリックで `/mylist` へ遷移 |
| 視聴履歴リンク | 「視聴履歴」テキストリンク。クリックで `/history` へ遷移 |
| ユーザーアイコン | プロフィール画像を円形で表示（未設定時はデフォルトアイコン）。クリックでユーザードロップダウンを展開 |

```
【PC・ログイン済み時のヘッダー右端】

マイリスト   視聴履歴   [ ⚬ アイコン ▼ ]
```

#### ユーザードロップダウン

ユーザーアイコンをクリックすると以下のドロップダウンを表示する。

```
┌──────────────────────┐
│  ユーザー名            │
│  example@email.com   │
├──────────────────────┤
│  プロフィール          │
│  設定                 │
├──────────────────────┤
│  ログアウト            │
└──────────────────────┘
```

| メニュー項目 | 動作 |
|------------|------|
| プロフィール | `/profile/[userId]`（自分のプロフィールページ）へ遷移 |
| 設定 | `/settings` へ遷移 |
| ログアウト | ログアウト処理を実行後、TOPページへ遷移 |

- ドロップダウン外をクリックするとドロップダウンを閉じる。
- 「ユーザー名」「メールアドレス」はクリック不可の表示専用。

### 2.7 モバイル版ハンバーガーメニュー

#### ハンバーガーアイコン

| 項目 | 仕様 |
|------|------|
| 表示位置 | ヘッダー右端 |
| アイコン | ☰（3本線）。メニュー展開中は × に変化 |
| クリック時 | ドロワーメニューを開く |

#### ドロワーの種別

右からスライドインするドロワー型を採用する。

#### ドロワーの構成（未ログイン時）

```
┌──────────────────────────────────┐
│               [×]                │
│                                  │
│  再生リストを探す                  │
│  ゲームから探す                    │
│  チャンネルから探す                 │
│  まとめ                           │
│                                  │
├──────────────────────────────────┤
│                                  │
│  [ログイン]                       │
│  [アカウントを作成する]             │
│                                  │
└──────────────────────────────────┘
```

#### ドロワーの構成（ログイン済み時）

```
┌──────────────────────────────────┐
│               [×]                │
│                                  │
│  ⚬ ユーザー名                     │
│     example@email.com            │
│                                  │
├──────────────────────────────────┤
│                                  │
│  再生リストを探す                  │
│  ゲームから探す                    │
│  チャンネルから探す                 │
│  まとめ                           │
│                                  │
├──────────────────────────────────┤
│                                  │
│  マイリスト                       │
│  視聴履歴                         │
│                                  │
├──────────────────────────────────┤
│                                  │
│  プロフィール                      │
│  設定                             │
│                                  │
├──────────────────────────────────┤
│                                  │
│  ログアウト                        │
│                                  │
└──────────────────────────────────┘
```

#### ドロワーの動作仕様

| 項目 | 仕様 |
|------|------|
| 開閉 | ハンバーガーアイコンのクリックで開閉トグル |
| 閉じる操作 | ドロワー内「×」ボタン、またはドロワー外（オーバーレイ）のクリックで閉じる |
| オーバーレイ | ドロワー展開時、画面左側に半透明の暗色オーバーレイを表示する |
| 各リンクのクリック | 遷移先へ移動するとともにドロワーを閉じる |
| スクロール | ドロワー展開中はボディのスクロールを無効化する |
| アニメーション | 右端からスライドインするアニメーション（duration: 200ms程度） |

### 2.8 ヘッダー共通仕様

| 項目 | 仕様 |
|------|------|
| 高さ（PC） | 実装時に決定。全ページで統一する |
| 最大幅（PC） | 1400px（共通 ページレイアウト 仕様書 §3 に準ずる） |
| 高さ（モバイル） | 実装時に決定。全ページで統一する |
| 背景色 | シネマダーク背景（`color-bg-base`: `#0f0f0f`） | 全ページで統一する |
| 固定表示 | スクロール時もヘッダーを最上部に固定表示する（`position: sticky` または `fixed`） |
| z-index | ページコンテンツより前面に表示する |

### 2.9 ブレークポイント

| ブレークポイント | 表示形式 |
|--------------|---------|
| 768px以上（PC） | グローバルナビゲーションを水平表示、ユーザーエリアを右端に表示 |
| 767px以下（モバイル） | ロゴ＋ハンバーガーアイコンのみ。ナビゲーションはドロワー内に収納 |

## 4. インフォメーションバー（お知らせバナー）

### 4.1 概要

管理者（オーナー・運営者）が設定した「一般ユーザー向けお知らせ」（メンテナンス告知・キャンペーン情報等）を、ヘッダー直下に帯状のバナーとして表示するコンポーネント。詳細仕様は「/document/specification/management/管理 お知らせ管理仕様書.md」第5章に従う。

### 4.2 表示位置

ヘッダーとページコンテンツの間に挿入する。

```
┌──────────────────────────────────────────────────┐
│  ヘッダー                                          │
├──────────────────────────────────────────────────┤
│  ■ メンテナンスのお知らせ  詳細を見る  [1/2] ＜ ＞ [×] │  ← インフォメーションバー
├──────────────────────────────────────────────────┤
│  ページコンテンツ                                   │
```

### 4.3 表示条件

- ステータスが「掲載中」のお知らせが1件以上あり、かつ対象ページが現在のページを含む場合に表示する。
- ログイン・未ログイン問わず全ユーザーに表示する。

### 4.4 表示要素

| 要素 | 内容 |
|------|------|
| タイトルテキスト | お知らせのタイトル（最大1行、超過時は省略記号で切り詰め） |
| 「詳細を見る」リンク | 本文が設定されている場合のみ表示。クリックで詳細モーダルを展開 |
| 件数インジケーター | 複数件の場合に「1/3」のように表示 |
| 切り替えボタン（＜＞） | 複数件の場合に前後に切り替える |
| 閉じるボタン（×） | クリックでバナー帯を非表示にする |

### 4.5 閉じた後の挙動

- LocalStorageに閉じたお知らせのIDを保存し、同一端末・同一ブラウザでは再表示しない。
- 複数件ある場合、「×」で閉じるのは表示中の1件のみ。他の件は引き続き表示される。

### 4.6 詳細モーダル

「詳細を見る」クリック時に表示するモーダル。

| 要素 | 内容 |
|------|------|
| タイトル | お知らせのタイトル |
| 本文 | お知らせの全文 |
| 掲載期間 | 掲載開始日〜掲載終了日 |
| 閉じるボタン | モーダル右上の「×」ボタン |

- モーダル外クリックまたは「×」ボタンで閉じる。

## 5. ログイン要求モーダル

### 5.1 概要

未ログインユーザーがログインを必要とする操作を行おうとした際に表示するモーダル。アカウント作成またはログインへ誘導する。詳細仕様は「/document/specification/page/ページ ログイン アカウント登録・ログイン仕様書.md」第3章に従う。

### 5.2 表示トリガー

| 操作 | 詳細 |
|------|------|
| 星評価をクリックしようとした | レビュー投稿機能仕様書参照 |
| 視聴ステータスを選択しようとした | レビュー投稿機能仕様書参照 |
| 「レビューを投稿する」ボタンを押した | レビュー投稿機能仕様書参照 |
| マイリストに追加しようとした | マイリスト機能仕様書参照 |
| 再生リストの提案ページにアクセスした | ログインページへリダイレクト |
| 「参考になった」ボタンを押した | レビュー投稿機能仕様書参照 |
| 「通報する」を選択した | レビュー投稿機能仕様書参照 |

### 5.3 モーダル構成

```
┌─────────────────────────────────────────────┐
│  ✕                                          │
│                                             │
│  [操作に応じたタイトル]                       │
│  例：「マイリストに追加するにはアカウントが必要です」│
│                                             │
│  プレミテのアカウントを作成すると、            │
│  視聴記録の管理やレビューの投稿ができます。     │
│                                             │
│  [    アカウントを作成する    ]               │
│                                             │
│  すでにアカウントをお持ちの方は ログイン       │
│                                             │
└─────────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| タイトル | 操作に応じて変化する（例：「マイリストに追加するにはアカウントが必要です」） |
| 本文 | 「プレミテのアカウントを作成すると、視聴記録の管理やレビューの投稿ができます。」 |
| CTA（主） | 「アカウントを作成する」ボタン → `/signup` へ遷移 |
| CTA（副） | 「ログイン」テキストリンク → `/login` へ遷移 |
| 閉じる | 右上「×」ボタン、またはモーダル外クリックで閉じる |

## 6. トースト通知

### 6.1 概要

ユーザー操作の結果（成功・失敗・警告・情報）を短時間表示するポップアップ通知コンポーネント。

### 6.2 表示位置・動作

| 項目 | 仕様 |
|------|------|
| 表示位置 | 画面上部中央（PCは右上も可。デザイン実装時に決定） |
| 自動消去 | 表示から3秒後に自動で消える |
| 手動消去 | トースト上の「×」ボタンで即時消去 |
| 複数表示 | 複数のトーストが発生した場合は縦方向に積み上げて表示する |
| アニメーション | フェードイン・フェードアウト（duration: 200ms程度） |

### 6.3 種別とスタイル

| 種別 | 用途 | 色イメージ |
|------|------|----------|
| 成功（success） | 操作が正常に完了した場合 | 緑系（詳細は共通 デザイントークン仕様書 §2.4 参照） |
| エラー（error） | 操作が失敗した場合 | 赤系 |
| 警告（warning） | 注意が必要な場合 | 黄系 |
| 情報（info） | 中立的な通知 | 青系・グレー系 |

### 6.4 使用箇所の例

| 操作 | 種別 | メッセージ例 |
|------|------|------------|
| マイリスト追加成功 | 成功 | 「マイリストに追加しました」 |
| マイリスト削除成功 | 成功 | 「マイリストから削除しました」 |
| 再生リスト提案送信成功 | 成功 | 「提案を送信しました」 |
| ネットワークエラー | エラー | 「通信エラーが発生しました。時間をおいて再試行してください」 |
| URL形式不正 | エラー | 「URLの形式が正しくありません」 |

## 7. テストモードウィジェット

### 7.1 概要

開発・動作確認時に、フロント画面上でログイン状態とログアウト状態を1ボタンで即座に切り替えるためのウィジェット。画面右下にスクロール追従で固定表示される。

**表示条件**: テストモードが有効な場合のみ表示する。本番環境では表示しない。

### 7.2 外観・レイアウト

画面右下に常駐する小さなフローティングウィンドウ。

```
┌─────────────────────────────┐
│ 🧪 テストモード               │
│                             │
│  [  ログアウト中に切替  ]     │  ← ログイン中の場合
│                             │
│  現在: ログイン中             │
└─────────────────────────────┘
```

```
┌─────────────────────────────┐
│ 🧪 テストモード               │
│                             │
│  [  ログイン中に切替  ]       │  ← ログアウト中の場合
│                             │
│  現在: ログアウト中           │
└─────────────────────────────┘
```

### 7.3 表示要素

| 要素 | 内容 |
|------|------|
| ラベル | 「🧪 テストモード」。テストモード有効中であることを常に明示する |
| 切り替えボタン | 現在の状態に応じて「ログアウト中に切替」または「ログイン中に切替」を表示。ブランドカラー（`color-brand-logo`: `#e03030`）で目立つボタンとする |
| 現在の状態表示 | 「現在: ログイン中」または「現在: ログアウト中」をテキストで表示 |

### 7.4 動作仕様

| 項目 | 仕様 |
|------|------|
| 表示位置 | 画面右下に固定（`position: fixed; bottom: 右下;`）。スクロールに追従する |
| ウィジェットのサイズ | コンパクト（実装時に決定）。ページコンテンツを過度に隠さないよう配慮する |
| 枠線 | ブランドカラー（`color-brand-logo`: `#e03030`）の枠線でウィジェットであることを視覚的に識別できるようにする |
| z-index | ページコンテンツの最前面に表示する |

#### ログイン状態への切り替え

「ログイン中に切替」ボタンをクリックすると、テストモード用会員の権限でログインした状態に切り替わる。

| 項目 | 仕様 |
|------|------|
| ログイン対象 | テストモード用会員（詳細は/document/specification/management/管理 ユーザー管理仕様書.md参照） |
| 切り替え後の動作 | ヘッダーのユーザーエリアがログイン済み表示に切り替わる。ページのリロードは行わない（状態のみ更新） |
| 現在の状態表示 | 「現在: ログイン中」に更新 |
| ボタンラベル | 「ログアウト中に切替」に更新 |

#### ログアウト状態への切り替え

「ログアウト中に切替」ボタンをクリックすると、ログアウト状態に切り替わる。

| 項目 | 仕様 |
|------|------|
| 切り替え後の動作 | ヘッダーのユーザーエリアが未ログイン表示に切り替わる。ページのリロードは行わない（状態のみ更新） |
| 現在の状態表示 | 「現在: ログアウト中」に更新 |
| ボタンラベル | 「ログイン中に切替」に更新 |

### 7.5 テストモード用会員

テストモードでログインする際に使用するテスト専用のユーザーアカウント。アカウントの定義・権限・設定方法については「/document/specification/management/管理 ユーザー管理仕様書.md」に記述する（未作成）。

## 8. 空状態 (Empty States)

### 8.1 概要

検索結果が0件の場合、マイリストが空の場合、あるいはデータが1件も登録されていないコンテンツエリアにおいて、ユーザーに状況を伝え、次のアクションを促すための共通コンポーネント。

### 8.2 基本構造

```
┌───────────────────────────────────────────┐
│                                           │
│               [ イメージ ]                 │
│                                           │
│            [ メインメッセージ ]             │
│                                           │
│            [ サブメッセージ ]               │
│                                           │
│           [ アクションボタン ]              │
│                                           │
└───────────────────────────────────────────┘
```

| 要素 | 仕様 |
|------|------|
| イメージ | 状態を象徴するイラスト、SVGアイコン、またはプレースホルダー画像。 |
| メインメッセージ | 現在の状況を簡潔に伝えるテキスト（例：「結果が見つかりませんでした」）。 |
| サブメッセージ | 理由や解決策を提示する補足テキスト（任意）。 |
| アクションボタン | 次のステップへ誘導するボタン（例：「検索条件をクリア」「動画を探す」）（任意）。 |

### 8.3 共通パターン

| パターン | メインメッセージ | サブメッセージ | アイコン等の推奨 |
|---------|----------------|---------------|----------------|
| 検索結果なし | お探しの再生リストは<br>見つかりませんでした | キーワードやフィルターを<br>変えてみてください | 虫眼鏡 (`search`) |
| マイリスト空 | マイリストは空です | 気になる再生リストを<br>保存して自分だけのリストを作ろう | ハート (`favorite`) |
| 視聴履歴なし | 視聴履歴はありません | 気になる動画を視聴して<br>記録を残しましょう | 時計 (`history`) |
| 該当データなし | 表示できるデータが<br>ありません | - | ボックス (`inventory_2`) |

### 8.4 レイアウト仕様

| 項目 | 仕様 |
|------|------|
| 配置 | コンテンツエリア内（`main` コンテナ内）で中央揃え（垂直・水平）。 |
| 余白 | 上下 60px 以上のパディングを確保し、ページ全体が寂しく見えないように調整する。 |
| テキスト位置 | 中央揃え（`text-align: center`）。 |
| アクションボタン | ブランドカラーのボタンを使用し、UIのプライマリな導線として機能させる。 |

## 9. ローディング (Loading)

### 9.1 概要
ユーザーにシステムの動作状況を伝え、ストレスを軽減するために「Skeleton Screen」と「Spinner」を使い分ける。

### 9.2 Skeleton Screen（スケルトン領域）
データ読み込み中に、実コンテンツのレイアウトを模したグレースケールな矩形を表示する手法。

#### 用途
- ページ全体の初回読み込み時
- リスト・カード等、構造が明確な大型コンポーネントの読み込み時
- ページ遷移時（Next.js `loading.tsx`）

#### 設計方針
- 実コンテンツと同じサイズ、角丸、余白を維持する。
- 静止させず、左から右へ流れるようなハイライト（シマーアニメーション）を付与する。
- テキスト行のスケルトンは、実コンテンツよりもやや短く（例：タイトルなら幅 60〜80%）設定し、複数行の場合は末尾を揃えないことで「テキストらしさ」を演出する。

### 9.3 Spinner（スピナー）
円形の回転アニメーション。

#### 用途
- ボタン内のアクション実行中（「保存中」など）
- 無限スクロールの次ページ読み込み時
- 設定変更後の微細なデータ更新時

#### 設計方針
- `color-spinner`（`#aaaaaa`）を使用する。
- 派手な色は使わず、UIに馴染む適度な線幅とする。

### 9.4 Next.js での使用イメージ

App Router の機能を活用し、一貫したローディング体験を提供する。

#### 1. 地点別 loading.tsx の役割
- **グローバル (`/app/loading.tsx`)**: 共通レイアウトに基づき、全ページ共通のスケルトンを表示。ヘッダー（ロゴ・ナビ）は表示されたまま、メインコンテンツ領域に汎用的なカードプレースホルダー等を表示する。
- **ルート単位 (`/app/[route]/loading.tsx`)**: ページ特有の構造に最適化したスケルトンを表示。
    - 例：`/playlists` なら再生リストカードのグリッドスケルトン。
    - 例：`/profile` ならプロフィール情報のヘッダーと各セクションのスケルトン。

#### 2. Streaming と Suspense
- 遅延の大きい特定のコンポーネント（サイドバーの「おすすめ」や「コメント一覧」等）のみを `Suspense` で囲い、ページ本体が表示された後に部分的にスケルトンを表示する。
