## 1. 概要

本仕様書は、プレミテのフロント（`/` 配下）において発生するシステムエラー・アクセスエラー時に表示するエラーページのレイアウトと挙動を定義する。

### 1.1 対象エラーの一覧

| エラーコード | 名称 | 発生条件の例 |
|------------|------|------------|
| 404 | Not Found | 存在しないURLへのアクセス、削除済みコンテンツへのアクセス |
| 403 | Forbidden | ログインが必要なページへの未ログインアクセス以外の権限エラー（非公開コンテンツ等） |
| 500 | Internal Server Error | サーバー内部でのエラー、予期しない例外 |
| 503 | Service Unavailable | メンテナンス中、サービス一時停止 |

> **ログイン必須ページへの未ログインアクセス（例：`/mylist`）は、エラーページではなくログインページ（`/login`）へリダイレクトする。** 403エラーページはその他の権限エラーに使用する。

### 1.2 実装方式

Next.js のファイルベースルーティングにおける以下のファイルで実装する。

| 対象 | ファイル | 備考 |
|------|---------|------|
| 404 | `app/not-found.tsx` | 存在しないルートへのアクセス時に自動適用 |
| 500 | `app/error.tsx` | サーバー・クライアントエラー時に自動適用 |
| 403 / 503 | 各ページコンポーネント内で条件分岐して表示、またはカスタムページとして実装 | |

## 2. 共通仕様

### 2.1 レイアウト構成

すべてのエラーページは共通の基本レイアウトを採用する。

```
┌─────────────────────────────────────────────────────────────────┐
│ グローバルヘッダー（共通UIコンポーネント仕様書に準ずる）             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                    エラーコンテンツエリア                          │
│                    （中央寄せ・縦方向中央付近）                    │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│ グローバルフッター（共通UIコンポーネント仕様書に準ずる）             │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 エラーコンテンツエリアの共通レイアウト

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│                       [イラスト・アイコン]                        │
│                                                                 │
│                     エラーコード（大きめに表示）                   │
│                     エラーメッセージ（タイトル）                   │
│                     補足説明テキスト                              │
│                                                                 │
│                     [ アクションボタン（1〜2個） ]                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 共通デザイン仕様

| 項目 | 仕様 |
|------|------|
| コンテンツ横幅 | 最大600px・中央寄せ |
| テキスト配置 | 中央揃え |
| エラーコード表示 | 大きなフォント（実装時に決定）、ブランドカラーまたはグレー系 |
| タイトルフォント | h1相当のサイズ |
| 補足説明テキスト | 本文フォントサイズ。グレー系で控えめに表示 |
| イラスト・アイコン | エラー種別ごとに異なるアイコンまたはイラストを使用（詳細は各節参照） |
| ボタン配置 | 中央寄せ。複数ある場合は縦並びまたは横並び（実装時に決定） |

### 2.4 ページタイトル（`<title>`）

| エラー | ページタイトル |
|--------|--------------|
| 404 | `ページが見つかりません | トレヨミ` |
| 403 | `アクセスできません | トレヨミ` |
| 500 | `エラーが発生しました | トレヨミ` |
| 503 | `メンテナンス中 | トレヨミ` |

### 2.5 ヘッダー・フッターの扱い

| エラー | ヘッダー | フッター |
|--------|---------|---------|
| 404 | 通常表示 | 通常表示 |
| 403 | 通常表示 | 通常表示 |
| 500 | 通常表示（読み込み失敗の場合はロゴのみのフォールバック） | 通常表示（同上） |
| 503 | ロゴのみのシンプル表示 | 非表示（メンテナンス中のためナビゲーションリンクは不要） |

> **500エラー時のフォールバック**: アプリケーション全体が応答不能な状態では、ヘッダー・フッターの共通コンポーネント自体が読み込めない可能性がある。その場合はロゴのみを表示するシンプルなフォールバックレイアウトを使用する。

## 3. 404 Not Found

### 3.1 発生条件

- 存在しないURLへのアクセス
- 削除済みの再生リスト・ゲームタイトル・チャンネル・まとめへのアクセス
- URLのtypo等

### 3.2 ページレイアウト（PC）

```
┌─────────────────────────────────────────────────────────────────┐
│ ヘッダー（通常表示）                                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                          🔍                                     │
│                    （虫眼鏡系のアイコン）                          │
│                                                                 │
│                           404                                   │
│                  ページが見つかりません                            │
│                                                                 │
│     お探しのページは存在しないか、削除された可能性があります。        │
│     URLをご確認の上、再度お試しください。                           │
│                                                                 │
│                  [ TOPページへ戻る ]                             │
│                  [ 再生リストを探す ]                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
│ フッター（通常表示）                                               │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 表示要素

| 要素 | 内容 |
|------|------|
| アイコン | 虫眼鏡またはゲームコントローラー系のイラスト・アイコン（デザイン実装時に決定） |
| エラーコード | `404` |
| タイトル | 「ページが見つかりません」 |
| 補足説明 | 「お探しのページは存在しないか、削除された可能性があります。URLをご確認の上、再度お試しください。」 |

### 3.4 アクションボタン

| ボタン | 遷移先 | 種別 |
|-------|-------|------|
| TOPページへ戻る | `/` | プライマリボタン |
| 再生リストを探す | `/playlists` | セカンダリボタン（テキストリンク形式でも可） |

### 3.5 補足仕様

- ブラウザの「戻る」ボタンで前のページに戻れる場合と戻れない場合があるため、明示的なナビゲーションボタンを提供する。
- 検索バーは表示しない（ユーザーを混乱させないためシンプルに保つ）。

## 4. 403 Forbidden（権限エラー）

### 4.1 発生条件

- 非公開に設定されたまとめへの他ユーザーによるアクセス
- その他、ログインリダイレクト対象外の権限エラー

### 4.2 ページレイアウト（PC）

```
┌─────────────────────────────────────────────────────────────────┐
│ ヘッダー（通常表示）                                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                          🔒                                     │
│                    （鍵・施錠系のアイコン）                        │
│                                                                 │
│                           403                                   │
│                  このページにはアクセスできません                   │
│                                                                 │
│          このページを閲覧する権限がありません。                     │
│          コンテンツが非公開に設定されているか、                     │
│          アクセス権がない可能性があります。                         │
│                                                                 │
│                  [ TOPページへ戻る ]                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
│ フッター（通常表示）                                               │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 表示要素

| 要素 | 内容 |
|------|------|
| アイコン | 鍵・施錠系のアイコン（デザイン実装時に決定） |
| エラーコード | `403` |
| タイトル | 「このページにはアクセスできません」 |
| 補足説明 | 「このページを閲覧する権限がありません。コンテンツが非公開に設定されているか、アクセス権がない可能性があります。」 |

### 4.4 アクションボタン

| ボタン | 遷移先 | 種別 |
|-------|-------|------|
| TOPページへ戻る | `/` | プライマリボタン |

### 4.5 ログイン状態による表示の分岐

| ログイン状態 | 補足説明の追記 | ボタン追加 |
|------------|-------------|---------|
| 未ログイン | 「ログインすることでアクセスできる場合があります。」を補足説明に追記 | 「ログインする」ボタンを追加（`/login` へ遷移） |
| ログイン済み | 追記なし | 追加なし |

## 5. 500 Internal Server Error

### 5.1 発生条件

- サーバー内部での予期しないエラー
- API Routes でのエラー
- Firestore・外部API連携時の予期しない失敗

### 5.2 ページレイアウト（PC）

```
┌─────────────────────────────────────────────────────────────────┐
│ ヘッダー（通常表示。読み込み失敗時はロゴのみ）                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                          ⚠️                                     │
│                  （警告・エラー系のアイコン）                       │
│                                                                 │
│                           500                                   │
│                  エラーが発生しました                              │
│                                                                 │
│       申し訳ありません。サーバーで問題が発生しました。              │
│       時間をおいてから再度お試しください。                          │
│       問題が解決しない場合はお問い合わせください。                   │
│                                                                 │
│                  [ 再試行する ]                                  │
│                  [ TOPページへ戻る ]                             │
│                  [ お問い合わせ ]                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
│ フッター（通常表示。読み込み失敗時はロゴのみ）                       │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 表示要素

| 要素 | 内容 |
|------|------|
| アイコン | 警告・レンチ系のアイコン（デザイン実装時に決定） |
| エラーコード | `500` |
| タイトル | 「エラーが発生しました」 |
| 補足説明 | 「申し訳ありません。サーバーで問題が発生しました。時間をおいてから再度お試しください。問題が解決しない場合はお問い合わせください。」 |

### 5.4 アクションボタン

| ボタン | 動作 | 種別 |
|-------|------|------|
| 再試行する | 現在のページをリロードする | プライマリボタン |
| TOPページへ戻る | `/` へ遷移 | セカンダリボタン |
| お問い合わせ | `/contact` へ遷移 | テキストリンク |

### 5.5 補足仕様

- Next.js の `error.tsx` コンポーネントで実装する。`reset()` 関数を「再試行する」ボタンに割り当てる。
- エラーの技術的詳細（スタックトレース等）はユーザーに表示しない。ログはサーバーサイドのみに記録する。

## 6. 503 Service Unavailable（メンテナンス）

### 6.1 発生条件

- 計画メンテナンス中
- サービスの一時停止

### 6.2 ページレイアウト（PC）

```
┌─────────────────────────────────────────────────────────────────┐
│  [アプリ ロゴ]  （ロゴのみのシンプルヘッダー）                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                         🛠️                                      │
│                    （工事・メンテナンス系のアイコン）                │
│                                                                 │
│                  メンテナンス中です                               │
│                                                                 │
│        ただいまメンテナンスのためサービスを一時停止しています。       │
│        ご不便をおかけして申し訳ありません。                         │
│                                                                 │
│        メンテナンス予定期間：                                      │
│        YYYY年MM月DD日 HH:MM 〜 HH:MM（予定）                     │
│        （期間未定の場合は非表示）                                   │
│                                                                 │
│        メンテナンス終了後、自動的にページが更新されます。            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
│  （フッターなし）                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 6.3 表示要素

| 要素 | 内容 |
|------|------|
| ヘッダー | ロゴのみ（グローバルナビゲーション・ユーザーエリアは非表示） |
| アイコン | 工具・メンテナンス系のアイコン（デザイン実装時に決定） |
| エラーコード | 非表示（ユーザー向けにはコードを見せない） |
| タイトル | 「メンテナンス中です」 |
| 補足説明 | 「ただいまメンテナンスのためサービスを一時停止しています。ご不便をおかけして申し訳ありません。」 |
| メンテナンス期間 | 期間が確定している場合のみ表示。未確定の場合は非表示。 |
| 自動更新の案内 | 「メンテナンス終了後、自動的にページが更新されます。」 |
| フッター | 非表示 |

### 6.4 アクションボタン

503ページにはアクションボタンを設置しない。メンテナンス中にユーザーを他ページへ誘導する必要がないため。

### 6.5 自動リロード

| 項目 | 仕様 |
|------|------|
| 自動リロード間隔 | 60秒ごとにページを自動リロードする |
| 目的 | メンテナンス終了後にユーザーが手動で更新する手間を省く |
| 実装方法 | `<meta http-equiv="refresh" content="60">` またはJavaScriptによるタイマー |

### 6.6 メンテナンスページの適用方式

| 方式 | 内容 |
|------|------|
| Next.js ミドルウェア | `middleware.ts` でリクエストをインターセプトし、メンテナンスフラグが有効な場合は全パスを503ページへリダイレクトする |
| メンテナンスフラグ | 環境変数（`NEXT_PUBLIC_MAINTENANCE_MODE=true`）またはFirestoreのシステム設定ドキュメントで管理する（実装時に決定） |
| 適用除外パス | なし（管理画面は `/admin` 配下のため、フロントのメンテナンス制御とは独立） |
