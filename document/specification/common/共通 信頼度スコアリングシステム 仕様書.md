## 1. 概要

### 1.1 目的

ゲムコメのコア価値は「ちゃんと視聴した人の、信頼できる評価」を再生リストスコアに反映させることにある。

単純な平均値では、1話だけ視聴したユーザーの評価と全話完走したユーザーの評価が同等に扱われてしまう。本システムはレビューごとに**信頼度スコア**を算出し、それを重みとした加重平均によって再生リストスコアを算出する。

### 1.2 信頼度スコアに影響する4要素

| 要素 | 概要 |
|------|------|
| **視聴深度** | どこまで視聴したか。完走ステータスも加味する |
| **参考になった数** | コミュニティから評価されているレビューを優遇する |
| **コメント有無** | 理由を書いたレビューを優遇する |
| **ジャンル専門性** | そのゲームのジャンルに詳しいレビュワーを優遇する |

---

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| **信頼度スコア** | 1件のレビューが持つ重みを表すスコア。再生リストスコアの加重平均計算に使用する |
| **再生リストスコア** | 再生リスト全体の評価値。全レビューの信頼度加重平均によって算出される。5点満点 |
| **星評価** | レビュー投稿者がつける評価値。★0.5〜★5.0（0.5刻み） |
| **視聴深度** | 「最後に開いた話数 ÷ 再生リスト全話数」で算出される0〜1の値 |
| **ジャンル専門性** | マイリスト登録傾向から判定される、特定ジャンルへの精通度 |

---

## 3. 信頼度スコアの構成要素

### 3.1 視聴深度（`D`）

**定義**

```
D = 最後に開いた話数 ÷ 再生リスト全話数
```

- 値の範囲: 0.0〜1.0
- 「最後に開いた話数」の記録仕様は動画プレーヤー仕様書に準ずる

**完走ステータスによる補完**

レビュー投稿者の視聴ステータスが「完走」の場合、ゲムコメ上での視聴記録がなくても（例：YouTube直接視聴）視聴深度を `1.0` として扱う。

| 条件 | 視聴深度 D の値 |
|------|---------------|
| 視聴ステータスが「完走」 | `1.0`（固定） |
| 上記以外 | `最後に開いた話数 ÷ 全話数` |

**設計思想**: ゲムコメ外での視聴実績を排除しない。完走という明確な意思表示を最高の視聴深度として信頼する。

---

### 3.2 参考になった数（`H`）

**定義**

```
H = log(参考になった数 + 1)
```

- 底は自然対数（ln）を使用する
- 値の範囲: 0.0〜上限なし（ただし増加は緩やか）

**特性**

| 参考になった数 | H の値（概算） |
|--------------|-------------|
| 0件 | 0.00 |
| 1件 | 0.69 |
| 5件 | 1.79 |
| 10件 | 2.40 |
| 50件 | 3.93 |
| 100件 | 4.62 |

序盤の「0→1」の変化を大きく評価しつつ、突出したレビューがスコアを独占しすぎないよう緩やかに収束させる。

---

### 3.3 コメント有無（`C`）

**定義**

| 状態 | C の値 |
|------|--------|
| コメントあり | `1.0` |
| コメントなし（星評価・ステータスのみ） | `0.5` |

コメント有無は信頼度スコア全体への**乗数**として作用する。コメントなしのレビューは信頼度が半分になる。

**設計思想**: 根拠を言語化したレビューの影響力を高め、単なる数値投票との差別化を図る。

---

### 3.4 ジャンル専門性（`G`）

**定義**

```
G = log(レビュー対象ゲームと同一ジャンルのマイリスト登録数 + 1)
```

- 底は自然対数（ln）を使用する
- 「同一ジャンル」の判定: レビュー対象の再生リストに紐づくゲームタイトルのジャンルと、投稿者のマイリスト登録済み再生リストに紐づくゲームタイトルのジャンルが1つ以上一致する場合
- ゲームタイトルのジャンルはDBのジャンルマスタに準ずる

**算出に使用するデータ**

投稿者のマイリストに登録されている再生リストを参照し、各再生リストに紐づくゲームタイトルのジャンルを集計する。アカウント作成時に選択した好きなジャンルは使用しない。

**特性**

| 同ジャンルのマイリスト登録数 | G の値（概算） |
|--------------------------|-------------|
| 0件 | 0.00 |
| 1件 | 0.69 |
| 5件 | 1.79 |
| 10件 | 2.40 |
| 50件 | 3.93 |

**設計思想**: 特定ジャンルをよく追っているユーザーの評価を重くする。ただし専門性が高いユーザーがスコアを独占しないよう対数で緩和する。

## 4. 信頼度スコアの計算式

### 4.1 計算式

```
信頼度スコア W = (D + H + G) × C
```

| 変数 | 説明 |
|------|------|
| `D` | 視聴深度（0.0〜1.0） |
| `H` | 参考になった数の対数: `log(参考になった数 + 1)` |
| `G` | ジャンル専門性の対数: `log(同ジャンルのマイリスト登録数 + 1)` |
| `C` | コメント有無（コメントあり: 1.0 / コメントなし: 0.5） |

### 4.2 計算例

**例1: 全話完走・コメントあり・参考になった10件・RPGを10本マイリスト登録済みのレビュワーがRPGゲームをレビュー**

```
D = 1.0（完走ステータス）
H = log(10 + 1) ≈ 2.40
G = log(10 + 1) ≈ 2.40
C = 1.0（コメントあり）

W = (1.0 + 2.40 + 2.40) × 1.0 = 5.80
```

**例2: 20%視聴・コメントなし・参考になった0件・同ジャンルのマイリスト登録なし**

```
D = 0.20
H = log(0 + 1) = 0.00
G = log(0 + 1) = 0.00
C = 0.5（コメントなし）

W = (0.20 + 0.00 + 0.00) × 0.5 = 0.10
```

**例3: 80%視聴・コメントあり・参考になった5件・同ジャンルのマイリスト登録1件**

```
D = 0.80
H = log(5 + 1) ≈ 1.79
G = log(1 + 1) ≈ 0.69
C = 1.0（コメントあり）

W = (0.80 + 1.79 + 0.69) × 1.0 = 3.28
```

## 5. 再生リストスコアの計算式

### 5.1 計算式

再生リストに紐づく全レビューの信頼度加重平均によって再生リストスコアを算出する。

```
再生リストスコア = Σ(レビューiの星評価 × Wi) / Σ(Wi)
```

| 変数 | 説明 |
|------|------|
| `レビューiの星評価` | ★0.5〜★5.0の値（0.5刻み） |
| `Wi` | レビューiの信頼度スコア |

### 5.2 計算例

| レビュー | 星評価 | 信頼度スコア W |
|---------|--------|-------------|
| レビューA | 5.0 | 5.80 |
| レビューB | 4.0 | 3.28 |
| レビューC | 2.0 | 0.10 |

```
再生リストスコア = (5.0×5.80 + 4.0×3.28 + 2.0×0.10) / (5.80 + 3.28 + 0.10)
              = (29.00 + 13.12 + 0.20) / 9.18
              = 42.32 / 9.18
              ≈ 4.61
```

### 5.3 レビューが0件の場合

再生リストスコアは算出しない。表示箇所では「評価なし」と表示する。

### 5.4 星評価がないレビューの扱い

視聴ステータスのみ・コメントのみで星評価が未設定のレビューは、再生リストスコアの計算対象から除外する。ただしレビュー件数のカウントには含める。

## 6. 再生リストスコアの表示仕様

### 6.1 表示スケール

5点満点。小数点第1位まで表示する。

| 表示形式 | 例 |
|---------|-----|
| 数値表示 | `4.6` |
| 星表示 | ★（塗りつぶし）・☆（空）・半星（半分塗りつぶし）を組み合わせて表現 |

### 6.2 半星の表示条件

| 再生リストスコアの小数部 | 星の表示 |
|-------------------|---------|
| 0.0〜0.24 | 切り捨て（空星） |
| 0.25〜0.74 | 半星 |
| 0.75〜1.0 | 切り上げ（塗りつぶし） |

**例**: スコア 4.6 → ★★★★☆（4.5相当の半星あり表示）

### 6.3 スコア数値の色

再生リストスコアの数値はゴールド（`#F9A825`）で表示する。
