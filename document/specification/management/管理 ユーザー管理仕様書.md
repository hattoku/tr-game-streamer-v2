## 1. 概要

本仕様書は、ゲムコメ管理画面における以下の2機能を定義する。

- **ユーザー管理**：一般ユーザーのアカウント情報を閲覧・管理する機能
- **管理者（オーナー・運営者）アカウント管理**：運営者アカウントおよびAIキャラクターを管理する機能（オーナーのみ）

---

## 2. アクセス権限

| 機能 | オーナー | 運営者 |
|------|:---:|:---:|
| ユーザー管理（全操作） | ○ | ○ |
| 管理者（オーナー・運営者）アカウント管理 | ○ | × |
| 担当ジャンルの設定・変更 | ○（設定のみ） | ○（閲覧のみ） |

## 3. ユーザー管理

### 3.1 ユーザー一覧・検索

一般ユーザーの一覧を表示し、条件で絞り込む画面。

#### 表示項目

| 項目 | 内容 |
|------|------|
| ユーザー名 | — |
| メールアドレス | — |
| メール認証状態 | 認証済み／未認証 |
| BANステータス | BANなし／BAN中 |
| 登録日 | — |
| 直近のログイン日時 | — |

#### 検索・絞り込み条件

| 条件 | 種別 |
|------|------|
| ユーザー名 | テキスト検索 |
| メールアドレス | テキスト検索 |
| 登録日 | 日付範囲 |
| 直近のログイン日時 | 日付範囲 |
| メール認証状態 | 認証済み／未認証 |
| BANステータス | BANなし／BAN中 |

---

### 3.2 ユーザー詳細

個別ユーザーの詳細情報を閲覧する画面。

#### 表示項目

| カテゴリ | 項目 |
|---------|------|
| 基本情報 | ユーザー名・メールアドレス・登録日・直近のログイン日時 |
| 認証状態 | メール認証状態（認証済み／未認証） |
| BANステータス | BANなし／BAN中。BAN中の場合はBAN理由・BAN設定日時を表示 |
| アクティビティ | 投稿レビュー数・マイリスト登録数・視聴履歴件数 |

---

### 3.3 BAN設定・解除

ユーザーのログインを禁止する、または禁止を解除する操作。

#### BAN設定フロー

```
ユーザー詳細画面の「BANする」ボタンをクリック
    ↓
確認ダイアログ表示
    ├─ BAN理由入力欄（任意）
    └─ 「BANする」ボタン / 「キャンセル」ボタン
    ↓
BAN設定完了 → ユーザー詳細画面に反映
```

#### BAN解除フロー

```
ユーザー詳細画面の「BAN解除する」ボタンをクリック
    ↓
確認ダイアログ表示
    └─ 「BAN解除する」ボタン / 「キャンセル」ボタン
    ↓
BAN解除完了 → ユーザー詳細画面に反映
```

#### BANの効果

BAN中のユーザーはログインを試みた際にエラーメッセージを表示され、ログインできない。既存のセッションは無効化される。

---

### 3.4 強制退会

規約違反ユーザーのアカウントを削除する操作。物理削除。

#### 強制退会フロー

```
ユーザー詳細画面の「強制退会する」ボタンをクリック
    ↓
確認ダイアログ表示
    ├─ 退会理由入力欄（任意）
    └─ 「強制退会する」ボタン / 「キャンセル」ボタン
    ↓
強制退会完了
```

#### 削除されるデータ

強制退会時に削除されるデータの詳細は[第6章「削除ポリシー」](#6-削除ポリシー)を参照。

## 4. 管理者（オーナー・運営者）アカウント管理（オーナーのみ）

### 4.1 管理者（オーナー・運営者）一覧

管理者（オーナー・運営者）アカウント（オーナー・運営者）およびAIキャラクターの一覧を表示する画面。

#### 表示項目

| 項目 | 人間（オーナー・運営者） | AIキャラクター |
|------|:---:|:---:|
| 表示名 | ○ | ○（`[AI]` バッジ付き） |
| メールアドレス | ○ | —（表示しない） |
| ロール | ○（オーナー／運営者） | ○（AIキャラクター） |
| 担当ジャンル | ○（オーナーは表示しない） | ○ |
| 有効・無効 | ○ | ○ |
| 追加日 | ○（運営者ロールが付与された日付） | ○（作成日） |

- AIキャラクターは `[AI]` バッジを表示名の前に付与して表示する。
- オーナーには有効・無効の概念を適用しない。

---

### 4.2 運営者の追加

既存ユーザーへのロール付与、またはアカウント未作成者への招待メール送信により運営者を追加する。

#### 追加フロー

```
「運営者を追加する」ボタンをクリック
    ↓
メールアドレスを入力して検索
    ↓
    ├─ ゲムコメのアカウントが存在する場合
    │      └─ 対象ユーザーの情報を表示
    │             ↓
    │          確認ダイアログ表示
    │             ├─ 担当ジャンル選択（複数選択可・任意）
    │             └─ 「運営者ロールを付与する」ボタン / 「キャンセル」ボタン
    │             ↓
    │          運営者ロール付与完了 → 管理者（オーナー・運営者）一覧に追加
    │
    └─ ゲムコメのアカウントが存在しない場合
           └─ 「このメールアドレスにはアカウントが存在しません。招待メールを送信しますか？」を表示
                  ↓
               「招待メールを送信する」ボタンをクリック
                  ↓
               招待メール送信完了
```

> **補足**: 招待メール経由でアカウント登録が完了した場合、担当ジャンルは未設定の状態で運営者ロールが付与される。担当ジャンルはロール付与後に別途設定する。

#### 招待メールの仕様

| 項目 | 仕様 |
|------|------|
| 招待リンクの有効期限 | 24時間（アカウント登録時の確認メールと同様） |
| 有効期限切れ時 | リンクをクリックした際に「リンクの有効期限が切れています」の旨を表示 |
| 受け取り側のフロー | リンクをクリック → アカウント登録フォームへ遷移 → 登録完了後に運営者ロールを付与 |

---

### 4.3 運営者の削除

運営者ロールを剥奪する操作。ゲムコメの一般ユーザーアカウント自体は削除されない。

#### 削除フロー

```
管理者（オーナー・運営者）一覧の対象運営者の「ロールを剥奪する」ボタンをクリック
    ↓
確認ダイアログ表示
    └─ 「ロールを剥奪する」ボタン / 「キャンセル」ボタン
    ↓
運営者ロール剥奪完了 → 管理者（オーナー・運営者）一覧から削除
```

> **注意**: オーナーのロールは剥奪・変更できない。

---

### 4.4 担当ジャンルの設定・変更（運営者）

運営者ごとに担当するゲームジャンルを設定する。担当ジャンルはマスタ管理における運営者の編集権限範囲と、審査ワークフローにおける審査者アサインの対象範囲を決定する。

#### 権限

| 操作 | オーナー | 運営者 |
|------|:---:|:---:|
| 担当ジャンルの設定・変更 | ○ | × |
| 担当ジャンルの閲覧 | ○ | ○ |

#### 設定フロー

```
管理者（オーナー・運営者）一覧の対象運営者の「担当ジャンルを設定する」ボタンをクリック
    ↓
担当ジャンル設定モーダルを表示
    ├─ 現在の担当ジャンルを表示（未設定の場合は「未設定」と表示）
    └─ ゲームジャンルを複数選択（チェックボックス形式）
    ↓
「保存する」ボタンをクリック
    ↓
担当ジャンル更新完了 → 管理者（オーナー・運営者）一覧に反映
```

#### 仕様

| 項目 | 仕様 |
|------|------|
| 選択形式 | チェックボックスによる複数選択 |
| 選択数 | 0件以上（未設定も許容） |
| 選択肢 | DBのゲームジャンルマスタに登録されている全ジャンル |
| 運営者自身による変更 | 不可（閲覧専用） |

> **補足**: 担当ジャンルが未設定の運営者は、マスタ管理において編集可能な対象が存在せず、審査ワークフローの審査者アサイン対象にもならない。

---

### 4.5 有効・無効の切り替え

管理者（オーナー・運営者）アカウント（運営者）およびAIキャラクターを一時的に審査から外す操作。無効化しても権限やデータは保持される。

#### 有効・無効の定義

| 状態 | 審査アサイン | 管理画面へのログイン（運営者のみ） |
|------|:---:|:---:|
| 有効 | 対象 | 可 |
| 無効 | 対象外 | 不可 |

- 無効化時点ですでにアサイン済みのワークフローは、アサインをそのまま維持する。
- オーナーには有効・無効の切り替えを適用しない。

#### 切り替えフロー

```
管理者（オーナー・運営者）一覧の対象の「無効にする」ボタンをクリック
    ↓
確認ダイアログ表示
    └─ 「無効にする」ボタン / 「キャンセル」ボタン
    ↓
無効化完了 → 管理者（オーナー・運営者）一覧のステータスに反映
```

```
管理者（オーナー・運営者）一覧の対象の「有効にする」ボタンをクリック
    ↓
確認ダイアログ表示
    └─ 「有効にする」ボタン / 「キャンセル」ボタン
    ↓
有効化完了 → 管理者（オーナー・運営者）一覧のステータスに反映
```

### 4.6 AIキャラクター管理

審査ワークフローのLv1で審査者として機能するAIキャラクターを管理する。AIキャラクターの審査動作仕様は管理_審査ワークフロー仕様書 第5章を参照。

#### 自動生成

ゲームジャンルマスタに新しいジャンルが追加された時点で、そのジャンルに対応するAIキャラクターが自動生成される。

| 項目 | 自動生成時の値 |
|------|------|
| 表示名 | `{ジャンル名}審査AI`（例：RPG審査AI） |
| 担当ジャンル | 生成のトリガーとなったジャンル |
| 有効・無効 | 有効 |

#### 編集

オーナーはAIキャラクターの以下の項目を編集できる。

| 編集項目 | 内容 |
|------|------|
| 表示名 | 任意の名称に変更可 |
| 担当ジャンル | 複数ジャンルの設定・変更が可能 |

#### 担当ジャンルの設定フロー

```
管理者（オーナー・運営者）一覧の対象AIキャラクターの「担当ジャンルを設定する」ボタンをクリック
    ↓
担当ジャンル設定モーダルを表示
    ├─ 現在の担当ジャンルを表示
    └─ ゲームジャンルを複数選択（チェックボックス形式）
    ↓
「保存する」ボタンをクリック
    ↓
担当ジャンル更新完了 → 管理者（オーナー・運営者）一覧に反映
```

#### 削除

AIキャラクターの削除はできない。ジャンル廃止等により審査から外す場合は[4.5節「有効・無効の切り替え」](#45-有効無効の切り替え)で無効化する。

> **補足**: 削除を不可とするのは、AIキャラクターが審査ワークフローの審査者として記録された履歴（審査コメント等）を保持するためである。

## 5. テストモード用会員

### 5.1 概要

フロントのテストモードウィジェット（ゲムコメ_共通UIコンポーネント_フロント_仕様書 第7章）において、ログイン状態に切り替える際に使用するテスト専用の一般ユーザーアカウント。開発・動作確認時のみ使用する。

### 5.2 アカウントの定義

| 項目 | 仕様 |
|------|------|
| アカウント種別 | 一般ユーザー（ロールは通常の一般ユーザーと同一） |
| 用意方法 | オーナーが管理画面の設定からテストモード用会員を指定する |
| 指定可能数 | 1件 |
| アカウントの実体 | 通常のアカウント登録フローで作成した一般ユーザーアカウントをテスト用に充てる |

### 5.3 権限・制約

| 項目 | 仕様 |
|------|------|
| 機能権限 | 通常の一般ユーザーと同一 |
| BAN設定 | 対象外（管理画面上でBAN操作を行えない） |
| 強制退会 | 対象外（管理画面上で強制退会操作を行えない） |

テストモード用会員に指定されている間は、ユーザー詳細画面のBAN・強制退会の操作ボタンを非活性（disabled）にする。

### 5.4 管理画面での設定フロー

オーナーのみアクセス可能な設定ページから指定する。

```
管理画面の設定ページ（オーナーのみアクセス可）
    ↓
「テストモード用会員」セクションを表示
    ├─ 現在の指定状況（ユーザー名・メールアドレス）を表示
    │     未指定の場合は「未指定」と表示
    └─ 「変更する」ボタンをクリック
         ↓
      ユーザー検索モーダルを表示
         ├─ メールアドレスまたはユーザー名で検索
         └─ 対象ユーザーを選択して「指定する」ボタンをクリック
              ↓
           確認ダイアログ表示
              └─ 「指定する」ボタン / 「キャンセル」ボタン
                   ↓
                指定完了 → 設定ページに反映
```

### 5.5 指定解除

| 項目 | 仕様 |
|------|------|
| 解除操作 | 設定ページの「テストモード用会員」セクションの「指定を解除する」ボタンから解除 |
| 解除後の挙動 | テストモードウィジェットでのログイン切り替えボタンが非活性になる |
| アカウントへの影響 | 指定解除してもアカウント自体は削除されない |

### 5.6 操作ログ

| 操作 | 記録内容 |
|------|---------|
| テストモード用会員の指定 | 操作日時・操作者・指定したユーザーID |
| テストモード用会員の指定解除 | 操作日時・操作者・解除したユーザーID |

## 6. 操作ログ

本仕様書が対象とする機能における操作ログの記録対象を定義する。操作ログの閲覧仕様はゲムコメ_管理画面_機能一覧_仕様書 3.9節を参照。

### 6.1 記録対象操作

| 操作 | 記録内容 |
|------|---------|
| BAN設定 | 操作日時・操作者・対象ユーザーID・BAN理由（入力された場合） |
| BAN解除 | 操作日時・操作者・対象ユーザーID |
| 強制退会 | 操作日時・操作者・対象ユーザーID・退会理由（入力された場合） |
| 運営者ロール付与 | 操作日時・操作者・対象ユーザーID |
| 運営者ロール剥奪 | 操作日時・操作者・対象ユーザーID |
| 担当ジャンル変更（運営者） | 操作日時・操作者・対象運営者ID・変更前ジャンル・変更後ジャンル |
| 有効・無効切り替え（運営者） | 操作日時・操作者・対象運営者ID・変更後ステータス |
| AIキャラクター表示名変更 | 操作日時・操作者・対象AIキャラクターID・変更前名称・変更後名称 |
| AIキャラクター担当ジャンル変更 | 操作日時・操作者・対象AIキャラクターID・変更前ジャンル・変更後ジャンル |
| AIキャラクター有効・無効切り替え | 操作日時・操作者・対象AIキャラクターID・変更後ステータス |
| テストモード用会員の指定 | 操作日時・操作者・指定したユーザーID |
| テストモード用会員の指定解除 | 操作日時・操作者・解除したユーザーID |

> 操作ログは管理者（オーナー・運営者）による編集・削除を不可とする。閲覧専用。

## 7. 削除ポリシー

### 削除方式

| 操作 | 削除方式 |
|------|---------|
| 強制退会 | 物理削除 |

### 強制退会時に削除されるデータ

| データ種別 | 扱い |
|-----------|------|
| ユーザーアカウント | 削除 |
| 投稿レビュー | 削除 |
| 視聴履歴 | 削除 |
| マイリスト登録 | 削除 |
| 他ユーザーのレビューへの「参考になった」押下記録 | 削除 |
| 他ユーザーのレビューに対する「参考になった」被カウント数 | 保持（カウントはそのまま残る） |
