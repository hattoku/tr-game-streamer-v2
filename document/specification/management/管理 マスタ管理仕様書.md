## 1. 概要

本仕様書は、プレミテ管理画面におけるマスタ管理機能を定義する。

マスタ管理では以下の4つのタブを切り替えて管理する。各タブはさらにサブタブで細分化される。

| タブ | サブタブ |
|------|---------|
| 再生リスト | 再生リスト／タグ |
| ゲームタイトル | ゲームタイトル／ゲームジャンル／ゲームテーマ／タグ |
| チャンネル | — |
| NGワード | — |

---

## 2. タグの概念定義

タグには2つの独立した軸がある。

### 2.1 出自による分類（タグマスタ上の属性）

| 種別 | 定義 | 登録方法 |
|------|------|---------|
| **運営者タグ** | 運営が命名規則に基づき事前に設計した公式タグ。タグ乱立を防ぐ語彙統制の役割を持つ（例：「ドラゴンクエスト」「パワプロ」）。マスタとして保護される | 管理画面のタグサブタブからのみ登録可能。ゲームタイトル・再生リストの登録・編集画面からは登録不可 |
| **ユーザータグ** | ログインユーザーがゲームタイトルや再生リストへの付与操作を通じて自由に追加するタグ | ユーザーが初めてタグを付与したタイミングで自動採番・登録される |

運営者タグはタグ名の変更・削除が不可であり、同一または類似のタグ名での重複登録もできない。ユーザーはゲームタイトルや再生リストへ運営者タグを**付与することはできる**。

### 2.2 付与状態による分類（ゲームタイトル・再生リストへの付与時の属性）

| 種別 | 定義 | 設定者 |
|------|------|-------|
| **固定タグ** | ゲームタイトルまたは再生リストの登録時に管理者（オーナー・運営者）が設定する。ユーザーが付与先から外すことができない | 管理者（オーナー・運営者）のみ設定・解除可能 |
| **非固定タグ** | ユーザーが付与先から自由に追加・削除できる | 全ログインユーザー |

### 2.3 2軸の組み合わせ

| | 運営者タグ | ユーザータグ |
|---|---|---|
| **固定（外せない）** | あり得る | 存在しない |
| **非固定（外せる）** | あり得る | あり得る |

> ユーザータグは常に非固定である。固定タグとして設定できるのは管理者（オーナー・運営者）のみであり、ユーザーが追加したタグが固定になることはない。

## 3. アクセス権限

### 3.1 閲覧権限

| 機能 | オーナー | 運営者 |
|------|:---:|:---:|
| 全マスタの閲覧 | ○ | ○ |

### 3.2 編集権限

オーナーはすべてのマスタを編集できる。運営者は担当ジャンルに基づいて編集範囲が制限される。

| 機能 | オーナー | 運営者 |
|------|:---:|:---:|
| ゲームタイトル（担当ジャンルに紐づくもの） | ○ | ○ |
| ゲームタイトル（担当外ジャンルのみに紐づくもの） | ○ | × |
| 再生リスト（担当ジャンルのゲームタイトルに紐づくもの） | ○ | ○ |
| 再生リスト（担当外ジャンルのゲームタイトルのみに紐づくもの） | ○ | × |
| チャンネル（担当ジャンルのゲームタイトルに紐づく再生リストを持つもの） | ○ | ○ |
| チャンネル（担当外のみに紐づくもの） | ○ | × |
| ゲームジャンル | ○ | × |
| ゲームテーマ | ○ | × |
| 運営者タグの新規追加 | ○ | ○ |
| 運営者タグの編集・削除 | × | × |
| ユーザータグの編集・削除 | ○ | ○ |
| NGワード | ○ | × |

> **補足**: 運営者への担当ジャンルの割り振りは別途仕様書で定義する。1人の運営者が複数のジャンルを担当することがある。

### 3.3 編集ボタンの制御

運営者が編集権限を持たない行については、編集ボタンおよび削除ボタンをグレーアウトして非活性にする。閲覧（一覧表示）は担当外のデータも含めて全件行える。

## 5. 共通UI仕様

### 5.1 リスト表示

各サブタブのコンテンツはリスト（テーブル）形式で表示する。管理者（オーナー・運営者）が一度に多くの情報を確認できるよう、行高さはコンパクトに設計する。

### 5.2 ページネーション

全サブタブのリストにページネーションを設ける。

| 項目 | 仕様 |
|------|------|
| デフォルト表示件数 | 20件 |
| 切り替え可能な表示件数 | 20 / 50 / 100件 |
| 切り替えUI | リスト右上にセレクトボックスで配置（例：「20件表示 ▼」） |
| ページ送りUI | リスト下部にページ番号リンク形式で配置 |

### 5.3 操作ボタンの配置

編集・削除等の操作ボタンは各行の右端に配置する。

```
| データ列 ... | [編集] [削除] |
```

### 5.4 確認ダイアログ

削除操作は必ず確認ダイアログを表示し、誤操作を防止する。

```
確認ダイアログ
    ├─ 削除内容の説明（紐づくデータへの影響を含む）
    └─ 「削除する」ボタン / 「キャンセル」ボタン
```

### 5.5 新規追加ボタン

新規追加が可能なサブタブ（ゲームタイトル・ゲームジャンル・ゲームテーマ・タグ）には、リスト右上に「+ 追加する」ボタンを配置する。

### 5.6 編集モーダル

「編集」ボタンをクリックするとモーダルを表示し、編集フォームを提供する。保存・キャンセルボタンをモーダル内に配置する。

### 5.7 検索選択モーダル

件数が膨大になりうる項目（再生リスト名・ゲームタイトル名・チャンネル名）の検索・選択には、共通の「検索選択モーダル」を使用する。絞り込み条件での使用と、編集モーダル内での紐づき選択の両方に適用する。

#### UI仕様

```
┌─────────────────────────────────────────────┐
│ ゲームタイトルを選択                    [ × ] │
├─────────────────────────────────────────────┤
│ [ キーワードで検索...              ]         │
├─────────────────────────────────────────────┤
│ □ ┌──┐ ゼルダの伝説 ブレス オブ ザ ワイルド  │
│   └──┘                                      │
│ □ ┌──┐ パワフルプロ野球2024-2025             │
│   └──┘                                      │
│ □ ┌──┐ モンスターハンターワールド             │
│   └──┘                                      │
│ ...                                         │
├─────────────────────────────────────────────┤
│ 3件選択中          [ キャンセル ] [ 決定 ]   │
└─────────────────────────────────────────────┘
```

#### 動作仕様

| 項目 | 仕様 |
|------|------|
| 起動 | 各絞り込み条件の「検索」ボタンをクリックするとモーダルが開く |
| キーワード検索 | テキスト入力に連動してリアルタイムに候補を絞り込む（部分一致・デバウンス300ms） |
| サムネイル表示 | 各候補行にサムネイルを小サイズで表示する。誤操作防止のための視覚的な識別用途。取得元は対象データ種別に応じて以下のとおり。ゲームタイトル：楽天ブックスのパッケージ画像（縦長・約3:4）／再生リスト：YouTubeの再生リストサムネイル（横長・16:9）／チャンネル：YouTubeのチャンネルアイコン（正方形） |
| 複数選択 | チェックボックスで複数選択可能 |
| 選択件数表示 | モーダル下部に「N件選択中」と表示 |
| 決定 | 「決定」ボタンをクリックすると選択内容を確定してモーダルを閉じる |
| キャンセル | 「キャンセル」ボタンまたは「×」ボタンで選択内容を破棄してモーダルを閉じる |
| 選択済み状態の表示 | モーダルを再度開いたとき、既に選択済みの項目はチェック済み状態で表示する |

#### 確定後の表示

絞り込み条件エリアでは、確定した選択内容をタグ形式（例：「パワプロ ×」）で検索ボタン横に表示する。「×」をクリックすることで個別に選択解除できる。

## 6. 再生リストタブ

### 6.1 再生リスト（サブタブ）

#### 検索・絞り込み

```
[ 再生リスト名を検索 ]  [ ゲームタイトルを検索 ]  [ チャンネルを検索 ]  [公開状態 ▼]
```

各ボタンをクリックすると検索選択モーダル（5.7節参照）が開く。公開状態のみドロップダウン選択。

| 条件 | 種別 |
|------|------|
| 再生リスト名 | 検索選択モーダル（複数選択可・部分一致） |
| ゲームタイトル | 検索選択モーダル（複数選択可・部分一致） |
| チャンネル | 検索選択モーダル（複数選択可・部分一致） |
| 公開状態 | ドロップダウン選択（公開／非公開） |

#### リスト表示項目

| 列 | 内容 |
|----|------|
| サムネイル | YouTubeの再生リストサムネイル画像（小・横長16:9） |
| 再生リスト名 | — |
| ゲームタイトル | 紐づくゲームタイトル名。未紐づけの場合は「-」を表示 |
| チャンネル名 | 紐づくチャンネル名。未紐づけの場合は「-」を表示 |
| タグ | 付与されているタグID（複数の場合はカンマ区切り） |
| 動画数 | 再生リスト内の動画数 |
| マイリスト登録数 | — |
| レビュー数 | — |
| 公開状態 | 公開／非公開 |
| 操作 | [編集] [公開／非公開切り替え] |

#### 編集

編集モーダルで以下の項目を変更できる。

| 項目 | 操作 |
|------|------|
| 再生リスト名 | テキスト入力 |
| ゲームタイトルの紐づき | 検索選択モーダル（単一選択・部分一致）で選択・解除 |
| チャンネルの紐づき | 検索選択モーダル（単一選択・部分一致）で選択・解除 |
| タグ | タグIDで追加・削除。固定タグの設定・解除を含む |

> 編集モーダル内のゲームタイトル・チャンネルの紐づきは1件のみ選択可能（単一選択）。絞り込み条件での使用とは異なる。

#### 公開・非公開の切り替え

| 操作 | 処理 |
|------|------|
| 公開 → 非公開 | 論理削除（削除フラグを立てる）。紐づくレビュー・視聴履歴・マイリスト登録は保持される |
| 非公開 → 公開 | 削除フラグを解除する |

> 再生リストの物理削除は本画面では行わない。非公開化（論理削除）が削除に相当する操作となる。

### 6.2 タグ（サブタブ）

再生リストに付与されるタグを管理する。

#### タグIDの採番ルール

| 項目 | 仕様 |
|------|------|
| 形式 | `TAG-{連番}` |
| 連番 | 1始まりの整数。ゼロ埋めなし（例：TAG-1、TAG-42、TAG-1000） |
| 採番タイミング（運営者タグ） | 管理画面のタグサブタブから新規追加した時点で自動採番 |
| 採番タイミング（ユーザータグ） | ユーザーが初めてタグを再生リストに付与した時点で自動採番 |
| 採番後の変更 | 不可 |

#### 検索・絞り込み

```
[ タグIDで検索 ]  [ タグ名で検索 ]  [種別 ▼]
```

| 条件 | 種別 |
|------|------|
| タグID | テキスト検索（完全一致） |
| タグ名 | テキスト検索（部分一致） |
| 種別 | ドロップダウン選択（すべて／運営者タグ／ユーザータグ） |

#### リスト表示項目

| 列 | 内容 |
|----|------|
| タグID | `TAG-{連番}` 形式 |
| タグ名 | — |
| 種別 | 運営者タグ／ユーザータグ |
| 使用再生リスト数 | このタグが付与されている再生リストの件数 |
| 操作 | [削除]（ユーザータグのみ。運営者タグはグレーアウト） |

#### 新規追加（運営者タグのみ）

リスト右上の「+ 追加する」ボタンをクリックするとモーダルを表示する。

| 項目 | 操作 |
|------|------|
| タグ名 | テキスト入力 |

- 既存の運営者タグと完全一致または類似するタグ名が入力された場合、候補を表示して重複登録を防ぐ。
- 保存時に自動でタグIDが採番される。
- 登録されたタグは運営者タグとして区別される。

> 管理画面から新規追加できるのは運営者タグのみ。ユーザータグはユーザーの付与操作によってのみ生成される。

#### 編集・削除

| 操作 | 運営者タグ | ユーザータグ |
|------|:---:|:---:|
| タグ名の編集 | × | ○ |
| 削除 | × | ○ |

> 運営者タグはタグ名の変更・削除が不可。タグマスタとして恒久的に保護される。ユーザータグの削除は物理削除。紐づく再生リストからタグの紐づきが外れる。詳細は[第11章「削除ポリシー」](#11-削除ポリシー)を参照。

## 7. ゲームタイトルタブ

### 7.1 ゲームタイトル（サブタブ）

#### 検索・絞り込み

```
[ ゲームタイトル名を検索 ]  [ゲームジャンル ▼]  [ゲームテーマ ▼]
```

| 条件 | 種別 |
|------|------|
| ゲームタイトル名 | 検索選択モーダル（複数選択可・部分一致） |
| ゲームジャンル | ドロップダウン選択 |
| ゲームテーマ | ドロップダウン選択 |

#### リスト表示項目

| 列 | 内容 |
|----|------|
| パッケージ画像 | 楽天ブックスから取得したサムネイル画像（小） |
| ゲームタイトル名 | — |
| ゲームジャンル | 紐づくジャンル名（複数の場合はカンマ区切り） |
| ゲームテーマ | 紐づくテーマ名（複数の場合はカンマ区切り） |
| タグ | 付与されているタグID（複数の場合はカンマ区切り） |
| 再生リスト数 | 紐づく再生リストの件数 |
| 操作 | [編集] [削除] |

#### 新規追加・編集

「+ 追加する」ボタンまたは「編集」ボタンをクリックするとモーダルを表示する。

モーダルで編集できる項目は以下のとおり。

| 項目 | 操作 |
|------|------|
| 楽天ブックス JANコード | テキスト入力（13桁）。入力後に楽天ブックスAPIで商品情報を自動取得 |
| 楽天ブックス 商品名 | テキスト入力。APIから自動取得、手動での上書き修正も可 |
| 楽天ブックス URL | テキスト入力。APIから自動取得 |
| ゲームジャンル | DBのジャンルマスタから複数選択。楽天ブックスのカテゴリパス（第4階層）と照合できた場合は自動入力 |
| ゲームテーマ | DBのテーママスタから複数選択。楽天ブックスのカテゴリパス（第5階層）と照合できた場合は自動入力 |
| タグ | タグIDで追加・削除。固定タグ（編集不可）と非固定タグを区別して設定 |

> JANコードの自動取得・編集フローの詳細はゲームタイトル詳細ページ仕様書 第7章「管理者（オーナー・運営者）による情報編集」に準ずる。

#### 修正提案履歴

編集モーダル内に、このゲームタイトルに対して一般ユーザーから提出された修正提案の履歴を一覧表示するセクションを設ける。

```
┌─────────────────────────────────────────────────────┐
│ 修正提案履歴                                          │
│                                                     │
│ 提案日時          提案者         処理結果  処理日時  │
│ 2026-03-01 12:00  user_abc       承認済み  2026-03-02│
│ 2026-02-20 09:30  user_xyz       却下済み  2026-02-21│
│ ...                                                 │
└─────────────────────────────────────────────────────┘
```

| 列 | 内容 |
|----|------|
| 提案日時 | 提案が提出された日時 |
| 提案者 | 提案したユーザー名 |
| 処理結果 | 承認済み／却下済み／審査中 |
| 処理日時 | 承認または却下が行われた日時。審査中の場合は「-」を表示 |

行をクリックすると、対応する審査フローの詳細画面へ遷移する。

> **注意**: 審査フローの詳細画面仕様は審査ワークフロー仕様書（未作成）に委ねる。本仕様書では遷移元としての一覧表示のみを定義する。

#### 削除

物理削除。紐づく再生リストは削除されず、ゲームタイトルとの紐づきのみが外れる。詳細は[第11章「削除ポリシー」](#11-削除ポリシー)を参照。

### 7.2 ゲームジャンル（サブタブ）

楽天ブックスのカテゴリパス第4階層に対応するマスタ。

#### リスト表示項目

| 列 | 内容 |
|----|------|
| ゲームジャンル名 | — |
| 紐づくゲームタイトル数 | このジャンルが設定されているゲームタイトルの件数 |
| 操作 | [編集] [削除] |

#### 新規追加

「保存する」をクリックすると以下の処理が同時に実行される。

1. ゲームジャンルをマスタに登録する
2. `{ジャンル名}審査AI` という名前のAI運営者をユーザーとして自動生成する

> AI運営者はシステム上AIフラグが立ったユーザーアカウントとして生成される。生成に際して管理者（オーナー・運営者）が設定できる項目はなく、ジャンル名から命名規則に従って完全自動で生成される。AI運営者の役割・動作仕様は/document/specification/management/管理 審査ワークフロー仕様書.md 第5章を参照。

|------|------|
| ゲームジャンル名 | テキスト入力 |

## 11. 削除ポリシー

### 11.1 削除方式

| 削除対象 | 削除方式 | 紐づくデータの扱い |
|---------|---------|----------------|
| ゲームタイトル | 物理削除 | 紐づく再生リストは残る。ゲームタイトルとの紐づきのみ外れる |
| 再生リスト | 論理削除（非公開化） | 紐づくレビュー・視聴履歴・マイリスト登録は保持される |
| チャンネル | 物理削除 | 紐づく再生リストは残る。チャンネルとの紐づきのみ外れる |
| ゲームジャンル | 物理削除 | 紐づくゲームタイトルからジャンルの紐づきが外れる。対応するAI運営者は削除されず残る |
| ゲームテーマ | 物理削除 | 紐づくゲームタイトルからテーマの紐づきが外れる |
| 運営者タグ | 削除不可 | タグマスタとして恒久的に保護される |
| 再生リストのユーザータグ | 物理削除 | 紐づく再生リストからタグの紐づきが外れる |
| ゲームタイトルのユーザータグ | 物理削除 | 紐づくゲームタイトルからタグの紐づきが外れる |
| NGワード | 物理削除 | 削除されたNGワードは即時バリデーション対象から外れる |
