## 1. 概要

本仕様書は、AI運営者が収集したYouTube再生リスト候補をオーナーが確認・承認・却下する管理画面UIの仕様を定義する。

Function C（AI運営者_コンテンツ収集機能仕様書）が定期的にFirestoreの `collection_candidates` コレクションに保存した収集候補に対して、オーナーが最終判断を行うことでゲムコメへの再生リスト登録が完了する。

---

## 2. アクセス権限

| 操作 | オーナー | 運営者 |
|------|:---:|:---:|
| 収集候補一覧の閲覧 | ○ | ✕ |
| 収集候補詳細の閲覧 | ○ | ✕ |
| 承認・却下 | ○ | ✕ |
| 検索キーワードの管理 | ○ | ✕ |

収集候補レビューUIはオーナー専用とする。AI運営者が収集したコンテンツの登録可否はプラットフォーム責任者であるオーナーが判断する。

## 3. 収集候補一覧画面

### 3.1 画面の位置

管理画面のサイドナビゲーションに「コンテンツ収集」セクションを設け、その配下に「収集候補」ページを配置する。

```
管理画面ナビゲーション
 ├─ ダッシュボード
 ├─ 審査ワークフロー
 ├─ コンテンツ収集          ← 新規追加
 │   ├─ 収集候補            ← 本画面
 │   └─ キーワード管理       ← 第6章
 ├─ マスタ管理
 └─ ...
```

### 3.2 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ コンテンツ収集候補                                            │
│                                                             │
│ 最終収集日時: 2026-03-16 03:12   新着: 47件                  │
│                                                             │
│ [すべて] [未レビュー] [承認済み] [却下済み] [登録済み]        │
│                                                             │
│ ─ 絞り込み ──────────────────────────────────────────────── │
│ [ ジャンル ▼ ]  [ フラグ ▼ ]  [ 検索キーワード入力 ]  [検索] │
│ ────────────────────────────────────────────────────────── │
│                                                             │
│ ┌──────────────────────────────────────────────────────┐   │
│ │ [サムネイル] タイトル                チャンネル名      │   │
│ │              ジャンル               動画数  最終投稿  │   │
│ │              [フラグバッジ...]                        │   │
│ │              AI: 「判定コメントの冒頭...」 [詳細を見る]│   │
│ │                             [承認する] [却下する]     │   │
│ └──────────────────────────────────────────────────────┘   │
│  （繰り返し）                                                │
│                                                             │
│ ページネーション                                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 ステータスタブ

| タブ | 表示対象 |
|------|---------|
| すべて | ステータスを問わず全件 |
| 未レビュー | `status: pending` のみ |
| 承認済み | `status: approved` のみ |
| 却下済み | `status: rejected` のみ |
| 登録済み | `status: registered` のみ |

- デフォルト表示は「未レビュー」タブ。
- ナビゲーションの「コンテンツ収集」に未レビュー件数バッジを表示する。

### 3.4 絞り込み条件

| 条件 | 種別 |
|------|------|
| ジャンル | ドロップダウン（全ジャンル一覧から選択） |
| フラグ | ドロップダウン（not_gameplay / update_stopped / completed / spam / non_japanese / フラグなし） |
| タイトル・チャンネル名 | テキスト入力（部分一致） |

### 3.5 一覧カードの表示項目

一覧はカード形式で表示する。

| 表示項目 | 内容 |
|---------|------|
| サムネイル | YouTubeの再生リストサムネイル画像（16:9） |
| 再生リストタイトル | — |
| チャンネル名 | — |
| ジャンル | AI運営者が担当するジャンル名 |
| 動画数 | — |
| 最終動画投稿日 | `latestVideoPublishedAt` を `YYYY-MM-DD` 形式で表示 |
| フラグバッジ | 該当フラグをバッジとして表示。フラグなしの場合は表示しない |
| AI判定コメント | `aiComment` の冒頭50文字を表示。「詳細を見る」リンクで詳細画面へ遷移 |
| 操作ボタン | 「承認する」「却下する」（未レビュー時のみ活性。承認済み・却下済み・登録済みはグレーアウト） |

#### フラグバッジの表示仕様

| フラグ | バッジラベル | 色 |
|-------|-----------|-----|
| `not_gameplay` | 実況以外の可能性 | 黄系 |
| `update_stopped` | 更新停止の可能性 | 黄系 |
| `completed` | 完結済みの可能性 | 青系 |
| `non_japanese` | 日本語以外の可能性 | 黄系 |
| `spam` | スパム疑い | 赤系 |

### 3.6 ページネーション

ゲムコメ_共通UIコンポーネント_管理_仕様書の共通ページネーション仕様に従う。デフォルト表示件数は20件。

| 表示項目 | 内容 |

| 表示項目 | 内容 |
|---------|------|
| 担当AI | AI運営者の表示名（例：RPG審査AI） |

収集時に取得した動画タイトル・公開日時を先頭から最大30件、番号付きで表示する。

| 列 | 内容 |
|----|------|
| # | 再生リスト内の順番（1始まり） |
| 動画タイトル | — |

オーナーが任意でコメントを記入できるテキストエリア。承認・却下時に入力内容が `reviewComment` としてFirestoreに保存される。入力は任意。

## 5. 承認・却下フロー

### 5.1 承認フロー

一覧カードまたは詳細画面の「承認する」ボタンから実行する。

```
「承認する」ボタンをクリック
    ↓
確認ダイアログを表示
  ├─ オーナーコメント入力欄（任意）
  └─ 「承認する」ボタン / 「キャンセル」ボタン
    ↓
再生リスト登録処理を実行：
  ① チャンネルがゲムコメ未登録の場合 → チャンネルレコードを新規作成
  ② 再生リストレコードをゲムコメ本番DB（playlists コレクション）に登録
  ③ collection_candidates のステータスを registered に更新
  ④ reviewedAt・reviewedBy・reviewComment を記録
    ↓
一覧画面に戻る（または次の候補に移動）
```

#### チャンネルの自動登録

承認時、対象再生リストの `channelId` がゲムコメの `channels` コレクションに存在しない場合、YouTube Data API v3（`channels.list`）でチャンネル情報を取得して自動登録する。チャンネル情報の取得に失敗した場合はエラーダイアログを表示し、承認処理を中断する。

#### ゲームタイトルの紐づけ

承認時、ゲームタイトルの紐づけは自動では行わない。登録後にオーナーまたは運営者がマスタ管理画面から手動で紐づける。

> **理由**: AI運営者が検索ジャンルからゲームタイトルを特定することは精度面で不確実であり、初期実装では人間が明示的に紐づける設計とする。将来フェーズで自動紐づけを検討する。

### 5.2 却下フロー

```
「却下する」ボタンをクリック
    ↓
確認ダイアログを表示
  ├─ 却下コメント入力欄（任意）
  └─ 「却下する」ボタン / 「キャンセル」ボタン
    ↓
collection_candidates のステータスを rejected に更新
reviewedAt・reviewedBy・reviewComment を記録
    ↓
一覧画面に戻る（または次の候補に移動）
```

### 5.3 却下済み候補の再審査

却下済み候補は「却下済み」タブから閲覧できる。誤って却下した場合のために、却下済み候補をオーナーが再度「承認する」ことができる。

## 6. 検索キーワード管理

### 6.1 画面の位置

管理画面ナビゲーションの「コンテンツ収集 > キーワード管理」ページ。

### 6.2 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ 検索キーワード管理                                           │
│                                                             │
│ ジャンル: [RPG ▼]                                           │
│                                                             │
│ 現在のキーワード一覧（RPG）                                  │
│ ┌──────────────────────────────────────────────────────┐   │
│ │  RPG 実況 プレイ                              [削除]  │   │
│ │  JRPG 実況                                   [削除]  │   │
│ │  ロールプレイング 実況                         [削除]  │   │
│ └──────────────────────────────────────────────────────┘   │
│                                                             │
│ キーワードを追加                                             │
│ [ テキスト入力欄 ]  [追加する]                              │
└─────────────────────────────────────────────────────────────┘
```

### 6.3 操作仕様

| 操作 | 仕様 |
|------|------|
| ジャンル切り替え | ドロップダウンでジャンルを選択すると、そのジャンルのキーワード一覧に切り替わる |
| キーワード追加 | テキスト入力欄に入力して「追加する」をクリック。即時Firestoreに反映する |
| キーワード削除 | 「削除する」ボタンで対象キーワードを削除。確認ダイアログは表示しない |
| 編集 | インプレース編集は行わない。削除して再追加する |

### 6.4 バリデーション

| 条件 | 処理 |
|------|------|
| 入力が空 | 「追加する」ボタンを非活性にする |
| 同一ジャンルに同一キーワードが既に存在する | 重複エラーを表示し追加しない |
| キーワードが50文字を超える | 入力不可（文字数制限） |

## 7. 通知

### 7.1 収集完了通知

Function C（AI運営者_コンテンツ収集バッチ）の実行完了時に、管理画面のお知らせ機能を通じてオーナーに通知する。

| 項目 | 内容 |
|------|------|
| 通知タイミング | Function Cの実行完了時 |
| 通知先 | オーナー |
| 通知内容 | 収集候補件数・収集日時・収集候補一覧画面へのリンク |

スキップ（クォータ不足）が発生した場合の通知は、AI運営者_コンテンツ収集機能仕様書 第7章に準ずる。
