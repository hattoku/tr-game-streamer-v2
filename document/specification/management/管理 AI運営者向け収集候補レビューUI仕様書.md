## 1. 概要

本仕様書は、AI運営者が収集したYouTube再生リスト候補をオーナーが確認・承認・却下する管理画面UIの仕様を定義する。

Function C（AI運営者_コンテンツ収集機能仕様書）が定期的にFirestoreの `collection_candidates` コレクションに保存した収集候補に対して、オーナーが最終判断を行うことでプレミテへの再生リスト登録が完了する。

---

## 2. アクセス権限

| 操作 | オーナー | 運営者 |
|------|:---:|:---:|
| 収集候補一覧の閲覧 | ○ | ✕ |
| 収集候補詳細の閲覧 | ○ | ✕ |
| 承認・却下 | ○ | ✕ |

収集候補レビューUIはオーナー専用とする。AI運営者が収集したコンテンツの登録可否はプラットフォーム責任者であるオーナーが判断する。

## 3. 収集候補一覧画面

### 3.1 画面の位置

管理画面のサイドナビゲーションに「コンテンツ収集」セクションを設け、その配下に「収集候補」ページを配置する。

```
管理画面ナビゲーション
 ├─ ダッシュボード
 ├─ 審査ワークフロー
 ├─ コンテンツ収集
 │   ├─ 収集候補            ← 本画面
 ├─ マスタ管理
 └─ ...
```

### 3.2 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ コンテンツ収集候補                                            │
│                                                             │
│ 最終収集日時: 2026-03-16 03:12   新着: 47件                  │
│                                                             │
│ [すべて] [未レビュー] [承認済み] [却下済み] [登録済み]        │
│                                                             │
│ ─ 絞り込み ──────────────────────────────────────────────── │
│ [ ジャンル ▼ ]  [ フラグ ▼ ]  [ タイトル・チャンネル名入力 ]  [検索] │
│ ────────────────────────────────────────────────────────── │
│                                                             │
│ ┌──────────────────────────────────────────────────────┐   │
│ │ [サムネイル] タイトル                チャンネル名      │   │
│ │              ジャンル               動画数  最終投稿  │   │
│ │              [フラグバッジ...]                        │   │
│ │              AI: 「判定コメントの冒頭...」 [詳細を見る]│   │
│ │                             [承認する] [却下する]     │   │
│ └──────────────────────────────────────────────────────┘   │

│                                                             │
│ ページネーション                                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 ステータスタブ

| タブ | 表示対象 |
|------|---------|
| すべて | ステータスを問わず全件 |
| 未レビュー | `status: pending` のみ |
| 承認済み | `status: approved` のみ |
| 却下済み | `status: rejected` のみ |
| 登録済み | `status: registered` のみ |

- デフォルト表示は「未レビュー」タブ。
- ナビゲーションの「コンテンツ収集」に未レビュー件数バッジを表示する。

### 3.4 絞り込み条件

| 条件 | 種別 |
|------|------|
| ジャンル | ドロップダウン（全ジャンル一覧から選択） |
| フラグ | ドロップダウン（not_gameplay / update_stopped / completed / spam / non_japanese / フラグなし） |
| タイトル・チャンネル名 | テキスト入力（部分一致） |

### 3.5 一覧カードの表示項目

一覧はカード形式で表示する。

| 表示項目 | 内容 |
|---------|------|
| サムネイル | YouTubeの再生リストサムネイル画像（16:9） |
| 再生リストタイトル | — |
| チャンネル名 | — |
| ジャンル | ゲームタイトルマスタに紐づくジャンル名 |
| 動画数 | — |
| 最終動画投稿日 | `latestVideoPublishedAt` を `YYYY-MM-DD` 形式で表示 |
| フラグバッジ | 該当フラグをバッジとして表示。フラグなしの場合は表示しない |
| AI判定コメント | `aiComment` の冒頭50文字を表示。「詳細を見る」リンクで詳細画面へ遷移 |
| 操作ボタン | 「承認する」「却下する」（未レビュー時のみ活性。承認済み・却下済み・登録済みはグレーアウト） |

#### フラグバッジの表示仕様

| フラグ | バッジラベル | 色 |
|-------|-----------|-----|
| `not_gameplay` | 実況以外の可能性 | 黄系 |
| `update_stopped` | 更新停止の可能性 | 黄系 |
| `completed` | 完結済みの可能性 | 青系 |
| `non_japanese` | 日本語以外の可能性 | 黄系 |
| `spam` | スパム疑い | 赤系 |

### 3.6 ページネーション

管理 マスタ管理仕様書.md (/document/specification/management/管理 マスタ管理仕様書.md) の共通ページネーション仕様に従う。
デフォルト表示件数は20件。

## 5. 承認・却下フロー

### 5.1 承認フロー

一覧カードまたは詳細画面の「承認する」ボタンから実行する。

```
「承認する」ボタンをクリック
    ↓
紐づけるゲームタイトルの登録状態を確認
    ├─ 登録済みの場合 → 承認ダイアログへ（下記）
    └─ 未登録の場合
           ↓
        ゲームタイトル新規登録モーダルを表示
        （管理_マスタ管理仕様書のゲームタイトル新規追加モーダルと同一仕様）
           ↓
        ゲームタイトルをマスタに登録
           ↓
        承認ダイアログへ
    ↓
承認ダイアログを表示
  ├─ 紐づけるゲームタイトル（必須・変更可）
  ├─ オーナーコメント入力欄（任意）
  └─ 「承認する」ボタン / 「キャンセル」ボタン
    ↓
登録処理を実行：
  ① チャンネルがプレミテ未登録の場合 → チャンネルレコードを新規作成
  ② 再生リストレコードをプレミテ本番DB（playlists コレクション）にゲームタイトル紐づけ済みで登録
  ③ collection_candidates のステータスを registered に更新
  ④ reviewedAt・reviewedBy・reviewComment を記録
    ↓
一覧画面に戻る（または次の候補に移動）
```

> **ゲームタイトルは必須**: 再生リストはゲームタイトルと紐づいていない状態でDBに登録されることはない。承認ダイアログの「承認する」ボタンは、ゲームタイトルが選択されている場合のみ活性になる。

#### チャンネルの自動登録

承認時、対象再生リストの `channelId` がプレミテの `channels` コレクションに存在しない場合、YouTube Data API v3（`channels.list`）でチャンネル情報を取得して自動登録する。チャンネル情報の取得に失敗した場合はエラーダイアログを表示し、承認処理を中断する。

#### ゲームタイトル未登録時の登録モーダル

ゲームタイトルがマスタに未登録の場合に表示するモーダルの仕様は、管理_マスタ管理仕様書 第7章「ゲームタイトル（サブタブ）」の新規追加モーダルと同一とする。登録完了後、自動的に承認ダイアログへ進む。

### 5.2 却下フロー

```
「却下する」ボタンをクリック
    ↓
確認ダイアログを表示
  ├─ 却下コメント入力欄（任意）
  └─ 「却下する」ボタン / 「キャンセル」ボタン
    ↓
collection_candidates のステータスを rejected に更新
reviewedAt・reviewedBy・reviewComment を記録
    ↓
一覧画面に戻る（または次の候補に移動）
```

### 5.3 却下済み候補の再審査

却下済み候補は「却下済み」タブから閲覧できる。誤って却下した場合のために、却下済み候補をオーナーが再度「承認する」ことができる。再承認時も5.1節の承認フローと同一の手順に従う。
