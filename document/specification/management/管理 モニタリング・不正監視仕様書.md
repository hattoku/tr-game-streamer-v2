## 1. 概要

本仕様書は、ゲムコメ管理画面におけるモニタリング機能および不正監視機能を定義する。

モニタリングはYouTube APIクォータの使用状況とバッチ実行ログの確認を目的とする。不正監視は異常なレビュー投稿行動の検知と対応を目的とする。

---

## 2. アクセス権限

本機能はオーナー専用である。運営者はアクセスできない。
未認証または運営者ロールの管理者（オーナー・運営者）がアクセスした場合は `/admin` へリダイレクトする。

---

## 3. モニタリング

### 3.1 YouTube APIクォータモニター

YouTube Data API v3の利用状況を確認する画面。

#### 2.1.1 データ記録

Function A・Function Bの実行完了時に、Cloud FunctionsがFirestoreの`api_quota_logs`コレクションにクォータ消費レコードを書き込む。クォータ消費ユニット数はFunction内でリクエスト数から自前で算出する。

#### 2.1.2 表示項目

| 表示項目 | 内容 |
|---------|------|
| 本日の消費ユニット数 | 当日の累計消費量と上限（10,000ユニット/日）に対する割合 |
| Function別消費内訳 | Function A（動画取得）・Function B（チャンネル更新）それぞれの当日消費量 |
| 消費推移グラフ | 過去7日間の日別消費ユニット数の推移 |

---

### 3.2 バッチ実行ログ

YouTube定期取得バッチ（Function A / Function B）の実行結果を確認する画面。

#### 2.2.1 データ記録

Function A・Function Bの実行完了時に、Cloud FunctionsがFirestoreの`batch_logs`コレクションに実行結果レコードを書き込む。

#### 2.2.2 実行履歴一覧

| 表示項目 | 内容 |
|---------|------|
| 実行日時 | バッチの実行開始日時 |
| 対象Function | Function A / Function B の別 |
| 処理件数 | 当該実行で処理した再生リスト数またはチャンネル数 |
| ステータス | 成功・スキップ・失敗の別 |

ステータスは以下の色で表示する。

| ステータス | 表示色 |
|----------|--------|
| 成功 | 緑 |
| スキップ | 黄 |
| 失敗 | 赤 |

#### 2.2.3 エラー詳細

ステータスがスキップまたは失敗のレコードを選択すると、以下の詳細を表示する。

| 表示項目 | 内容 |
|---------|------|
| エラー内容 | エラーの種別・メッセージ |
| 対象再生リスト | スキップ・失敗が発生した再生リストの識別情報 |

> **補足**: クォータ超過によるスキップが発生した場合、ステータスが黄色で表示されるためここで検知できる。

## 4. 不正監視

### 4.1 概要

異常なレビュー投稿行動を検知し、運営者が迅速に対応するための機能。熱心なファンによる組織的な水増しレビューのリスクに対応する。

### 4.2 異常検知の仕組み

#### 3.2.1 検知タイミング

レビューがFirestoreに書き込まれたタイミングで、FirestoreトリガーによりCloud Functionsが起動し、投稿ユーザーの直近の投稿数を集計して異常判定を行う。

#### 3.2.2 判定対象

一般ユーザーのみを対象とする。管理者（オーナー・運営者）・オーナーアカウントは判定から除外する。

#### 3.2.3 判定条件

| 条件 | 閾値 |
|------|------|
| 24時間以内の投稿数 | 同一ユーザーが15件以上 |

閾値はFirestoreの設定値として管理し、コード変更なしに運用調整できる。

異常と判定されたレビューおよびユーザーは`fraud_alerts`コレクションにレコードを作成する。

### 4.3 異常検知一覧

異常と判定されたユーザーの一覧を表示する画面。

| 表示項目 | 内容 |
|---------|------|
| 検知日時 | 異常判定が発生した日時 |
| ユーザー | 対象ユーザー名 |
| 24時間以内の投稿数 | 判定時点での投稿数 |
| 対応ステータス | 未対応・対応済みの別 |

### 4.4 ユーザー別投稿履歴

異常検知一覧から対象ユーザーを選択すると、そのユーザーのレビュー投稿履歴を時系列で表示する。

| 表示項目 | 内容 |
|---------|------|
| 投稿日時 | レビューの投稿日時 |
| 対象再生リスト | レビューが紐づく再生リスト名 |
| 星評価 | 投稿された星評価 |
| コメント | 投稿されたコメント内容 |

### 4.5 対応操作

ユーザー別投稿履歴画面から以下の操作を直接実行できる。対応操作（BAN・レビュー削除）はオーナーのみが実行可能とする。

| 操作 | 内容 |
|------|------|
| BAN | 対象ユーザーへのログイン禁止を設定する |
| レビュー削除 | 対象ユーザーの特定レビューを物理削除する |

## 5. 管理者（オーナー・運営者）通知

モニタリング・不正監視で検知したイベントは、Firestoreの`admin_notifications`コレクションに通知レコードを自動生成し、管理画面のお知らせ機能を通じて運営者・オーナーに通知する。

### 5.1 通知レコードの生成条件

| 通知種別 | 生成タイミング | 生成主体 |
|---------|-------------|---------|
| クォータ超過 | Function A実行時にクォータ超過によるスキップが発生した場合 | Cloud Functions（Function A） |
| 不正レビュー検知 | 異常判定条件に該当するレビューが投稿された場合 | Cloud Functions（Firestoreトリガー） |

### 5.2 重複抑制

クォータ超過通知は、当日分のレコードがすでに`admin_notifications`に存在する場合は新規生成しない。
