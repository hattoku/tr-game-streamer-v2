## 1. 概要

本仕様書は、ゲムコメにおけるお知らせ管理機能を定義する。

お知らせ機能は以下の2つの役割を持つ。

| 種別 | 対象 | 目的 |
|------|------|------|
| 管理者（オーナー・運営者）向け通知 | オーナー・運営者 | システムが自動生成する内部通知を管理者（オーナー・運営者）に届ける |
| 一般ユーザー向けお知らせ | 一般ユーザー | メンテナンス告知・キャンペーン情報等をサイト上に掲載する |

---

## 2. アクセス権限

### 2.1 管理者（オーナー・運営者）向け通知

| 操作 | オーナー | 運営者 |
|------|:---:|:---:|
| 通知の閲覧（ベルアイコン） | ○ | ○（自分宛てのもののみ） |
| 通知の既読（個別・一括） | ○ | ○（自分宛てのもののみ） |
| 通知一覧ページの閲覧 | ○ | ○（自分宛てのもののみ） |

### 2.2 一般ユーザー向けお知らせ

| 操作 | オーナー | 運営者 |
|------|:---:|:---:|
| お知らせ一覧の閲覧 | ○ | ○ |
| お知らせの新規作成 | ○ | ○ |
| お知らせの編集 | ○ | ○ |
| お知らせの削除 | ○ | ○ |

## 3. 管理者（オーナー・運営者）向け通知（ヘッダーベルアイコン）

### 3.1 概要

管理画面のヘッダーに常駐するベルアイコンを通じて、システムが自動生成した通知を管理者（オーナー・運営者）に届ける機能。

通知レコードは各機能のCloud FunctionsがFirestoreの`admin_notifications`コレクションに自動生成する。生成条件の詳細は各仕様書を参照すること。

| 通知種別 | 生成主体 | 通知対象 | 参照先 |
|---------|---------|---------|-------|
| 審査提出（Lv1） | Cloud Functions | アサインされた運営者 | 管理_審査ワークフロー仕様書 第12章 |
| 審査Lv2移行 | Cloud Functions | オーナー | 管理_審査ワークフロー仕様書 第12章 |
| クォータ超過 | Cloud Functions（Function A） | **オーナーのみ** | ゲムコメ_モニタリング_不正監視仕様書 第5章 |
| 不正レビュー検知 | Cloud Functions（Firestoreトリガー） | **オーナーのみ** | ゲムコメ_モニタリング_不正監視仕様書 第5章 |

### 3.2 ベルアイコンの表示仕様

管理画面の全画面共通ヘッダーにベルアイコンを配置する。

| 状態 | 表示 |
|------|------|
| 未読通知あり | ベルアイコン＋未読件数バッジ（赤丸）を表示 |
| 未読通知なし | ベルアイコンのみ表示（バッジなし） |

- 未読件数バッジは99件を上限とし、100件以上の場合は「99+」と表示する。
- ベルアイコンをクリックすると通知ドロップダウンを展開する。

### 3.3 通知ドロップダウン

#### 表示件数

未読通知のみを最大10件表示する。10件を超える場合は「すべて見る」リンクを表示し、通知一覧ページへ遷移する。

#### ドロップダウンの構成

```
┌────────────────────────────────────┐
│ 通知                  すべて既読にする │
├────────────────────────────────────┤
│ 🔴 不正レビュー検知                  │
│    ユーザー〇〇が異常な投稿を行いました │
│    3分前                            │
├────────────────────────────────────┤
│ 🔴 審査提出（Lv1）                   │
│    再生リスト登録提案が提出されました   │
│    1時間前                          │
├────────────────────────────────────┤
│ ...                                │
├────────────────────────────────────┤
│         すべて見る →               │
└────────────────────────────────────┘
```

#### 通知1件あたりの表示要素

| 要素 | 内容 |
|------|------|
| 未読インジケーター | 未読の場合は赤丸を表示。既読の場合は非表示 |
| 通知種別 | 通知の種別名（例：不正レビュー検知・審査提出） |
| 通知内容 | 通知の概要テキスト。最大2行、超過時は省略記号（…）で切り詰め |
| 経過時間 | 通知生成からの経過時間（例：3分前・1時間前・2日前） |

#### クリック時の動作

| 操作対象 | 動作 |
|---------|------|
| 通知1件をクリック | 個別既読にし、関連画面へ遷移する |
| 「すべて既読にする」をクリック | ドロップダウン内の全件を既読にする |
| 「すべて見る」をクリック | 通知一覧ページへ遷移する |
| ドロップダウン外をクリック | ドロップダウンを閉じる |

#### 通知種別ごとの遷移先

| 通知種別 | 遷移先 |
|---------|-------|
| 審査提出（Lv1） | 該当ワークフロー詳細画面 |
| 審査Lv2移行 | 該当ワークフロー詳細画面 |
| クォータ超過 | モニタリング画面（YouTube APIクォータモニター） |
| 不正レビュー検知 | 不正監視画面（異常検知一覧） |

### 3.4 通知一覧ページ

「すべて見る」から遷移する、管理者（オーナー・運営者）向け通知の全件一覧ページ。

#### 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ 通知一覧                          [ すべて既読にする ]        │
│                                                             │
│ [ 通知種別 ▼ ]  [ 既読状態 ▼ ]                              │
│                                                             │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ ● 不正レビュー検知  ユーザー〇〇が…  2026-03-08 14:32   │ │
│ │ ● 審査提出（Lv1）  再生リスト登録提案が…  1時間前        │ │
│ │   クォータ超過      クォータ超過が発生…  2026-03-07      │ │
│ │ ...                                                     │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                             │
│ 20件表示 ▼                    ← 1 2 3 ... →               │
└─────────────────────────────────────────────────────────────┘
```

#### 絞り込み条件

| 条件 | 種別 |
|------|------|
| 通知種別 | ドロップダウン（審査提出・審査Lv2移行・クォータ超過・不正レビュー検知） |
| 既読状態 | ドロップダウン（すべて・未読のみ・既読のみ） |

- 複数条件を指定した場合はAND条件で絞り込む。

#### 一覧テーブルの表示項目

| 列 | 内容 |
|----|------|
| 未読インジケーター | 未読の場合は●を表示 |
| 通知種別 | 通知の種別名 |
| 通知内容 | 通知の概要テキスト（1行、超過時は省略記号で切り詰め） |
| 通知日時 | 通知が生成された日時 |

- 行をクリックすると個別既読にし、関連画面へ遷移する。
- デフォルトのソート順は通知日時の降順（新しい順）。

#### ページネーション

| 項目 | 仕様 |
|------|------|
| デフォルト表示件数 | 20件 |
| 切り替え可能な表示件数 | 20 / 50 / 100件 |
| 切り替えUI | リスト右上にセレクトボックスで配置（例：「20件表示 ▼」） |
| ページ送りUI | リスト下部にページ番号リンク形式で配置 |

### 3.5 既読管理の仕様

| 操作 | 対象 | 動作 |
|------|------|------|
| 個別既読 | ドロップダウンまたは一覧の1件をクリック | 該当通知の既読フラグをONにする |
| 一括既読 | 「すべて既読にする」ボタンをクリック | 自分宛ての未読通知をすべて既読にする |

- 既読にした通知は取り消し（未読に戻す）できない。
- 既読状態は管理者（オーナー・運営者）アカウントごとに独立して管理する（Aさんが既読にしてもBさんの未読状態には影響しない）。

## 4. 一般ユーザー向けお知らせ管理

### 4.1 概要

サイト上部のバナー帯（インフォメーションバー）に掲載するお知らせを管理する機能。メンテナンス告知・キャンペーン情報等に使用する。

### 4.2 お知らせ一覧

#### 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ お知らせ管理                       [ 新規作成 ]              │
│                                                             │
│ [ ステータス ▼ ]                                             │
│                                                             │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ ステータス  タイトル          掲載期間            操作    │ │
│ │ 掲載中      メンテナンスのお知らせ  03/08〜03/09   編集 削除│ │
│ │ 掲載予定    春のキャンペーン   03/10〜03/31        編集 削除│ │
│ │ 掲載終了    旧キャンペーン     02/01〜02/28        編集 削除│ │
│ │ ...                                                     │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

#### ステータスの定義

掲載開始日時・掲載終了日時をもとに自動的に判定する。

| ステータス | 条件 |
|----------|------|
| 掲載予定 | 現在日時が掲載開始日時より前 |
| 掲載中 | 現在日時が掲載開始日時以降かつ掲載終了日時以前 |
| 掲載終了 | 現在日時が掲載終了日時より後 |

#### 絞り込み条件

| 条件 | 種別 |
|------|------|
| ステータス | ドロップダウン（すべて・掲載中・掲載予定・掲載終了） |

#### 一覧テーブルの表示項目

| 列 | 内容 |
|----|------|
| ステータス | 掲載中・掲載予定・掲載終了のバッジ表示 |
| タイトル | お知らせのタイトル |
| 掲載期間 | 掲載開始日〜掲載終了日 |
| 掲載対象ページ | 設定されている対象ページ名（複数の場合はカンマ区切り。「全ページ」の場合は「全ページ」と表示） |
| 操作 | 「編集」「削除」ボタン |

- デフォルトのソート順は掲載開始日時の降順（新しい順）。

### 4.3 お知らせの新規作成・編集

新規作成・編集は同一フォームで行う。

#### フォーム項目

| 項目 | 種別 | 必須 | 仕様 |
|------|------|:---:|------|
| タイトル | テキスト入力 | ○ | バナー帯に表示するテキスト。最大50文字 |
| 本文 | テキストエリア | × | モーダルに表示する全文。最大1,000文字 |
| 掲載開始日時 | 日時入力 | ○ | — |
| 掲載終了日時 | 日時入力 | ○ | 掲載開始日時より後の日時を指定すること |
| 掲載対象ページ | チェックボックス（複数選択） | ○ | 後述の掲載対象ページ一覧を参照 |

#### 掲載対象ページ一覧

| 選択肢 | 対象 |
|-------|------|
| 全ページ共通 | サイト内の全ページ |
| トップページ | トップページのみ |
| 再生リストを探す | 再生リストを探すページのみ |
| ゲームタイトルから探す | ゲームタイトルから探すページのみ |
| チャンネル名から探す | チャンネル名から探すページのみ |
| ゲームタイトル詳細 | ゲームタイトル詳細ページのみ |
| チャンネル詳細 | チャンネル詳細ページのみ |
| 動画プレーヤー | 動画プレーヤーページのみ |

> **補足**: 「全ページ共通」と個別ページを同時に選択した場合は「全ページ共通」が優先される。

#### バリデーション

| チェック内容 | エラーメッセージ |
|------------|--------------|
| タイトルが未入力 | 「タイトルを入力してください」 |
| 掲載開始日時が未入力 | 「掲載開始日時を入力してください」 |
| 掲載終了日時が未入力 | 「掲載終了日時を入力してください」 |
| 掲載終了日時が掲載開始日時以前 | 「掲載終了日時は掲載開始日時より後に設定してください」 |
| 掲載対象ページが未選択 | 「掲載対象ページを1つ以上選択してください」 |

#### 保存フロー

```
フォーム入力
    ↓
バリデーション
    ↓
「保存する」ボタンをクリック
    ↓
保存完了 → お知らせ一覧へ戻る
```

### 4.4 お知らせ公開時のプッシュ通知

**概要**
お知らせドキュメントのステータスが「掲載中」に変化した時点で、Firestoreトリガーにより
プッシュ通知をリアルタイムで送信する。

**送信トリガー**
Firestoreのお知らせドキュメントのステータスフィールドが「掲載中」に更新されたとき、
Cloud Functions（Firestoreトリガー）が起動して通知送信処理を実行する。
管理者が「保存する」ボタンを押して掲載開始日時が現在日時以降の場合は、
掲載開始日時を過ぎた瞬間に別途Cloud Schedulerがステータス更新をトリガーする。

**送信対象**
- FCMトークンを保持しているすべてのログイン済みユーザー
- かつ設定ページでお知らせ通知がONになっているユーザー
- 詳細はゲムコメ_FCMプッシュ通知基盤仕様書の第6.2節を参照すること

**重複送信の防止**
同一お知らせに対してプッシュ通知は1回限り送信する。
送信完了後、Firestoreのお知らせドキュメントに送信済みフラグを記録する。
Cloud Functions実行時にこのフラグを確認し、送信済みの場合は処理を終了する。

**通知ペイロード**
ゲムコメ_FCMプッシュ通知基盤仕様書の第5.2節「お知らせ通知」に準ずる。

### 4.5 お知らせの削除

| 項目 | 仕様 |
|------|------|
| 削除方式 | 物理削除 |
| 確認ダイアログ | 「このお知らせを削除しますか？」の確認ダイアログを表示する |
| 掲載中のお知らせの削除 | 可能。即時フロントへの表示が停止される |

## 5. 一般ユーザー向けお知らせの表示仕様（フロント）

### 5.1 バナー帯（インフォメーションバー）

掲載対象ページのヘッダー直下に、1行のバナー帯として表示する。

#### 表示条件

- 対象ページにアクセスした時点で、ステータスが「掲載中」のお知らせが1件以上存在する場合に表示する。
- ログイン・未ログイン問わず全ユーザーに表示する。

#### 表示内容

| 要素 | 内容 |
|------|------|
| タイトルテキスト | お知らせのタイトル。最大1行、超過時は省略記号（…）で切り詰め |
| 「詳細を見る」リンク | 本文が設定されている場合のみ表示。クリックで詳細モーダルを展開 |
| 閉じるボタン（×） | バナー帯を非表示にする |
| 件数インジケーター | 掲載中のお知らせが複数件の場合、「1/3」のようにカウンターを表示 |
| 切り替えボタン | 複数件の場合、前後に切り替える「＜」「＞」ボタンを表示 |

#### 閉じた後の挙動

ユーザーがバナー帯を閉じた場合、LocalStorageに閉じたお知らせのIDを保存し、同一端末・同一ブラウザでは再表示しない。

#### 複数件の場合の挙動

掲載中のお知らせが複数件ある場合、件数インジケーターと切り替えボタンで順番に閲覧できる。閉じるボタンで閉じた場合は、表示中の1件のみを非表示にする（他の件は引き続き表示される）。

### 5.2 詳細モーダル

バナー帯の「詳細を見る」リンクをクリックすると、お知らせの全文をモーダルで表示する。

#### 表示要素

| 要素 | 内容 |
|------|------|
| タイトル | お知らせのタイトル |
| 本文 | お知らせの全文 |
| 掲載期間 | 掲載開始日〜掲載終了日 |
| 閉じるボタン | モーダル右上の「×」ボタン |

#### モーダルの閉じ方

モーダル右上の「×」ボタン、またはモーダル外領域のクリックで閉じる。
