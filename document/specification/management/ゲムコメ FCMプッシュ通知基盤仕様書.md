## 1. 概要

本仕様書は、ゲムコメにおけるPWAプッシュ通知の基盤仕様を定義する。

通知基盤にはFirebase Cloud Messaging（FCM）を採用する。フロントエンド（Next.js + next-pwa）でFCMトークンを取得・管理し、Cloud FunctionsからFCM Admin SDK経由で通知を送信する構成をとる。

**対象ユーザー**: ログイン済みユーザーのみ。未ログインユーザーはプッシュ通知の対象外とする。

---

## 2. 通知種別と送信対象

ゲムコメで送信するプッシュ通知は以下の2種別とする。

| 通知種別 | 送信トリガー | 送信対象 |
|---------|------------|---------|
| シリーズ新着通知 | YouTube定期取得（Function A）で新着動画が検知されたとき | 該当再生リストをマイリスト登録しているユーザー |
| お知らせ通知 | 管理者がお知らせを公開したとき（掲載開始日時に達した時点） | 全ログイン済みユーザー |

### 2.1 送信対象の絞り込み条件

いずれの通知種別においても、以下の条件をすべて満たすユーザーのみを送信対象とする。

| 条件 | 内容 |
|------|------|
| ログイン済み | Firebase Authenticationでログイン中であること |
| 通知許可済み | ブラウザの通知許可状態が `granted` であること（FCMトークンを保持していること） |
| 通知種別ON | 設定ページの通知種別トグルが ON であること |

通知種別トグルがOFFのユーザー、またはFCMトークンを持たないユーザーはスキップする。

## 3. FCMトークンのライフサイクル

### 3.1 トークン取得タイミング

FCMトークンは、ユーザーがブラウザの通知許可ダイアログで「許可」を選択した直後に取得する。ページ訪問のたびに毎回取得するのではなく、許可付与イベントを契機として1度だけ取得・保存する。

### 3.2 トークンの保存方式

取得したFCMトークンは、FirestoreのユーザードキュメントにFirebaseのUIDをキーとして保存する。

| 保存先 | フィールド名 | 型 | 内容 |
|--------|------------|-----|------|
| Firestoreユーザードキュメント | `fcmTokens` | 配列（文字列） | FCMトークンの一覧 |

トークンは端末・ブラウザごとに発行される。1ユーザーが複数のデバイスやブラウザでゲムコメを利用する場合、`fcmTokens` 配列に複数のトークンを保持する。

### 3.3 トークンの更新

FCMトークンはブラウザ側で自動的に更新される場合がある。アプリ起動時（ログイン済みかつ通知許可済みの状態でのページロード時）にトークンの最新状態を確認し、Firestoreに保存済みのトークンと差異がある場合は更新する。

### 3.4 トークンの削除（無効トークンの処理）

Cloud FunctionsからFCM送信を行った際に `messaging/registration-token-not-registered` エラーが返却された場合、そのトークンは無効と判断し、Firestoreの `fcmTokens` 配列から該当トークンを削除する。

### 3.5 複数デバイス対応

1ユーザーの `fcmTokens` 配列に複数のトークンが存在する場合、通知送信時にすべてのトークンへ送信する。一部のトークンが無効であっても、他のトークンへの送信は継続する。

## 4. 通知許可の取得フロー

### 4.1 許可要求の導線

通知許可の要求は以下の2つの導線から行う。

#### 導線①：設定ページからの能動的許可

設定ページ（`/settings`）の「通知設定」セクションに「通知を許可する」ボタンを設置する。ユーザーがこのボタンをクリックした時点でブラウザの許可ダイアログを表示する。

詳細UIは /document/specification/page/ページ 設定ページ仕様書.md の「通知設定」セクションを参照すること。

#### 導線②：初回マイリスト登録時のバナー案内

ログイン済みユーザーがはじめてマイリストに再生リストを登録したとき、画面上部にバナーを表示し設定ページの通知設定セクションへ誘導する。バナーからは直接許可ダイアログを出さない。

**バナーの表示条件**

- ログイン済みユーザーであること
- マイリスト登録が今回初めてであること（それ以前のマイリスト登録が0件）
- 通知許可状態が `default`（まだ一度も許可要求をしていない状態）であること

**バナーの表示仕様**

| 項目 | 仕様 |
|------|------|
| 表示位置 | 画面上部（トースト通知と同様の位置） |
| 表示内容 | 「🔔 新着動画をプッシュ通知で受け取れます。[通知を設定する →]」 |
| 「通知を設定する」リンク | 設定ページ（`/settings`）の通知設定セクションへ遷移（`/settings#notification`）|
| 自動消去 | 表示から8秒後に自動で消える |
| 手動消去 | バナー上の「×」ボタンで即時消去 |
| 再表示 | 消去後は再表示しない（LocalStorageに表示済みフラグを保存） |

### 4.2 許可状態の種別と挙動

ブラウザの通知許可状態は `Notification.permission` APIで取得する3つの状態に応じて、設定ページの表示と動作を切り替える。

| 状態 | 意味 | 設定ページの表示 | アクション |
|------|------|----------------|----------|
| `default` | まだ許可を求めていない | 「通知を許可する」ボタンを表示 | ボタンクリックでブラウザのダイアログを表示 |
| `granted` | ブラウザで通知を許可している | 許可済みバッジ＋通知種別トグルを表示 | トグルでON/OFFを切り替え可能 |
| `denied` | ブラウザで通知を拒否している | 拒否中バッジ＋ブラウザ設定変更の案内テキストを表示 | アプリ側からの再要求は不可 |

`denied` 状態では、アプリ側からダイアログを再表示する手段がない。そのため「ブラウザの設定から通知をONにしてください」という案内テキストを表示するにとどめる。ボタンは設置しない。

### 4.3 「通知を許可する」ボタンのフロー

```
「通知を許可する」ボタンをクリック
    ↓
ブラウザの許可ダイアログを表示
    ↓
    ├─ 許可（granted）
    │       ↓
    │   FCMトークンを取得
    │       ↓
    │   Firestoreのfcmトークン配列に保存
    │       ↓
    │   設定ページの表示を「許可済み」状態に切り替える
    │   通知種別トグルを表示する
    │
    └─ 拒否（denied）
            ↓
        設定ページの表示を「拒否中」状態に切り替える
        ブラウザ設定変更の案内テキストを表示する
```

### 4.4 Service Workerとの連携

FCMのバックグラウンド通知受信にはService Workerが必要である。next-pwaが管理するService Workerに加え、FCM専用のService Workerファイル（`firebase-messaging-sw.js`）を `/public` 配下に配置する。

`firebase-messaging-sw.js` はFCMのバックグラウンドメッセージハンドラを実装し、アプリが非表示・ブラウザが閉じた状態でも通知を受信・表示できるようにする。

## 5. 通知ペイロード設計

### 5.1 シリーズ新着通知

| フィールド | 内容 | 例 |
|----------|------|-----|
| `title` | 再生リストのタイトル | 「【ドラゴンクエストXI実況】に新着動画が追加されました」 |
| `body` | 新着動画数のメッセージ | 「新しい動画が2件追加されました」 |
| `icon` | ゲムコメのアイコン画像URL | `/icons/icon-192x192.png` |
| `click_action` | クリック時の遷移先 | `/playlists/[playlistId]` |

新着動画が複数本ある場合（1バッチで複数本検知）、1通知にまとめて送信する。本文には「X件追加されました」と本数を明記する。

### 5.2 お知らせ通知

| フィールド | 内容 | 例 |
|----------|------|-----|
| `title` | 固定テキスト | 「【ゲムコメ】お知らせ」 |
| `body` | お知らせのタイトル文字列 | 「メンテナンスのお知らせ（3/20 2:00〜4:00）」 |
| `icon` | ゲムコメのアイコン画像URL | `/icons/icon-192x192.png` |
| `click_action` | TOPページ | `/` |

## 6. Cloud Functionsによる送信処理

### 6.1 シリーズ新着通知の送信フロー

YouTube定期取得仕様書のFunction Aで新着動画が検知された再生リストを対象に、以下の処理を非同期で実行する。Function Aの完了を待たず、新着検知のイベントをトリガーとして独立した処理として実行する。

```
Function Aで新着動画を検知（再生リスト単位）
    ↓
該当再生リストのplaylistIdをキーにFirestoreを参照し、
マイリスト登録しているユーザーの一覧を取得
    ↓
各ユーザーのFirestoreドキュメントから fcmTokens を取得
    ↓
    ├─ fcmTokensが空 / シリーズ新着通知がOFF → スキップ
    └─ 送信対象として処理を続行
            ↓
        FCM Admin SDK で通知を送信（第5.1節のペイロード）
            ↓
        送信結果を確認
            ├─ 成功 → 処理完了
            └─ 無効トークンエラー → Firestoreから該当トークンを削除
```

**Function Aとの関係**

通知送信処理はFunction Aの本体処理（Firestoreへの動画情報更新）とは非同期で実行する。通知送信の成否はFunction Aの成否に影響しない。

### 6.2 お知らせ通知の送信フロー

管理者がお知らせを公開した際（Firestoreのお知らせドキュメントのステータスが「掲載中」に変化した時点）、Firestoreトリガーによって以下の処理をリアルタイムで実行する。

```
Firestoreのお知らせドキュメントが更新される
（ステータスが「掲載中」に変化）
    ↓
送信済みフラグを確認
    ├─ 送信済み → 処理終了（重複送信防止）
    └─ 未送信 → 続行
            ↓
        FCMトークンを持つ全ログイン済みユーザーを取得
        （FirestoreのusersコレクションからfcmTokensが
          空でないユーザーを取得）
            ↓
        各ユーザーのお知らせ通知設定を確認
            ├─ OFF → スキップ
            └─ ON → FCM Admin SDKで送信（第5.2節のペイロード）
            ↓
        Firestoreのお知らせドキュメントに送信済みフラグを記録
```

**重複送信の防止**

同一お知らせドキュメントへの送信は1回限りとする。Firestoreのお知らせドキュメントに `pushNotificationSentAt` フィールドを設け、送信完了後にタイムスタンプを記録する。Cloud Functions実行時にこのフィールドを確認し、値が存在する場合は処理を終了する。

### 6.3 送信失敗時のハンドリング

| エラー種別 | 対処 |
|----------|------|
| `messaging/registration-token-not-registered` | Firestoreの `fcmTokens` 配列から該当トークンを削除する |
| `messaging/invalid-argument` | エラーログをCloud Loggingに記録する。処理は継続する |
| その他のFCMエラー | エラーログをCloud Loggingに記録する。処理は継続する |
| ユーザー数が多い場合のバッチ処理 | FCM Admin SDKのマルチキャスト送信（`sendEachForMulticast`）を使用し、一度に送信できる上限（500件）を超える場合は分割して送信する |

## 7. 通知のクリック動作

### 7.1 フォアグラウンド時（アプリ表示中）

アプリがブラウザで表示中の状態で通知が届いた場合、ブラウザのシステム通知は表示しない（FCMのデフォルト動作）。アプリ内での通知表示方法は別途検討する（例：トースト通知による案内）。

### 7.2 バックグラウンド時（アプリ非表示・ブラウザ閉じた状態）

Service Worker（`firebase-messaging-sw.js`）がバックグラウンドで通知を受信し、OS・ブラウザのシステム通知として表示する。

ユーザーが通知をクリックすると、ペイロードの `click_action` に指定したURLへ遷移する。

| 通知種別 | クリック時の遷移先 |
|---------|----------------|
| シリーズ新着通知 | 該当再生リストの再生リスト詳細ページ（`/playlists/[playlistId]`） |
| お知らせ通知 | TOPページ（`/`） |

ゲムコメが既にブラウザで開かれている場合は該当タブをフォーカスし、開かれていない場合は新しいタブで開く。
