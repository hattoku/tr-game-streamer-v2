## 1. 概要

本仕様書は、ゲムコメ管理画面のトップページ（ダッシュボード）の仕様を定義する。

ダッシュボードは管理者（オーナー・運営者）がログイン直後に最初に目にするページであり、サービス運営上の要対応事項を素早く把握し、各管理機能へのナビゲーションを提供する。

### 1.1 ページURL・基本情報

|項目     |内容                                       |
|-------|-----------------------------------------|
|URL    |`/admin`                                 |
|ページタイトル|ダッシュボード                                  |
|アクセス条件 |オーナー・運営者のみ。未認証の場合は `/admin/login` へリダイレクト|
|レイアウト  |共通UIコンポーネント（管理画面）仕様書の3ペイン構造に従う           |

### 1.2 流入元

ダッシュボードへの流入経路は以下の通り。

|流入元               |条件                             |挙動                      |
|------------------|-------------------------------|------------------------|
|ログインフォーム（`/login`）|管理者（オーナー・運営者）アカウント（オーナー・運営者）でログイン成功時     |ログイン完了後、`/admin` へリダイレクト|
|管理画面サイドバー         |ログイン済みの管理者（オーナー・運営者）がサイドバーの「ダッシュボード」をクリック|`/admin` へ遷移            |
|直接URLアクセス         |ログイン済みの管理者（オーナー・運営者）が `/admin` に直接アクセス   |そのまま表示                  |
|直接URLアクセス         |未認証の場合                         |`/login` へリダイレクト        |


> ログインフォームは一般ユーザーと管理者（オーナー・運営者）で共通（`/login`）。ログイン成功後のリダイレクト先はFirebase Authで付与されたロールによって分岐する。一般ユーザーはTOPページ（`/`）へ、管理者（オーナー・運営者）はダッシュボード（`/admin`）へ遷移する。

### 1.3 設計方針

ダッシュボードは「今すぐ対応が必要なことをひとつの画面で把握できる」ことを最優先に設計する。詳細データの閲覧・編集は各機能ページへ誘導し、ダッシュボード自体は情報の集約・通知に徹する。

## 4. 要対応アラートセクション

### 4.1 概要

管理者（オーナー・運営者）がすぐに対応すべき事象を一覧表示するセクション。件数が0件の場合は「対応が必要な事項はありません」のメッセージを表示し、セクション自体は非表示にしない。

### 4.2 表示条件

以下の条件に合致する事象をアラートカードとして表示する。

|アラート種別    |表示条件                                                                             |遷移先                               |表示対象      |
|----------|---------------------------------------------------------------------------------|----------------------------------|----------|
|審査待ち（Lv1） |自分がLv1審査者にアサインされているワークフローのうち、ステータスが「審査中 Lv1」のもの。オーナーは全件、運営者は自分にアサインされた件数のみカウント|`/admin/workflows?tab=pending_lv1`|オーナー・運営者  |
|審査待ち（Lv2） |ステータスが「審査中 Lv2」のワークフロー                                                           |`/admin/workflows?tab=pending_lv2`|**オーナーのみ**|
|未対応のお問い合わせ|ステータスが「未対応」のお問い合わせ                                                               |`/admin/inquiries?status=open`    |オーナー・運営者  |
|不正レビュー検知  |`fraud_alerts` コレクションで対応ステータスが「未対応」のアラート                                         |`/admin/monitoring`               |**オーナーのみ**|
|クォータ超過    |本日分のクォータ超過フラグが立っている場合                                                            |`/admin/monitoring`               |**オーナーのみ**|

### 4.3 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ 要対応事項                                                    │
│                                                             │
│ ┌──────────────────────┐  ┌──────────────────────┐          │
│ │ ⚠️ 審査待ち（Lv1）    │  │ ⚠️ 未対応のお問い合わせ │          │
│ │                      │  │                       │          │
│ │  3 件                │  │  5 件                 │          │
│ │                      │  │                       │          │
│ │  確認する →          │  │  確認する →            │          │
│ └──────────────────────┘  └──────────────────────┘          │
│                                                             │
│ ┌──────────────────────┐                                    │
│ │ 🚨 不正レビュー検知   │                                    │
│ │                      │                                    │
│ │  1 件（未対応）       │                                    │
│ │                      │                                    │
│ │  確認する →          │                                    │
│ └──────────────────────┘                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**件数が0件の場合（すべてのアラート種別が0件のとき）:**

```
┌─────────────────────────────────────────────────────────────┐
│ 要対応事項                                                    │
│                                                             │
│  ✅ 対応が必要な事項はありません                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.4 アラートカードの仕様

|要素         |内容                                       |
|-----------|-----------------------------------------|
|アイコン       |アラート種別を示すアイコン（⚠️：審査・問い合わせ / 🚨：不正検知・クォータ超過）|
|アラート種別名    |アラートの種別名                                 |
|件数         |対応が必要な件数（大きめの数字で表示）                      |
|「確認する →」リンク|クリックで該当ページへ遷移                            |

- アラートカードは横並び（グリッド）で表示する。画面幅に応じて折り返す。
- 件数が0件のアラート種別はカードを表示しない。
- クォータ超過は件数ではなく「発生中」と表示する。

## 5. 審査ワークフローサマリーセクション

### 5.1 概要

審査ワークフローの現在の状況をステータス別の件数で表示する。詳細は審査ワークフロー一覧ページへ誘導する。

### 5.2 画面構成

```
┌──────────────────────────────────────────────┐
│ 審査ワークフロー                  一覧を見る → │
│                                              │
│  ┌─────────────┐  ┌─────────────┐            │
│  │ 審査中 Lv1  │  │ 審査中 Lv2  │            │
│  │    8 件     │  │    2 件     │            │
│  └─────────────┘  └─────────────┘            │
│                                              │
│  今日の新着提案・通報:  3 件                   │
│                                              │
└──────────────────────────────────────────────┘
```

### 5.3 表示項目

|項目          |オーナー表示内容               |運営者表示内容                             |
|------------|-----------------------|------------------------------------|
|審査中 Lv1 件数  |ステータスが「審査中 Lv1」の**全件**数|自分がLv1審査者にアサインされている「審査中 Lv1」の件数のみ|
|審査中 Lv2 件数  |ステータスが「審査中 Lv2」の全件数    |**非表示**（運営者はLv2に関与しないため）            |
|今日の新着       |本日提出されたワークフロー件数（全件）    |本日提出されたワークフロー件数（全件）                 |
|「一覧を見る →」リンク|`/admin/workflows` へ遷移 |`/admin/workflows` へ遷移              |


> 運営者の「審査中 Lv1」件数は、自分にアサインされた案件のみを対象とする。担当ジャンル外の案件は操作権限がないためカウントしない。

## 7. モニタリングサマリーセクション

### 7.1 概要

YouTube APIクォータの当日使用状況と、不正検知の最新状況を表示する。**オーナーのみに表示する。運営者には表示しない。**

詳細はモニタリング・不正監視ページへ誘導する。

### 7.2 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ モニタリング                              詳細を見る →        │
│                                                             │
│  ┌──────────────────────────────────┐  ┌─────────────────┐  │
│  │ YouTube APIクォータ（本日）       │  │ 不正レビュー検知  │  │
│  │                                  │  │                 │  │
│  │  2,340 / 10,000 ユニット          │  │  未対応: 1 件    │  │
│  │  ██████░░░░░░░░░░░░░░  23%       │  │                 │  │
│  │                                  │  │                 │  │
│  │  最終バッチ: 成功（22:00実行）    │  │                 │  │
│  └──────────────────────────────────┘  └─────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 7.3 YouTube APIクォータ表示項目

|項目       |内容                                |
|---------|----------------------------------|
|当日消費ユニット数|当日の累計消費量（数値）                      |
|上限       |10,000ユニット/日（固定表示）                |
|プログレスバー  |消費割合を視覚的に表示                       |
|使用率（%）   |消費量 ÷ 10,000 × 100（小数点以下切り捨て）     |
|最終バッチ実行状況|直近のFunction A実行結果（成功・スキップ・失敗）と実行時刻|

**プログレスバーの色分け:**

|使用率     |色|
|--------|-|
|0〜79%   |緑|
|80〜99%  |黄|
|100%（超過）|赤|

### 7.4 不正レビュー検知表示項目

|項目          |内容                                    |
|------------|--------------------------------------|
|未対応件数       |`fraud_alerts` コレクションで対応ステータスが「未対応」の件数|
|「詳細を見る →」リンク|クリックで `/admin/monitoring` へ遷移         |

- 未対応が0件の場合は「未対応: なし」と表示する。
- 未対応件数が1件以上の場合は件数を赤色テキストで表示する。

## 8. クイックリンクセクション

### 8.1 概要

各管理機能への直接リンクを並べたショートカットセクション。サイドバーとの重複にはなるが、ダッシュボード上で主要機能へのアクセスを補完する目的で設置する。

### 8.2 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ クイックリンク                                                │
│                                                             │
│  [審査ワークフロー]  [マスタ管理]  [ユーザー管理]            │
│  [お知らせ管理]    [お問い合わせ管理]  [操作ログ]            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 8.3 表示項目

|リンク名       |遷移先URL             |表示権限      |
|-----------|-------------------|----------|
|審査ワークフロー   |`/admin/workflows` |オーナー・運営者  |
|マスタ管理      |`/admin/master`    |オーナー・運営者  |
|ユーザー管理     |`/admin/users`     |オーナー・運営者  |
|管理者（オーナー・運営者）アカウント管理 |`/admin/operators` |オーナーのみ    |
|お知らせ管理     |`/admin/notices`   |オーナー・運営者  |
|お問い合わせ管理   |`/admin/inquiries` |オーナー・運営者  |
|モニタリング・不正監視|`/admin/monitoring`|**オーナーのみ**|
|操作ログ       |`/admin/logs`      |オーナー・運営者  |

- 「管理者（オーナー・運営者）アカウント管理」はオーナーのみに表示する。
- ボタン形式（テキスト＋枠）で横並びに配置する。
