## 1. 概要

本仕様書は、ゲムコメ管理画面における操作ログ機能の閲覧画面を定義する。

操作ログは、管理者（オーナー・運営者）が管理画面上で実行した操作を自動的に記録する機能である。不正操作の追跡および運営上のトレーサビリティ確保を目的とする。

各管理機能における記録対象操作の定義は、それぞれの機能仕様書を参照すること。

| 参照先 | 記録対象の定義箇所 |
|--------|-----------------|
| /document/specification/management/管理 ユーザー管理仕様書.md | 第5章「操作ログ」 |
| /document/specification/management/管理 マスタ管理仕様書.md | 第9章「操作ログ」 |

> **補足**: 操作ログは管理者（オーナー・運営者）による編集を不可とする。削除は本仕様書第5章に定める手順に限り、オーナーのみ実行できる。

## 3. 操作ログ一覧

### 3.1 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ 操作ログ                                                      │
│                                                             │
│ [ 操作者 ▼ ]  [ 操作種別 ▼ ]  [ 日付範囲: 開始 〜 終了 ]        │
│                                                             │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 操作日時         操作者    操作種別       対象データ        │ │
│ │ 2026-03-08 14:32 田中      BAN設定        user_abc        │ │
│ │ 2026-03-08 13:10 鈴木      ゲームタイトル編集 ゼルダの伝説... │ │
│ │ ...                                                     │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                             │
│ 20件表示 ▼                    ← 1 2 3 ... →               │
└─────────────────────────────────────────────────────────────┐
```

### 3.2 絞り込み条件

| 条件 | 種別 |
|------|------|
| 操作者 | ドロップダウン（管理者（オーナー・運営者）アカウント一覧から選択） |
| 操作種別 | ドロップダウン（記録対象の操作種別一覧から選択） |
| 日付範囲 | 日付入力（開始日・終了日） |

- 絞り込み条件はすべて任意。未指定の場合は全件を対象とする。
- 複数の条件を指定した場合はAND条件で絞り込む。

### 3.3 一覧テーブルの表示項目

| 列 | 内容 |
|----|------|
| 操作日時 | 操作が実行された日時 |
| 操作者 | 操作を行った管理者（オーナー・運営者）名 |
| 操作種別 | 実行された操作の種別（例：BAN設定、ゲームタイトル削除） |
| 対象データ | 操作の対象となったデータの名称またはID |

- 行をクリックすると操作ログ詳細モーダルを表示する。
- デフォルトのソート順は操作日時の降順（新しい順）。

### 3.4 ページネーション

| 項目 | 仕様 |
|------|------|
| デフォルト表示件数 | 20件 |
| 切り替え可能な表示件数 | 20 / 50 / 100件 |
| 切り替えUI | リスト右上にセレクトボックスで配置（例：「20件表示 ▼」） |
| ページ送りUI | リスト下部にページ番号リンク形式で配置 |

## 4. 操作ログ詳細モーダル

一覧の行をクリックすると、該当ログの詳細をモーダルで表示する。

### 4.1 表示項目

| 項目 | 内容 |
|------|------|
| 操作日時 | 操作が実行された日時 |
| 操作者 | 操作を行った管理者（オーナー・運営者）名 |
| 操作種別 | 実行された操作の種別 |
| 対象データ | 操作の対象となったデータの名称またはID |
| 変更詳細 | 変更前後の値・入力された理由など、操作種別に応じた詳細情報 |

### 4.2 変更詳細の表示例

操作種別によって変更詳細の内容は異なる。以下に代表的な例を示す。

| 操作種別 | 変更詳細の内容 |
|----------|-------------|
| BAN設定 | BAN理由（入力された場合） |
| 強制退会 | 退会理由（入力された場合） |
| ゲームタイトル編集 | 変更前の値・変更後の値（項目ごと） |
| 担当ジャンル変更 | 変更前のジャンル・変更後のジャンル |
| 操作ログ削除 | 削除対象の日付範囲 |

> 変更詳細に記録される具体的な項目は、各機能仕様書の操作ログ章（/document/specification/management/管理 ユーザー管理仕様書.md 第5章・/document/specification/management/管理 マスタ管理仕様書.md 第9章）を参照すること。

### 4.3 モーダルの閉じ方

モーダル右上の「×」ボタン、またはモーダル外領域のクリックで閉じる。

## 5. ログの削除

### 5.1 概要

オーナーのみが、日付範囲を指定して操作ログを一括削除できる。削除操作自体も操作ログとして記録される。

### 5.2 削除フロー

```
一覧画面の「ログを削除する」ボタンをクリック（オーナーのみ表示）
    ↓
削除対象の日付範囲を入力
    ├─ 開始日（必須）
    └─ 終了日（必須）
    ↓
確認ダイアログ表示
    ├─ 削除対象期間の説明
    ├─ 対象件数の表示
    └─ 「削除する」ボタン / 「キャンセル」ボタン
    ↓
削除完了 → 一覧に反映
```

### 5.3 削除操作のログ記録

削除が実行された時点で、以下の内容を操作ログとして記録する。

| 項目 | 記録内容 |
|------|---------|
| 操作日時 | 削除を実行した日時 |
| 操作者 | 削除を実行したオーナー名 |
| 操作種別 | 操作ログ削除 |
| 変更詳細 | 削除対象の日付範囲・削除件数 |

> 削除操作のログ自体は削除対象に含まれない。
