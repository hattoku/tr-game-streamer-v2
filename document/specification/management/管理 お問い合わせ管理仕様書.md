## 1. 概要

本仕様書は、ゲムコメにおけるお問い合わせ機能を定義する。

ユーザーがフロントのお問い合わせフォームから送信した内容を、管理画面で一元管理する。ログイン済みユーザーに対してはスレッド形式のやり取りを提供し、管理者（オーナー・運営者）とユーザー間でメッセージのやり取りができる。未ログインユーザーに対してはメール折り返しのみとする。

## 3. お問い合わせフォーム（フロント）

### 3.1 設置場所・導線

| 導線 | 内容 |
|------|------|
| フッター | 「お問い合わせ」リンクを常時表示。ログイン状態を問わず全ユーザーがアクセス可能 |
| 設定ページ（`/settings`） | ページ末尾に「お問い合わせはこちら」リンクを設置し `/contact` へ遷移 |

| 項目 | 内容 |
|------|------|
| URL | `/contact` |
| アクセス条件 | ログイン・未ログイン問わず全ユーザーがアクセス可能 |

### 3.2 フォーム項目

| 項目 | 種別 | 必須 | ログイン済み | 未ログイン |
|------|------|:---:|:---:|:---:|
| お問い合わせ種別 | セレクトボックス | ○ | 入力 | 入力 |
| お問い合わせ内容 | テキストエリア | ○ | 入力 | 入力 |
| 関連URL | テキスト入力（最大5件） | × | 入力 | 入力 |
| メールアドレス | テキスト入力 | ○ | 自動入力（編集不可） | 入力 |

> **メールアドレスの自動入力について**: ログイン済みユーザーはアカウントに登録されているメールアドレスを自動入力し、編集不可とする。返信先を確実に確保するための仕様。

### 3.3 お問い合わせ種別

| 種別 | 説明（フォーム内の補足テキスト） |
|------|------|
| 不具合・バグ報告 | サービスの動作に関する不具合・バグのご報告 |
| アカウントについて | ログイン・退会・アカウント情報に関するお問い合わせ |
| 掲載内容について | 誤情報・権利侵害・不適切なコンテンツ等に関するご連絡 |
| ご意見・ご提案 | サービスへのご意見・新機能のご提案 |
| その他 | 上記に該当しないお問い合わせ |

### 3.4 関連URLの入力仕様

| 項目 | 仕様 |
|------|------|
| 最大件数 | 5件 |
| 追加方法 | 「URLを追加する」ボタンをクリックするとフィールドを1件追加する |
| 削除方法 | 各フィールド右端の「×」ボタンで該当フィールドを削除する |
| 初期表示 | 1件のみ表示。未入力のまま送信した場合は無視する |
| バリデーション | 入力がある場合は `http://` または `https://` から始まるURL形式であること |

### 3.5 バリデーション

| チェック内容 | エラーメッセージ |
|------------|--------------|
| 種別が未選択 | 「お問い合わせ種別を選択してください」 |
| 内容が未入力 | 「お問い合わせ内容を入力してください」 |
| 内容が文字数上限超過（2,000文字） | 「2,000文字以内で入力してください」 |
| メールアドレスが未入力（未ログイン時） | 「メールアドレスを入力してください」 |
| メールアドレスの形式が不正（未ログイン時） | 「正しいメールアドレスの形式で入力してください」 |
| 関連URLの形式が不正 | 「正しいURL形式で入力してください」 |

### 3.6 送信フロー

```
フォーム入力
    ↓
バリデーション
    ↓
「送信する」ボタンをクリック
    ↓
送信完了画面を表示
    ├─ ログイン済み：「お問い合わせを受け付けました。
    │                マイページのお問い合わせ履歴からやり取りを確認できます。」
    └─ 未ログイン ：「お問い合わせを受け付けました。
                    ご入力のメールアドレスに折り返しご連絡いたします。」
```

### 3.7 画面レイアウト

```
┌─────────────────────────────────────────────────┐
│ ヘッダー                                         │
├─────────────────────────────────────────────────┤
│                                                 │
│  お問い合わせ                                    │
│                                                 │
│  お問い合わせ種別 *                              │
│  [ 選択してください             ▼ ]              │
│  （種別に応じた補足テキスト）                     │
│                                                 │
│  お問い合わせ内容 *                              │
│  [ テキストエリア                  ]             │
│                                  0 / 2000       │
│                                                 │
│  関連URL（任意）                                 │
│  [ https://                    ] [ × ]          │
│  [ URLを追加する + ]                             │
│                                                 │
│  メールアドレス *                                │
│  [ example@email.com           ]                │
│  ※ログイン済みの場合は自動入力・編集不可           │
│                                                 │
│  [ 送信する ]                                    │
│                                                 │
└─────────────────────────────────────────────────┘
```

## 4. お問い合わせ一覧（管理画面）

### 4.1 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ お問い合わせ管理                                              │
│                                                             │
│ [ 種別 ▼ ]  [ ステータス ▼ ]  [ 日付範囲: 開始 〜 終了 ]      │
│ [ キーワード検索                                    ]        │
│                                                             │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ 受付日時          種別           ユーザー  ステータス     │ │
│ │ 2026-03-08 14:32  不具合・バグ報告  田中     未対応        │ │
│ │ 2026-03-08 12:10  アカウントについて  —      対応済み      │ │
│ │ ...                                                     │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                             │
│ 20件表示 ▼                    ← 1 2 3 ... →               │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 絞り込み条件

| 条件 | 種別 |
|------|------|
| お問い合わせ種別 | ドロップダウン（全種別・不具合・バグ報告・アカウントについて・掲載内容について・ご意見・ご提案・その他） |
| ステータス | ドロップダウン（すべて・未対応・対応中・保留中・対応済み） |
| 受付日時 | 日付範囲（開始日・終了日） |
| キーワード | テキスト検索（お問い合わせ内容・メールアドレスを対象） |

- 複数条件を指定した場合はAND条件で絞り込む。

### 4.3 一覧テーブルの表示項目

| 列 | 内容 |
|----|------|
| 受付日時 | お問い合わせが送信された日時 |
| 種別 | お問い合わせ種別 |
| ユーザー | 送信者のユーザー名。未ログインユーザーの場合は「—」を表示 |
| ステータス | 未対応・対応中・保留中・対応済みのバッジ表示 |

- 行をクリックするとお問い合わせ詳細画面へ遷移する。
- デフォルトのソート順は受付日時の降順（新しい順）。

### 4.4 ステータスの定義

| ステータス | 意味 |
|----------|------|
| 未対応 | 受付済みだが誰も対応を開始していない |
| 対応中 | 担当者がスレッドに返信・調査中 |
| 保留中 | 確認待ち・調査中など、すぐに返信できない状態 |
| 対応済み | スレッドがクローズされた状態 |

### 4.5 ページネーション

| 項目 | 仕様 |
|------|------|
| デフォルト表示件数 | 20件 |
| 切り替え可能な表示件数 | 20 / 50 / 100件 |
| 切り替えUI | リスト右上にセレクトボックスで配置 |
| ページ送りUI | リスト下部にページ番号リンク形式で配置 |

### 5.1 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ お問い合わせ詳細                                              │
│                                                             │
│ ┌─────────────────────────────────────────────────────────┐ │

│ │ 種別      不具合・バグ報告                                │ │
│ │ ユーザー   田中（user_abc）                               │ │
│ │ メール    example@email.com                              │ │
│ │ ステータス [ 未対応 ▼ ]                                   │ │
│ └─────────────────────────────────────────────────────────┘ │
│                                                             │
│ ── お問い合わせ内容 ──────────────────────────────────────── │
│ 動画プレーヤーで○○をすると画面が固まります。                  │
│                                                             │
│ 関連URL                                                     │
│ https://gamecome.jp/player/xxxxx                            │
│                                                             │
│ ── 対応メモ（内部共有） ──────────────────────────────────── │
│ [ テキストエリア                                  ]          │
│                                          [ 保存する ]       │
│                                                             │
│ ── チャットスレッド ─────────────────────────────────────── │
│                                                             │
│  田中 2026-03-08 14:32                                      │
│  動画プレーヤーで○○をすると画面が固まります。               │
│                                                             │
│                      運営 2026-03-08 15:00                  │
│                      ご報告ありがとうございます。確認します。 │
│                                                             │
│ [ メッセージを入力                              ]            │
│                                        [ 送信 ] [ クローズ ] │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 お問い合わせ情報

| 項目 | 内容 |
|------|------|
| 受付日時 | お問い合わせが送信された日時 |
| 種別 | お問い合わせ種別 |
| ユーザー | ユーザー名とユーザーID。未ログインユーザーの場合は「—」を表示 |
| メールアドレス | 返信先メールアドレス |
| ステータス | ドロップダウンで直接変更可能 |
| 関連URL | 送信された関連URL（最大5件）。クリックで別タブ遷移 |

### 5.3 ステータスの変更

ステータスはお問い合わせ情報エリアのドロップダウンから直接変更できる。変更は即時保存される。

| 遷移 | 条件 |
|------|------|
| 未対応 → 対応中・保留中・対応済み | 自由に変更可 |
| 対応済み → 未対応・対応中・保留中 | 自由に変更可（再オープンに相当） |

> **補足**: スレッドのクローズボタン押下時はステータスが自動的に「対応済み」に変更される。ユーザーが再オープンした場合はステータスが自動的に「未対応」に変更される。

### 5.4 対応メモ

管理者（オーナー・運営者）間での内部共有メモ。一般ユーザーには表示されない。

| 項目 | 仕様 |
|------|------|
| 編集権限 | オーナー・運営者ともに編集可 |
| 共有範囲 | 管理者（オーナー・運営者）全員で共有。一般ユーザーには非表示 |
| 保存方式 | 「保存する」ボタンをクリックして上書き保存 |
| 文字数制限 | 最大1,000文字 |

### 5.5 チャットスレッド

ログイン済みユーザーからのお問い合わせのみ表示される。未ログインユーザーからのお問い合わせはスレッドが存在せず、メール折り返しで対応する。

#### メッセージの表示仕様

| 要素 | 内容 |
|------|------|
| 送信者名 | ユーザー名または管理者（オーナー・運営者）名（「運営」と表示） |
| 送信日時 | メッセージの送信日時 |
| メッセージ本文 | テキスト内容 |

- ユーザーのメッセージは左寄せ、管理者（オーナー・運営者）のメッセージは右寄せで表示する。
- メッセージは送信日時の昇順（古い順）で表示する。

#### 返信入力エリア

| 項目 | 仕様 |
|------|------|
| 入力形式 | テキストエリア |
| 文字数制限 | 最大2,000文字 |
| 送信ボタン | 「送信」。クリックでメッセージを送信する |
| クローズボタン | 「クローズ」。クリックで確認ダイアログを表示後、スレッドをクローズする |
| クローズ後の表示 | 入力エリアおよびクローズボタンを非表示にする。「再オープンする」ボタンを表示する |

#### クローズフロー

```
「クローズ」ボタンをクリック
    ↓
確認ダイアログ表示
    └─ 「クローズする」ボタン / 「キャンセル」ボタン
    ↓
スレッドをクローズ
ステータスを「対応済み」に自動変更
```

#### 再オープンフロー（管理画面から）

```
「再オープンする」ボタンをクリック
    ↓
スレッドを再オープン
ステータスを「対応中」に自動変更
入力エリアおよびクローズボタンを再表示
```

## 6. チャットスレッド（フロント）

ログイン済みユーザーが自分のお問い合わせ履歴とスレッドを確認・返信できる画面。

### 6.1 アクセス

| 項目 | 内容 |
|------|------|
| URL | `/contact/history` |
| アクセス条件 | ログイン済みユーザーのみ |
| 導線 | お問い合わせ送信完了画面のリンク、設定ページのリンク |

### 6.2 お問い合わせ履歴一覧

#### 表示項目

| 列 | 内容 |
|----|------|

| 種別 | お問い合わせ種別 |
| ステータス | 未対応・対応中・保留中・対応済みのバッジ表示 |
| 未読インジケーター | 管理者（オーナー・運営者）からの未読メッセージがある場合に表示 |

- 行をクリックするとスレッド詳細画面へ遷移する。
- デフォルトのソート順は受付日時の降順（新しい順）。

### 6.3 スレッド詳細画面

## 7. 通知

### 7.1 管理者（オーナー・運営者）への通知

新規お問い合わせの受信およびユーザーからの返信受信時に、管理者（オーナー・運営者）へ通知を送信する。

#### 通知タイミングと対象

| 通知種別 | タイミング | 通知先 |
|---------|----------|-------|
| 新規お問い合わせ | ユーザーがフォームを送信した時点 | オーナー・運営者全員 |
| ユーザーからの返信 | ユーザーがスレッドにメッセージを送信した時点 | オーナー・運営者全員 |
| ユーザーによる再オープン | ユーザーがスレッドを再オープンした時点 | オーナー・運営者全員 |

#### 通知方法

| 方法 | 内容 |
|------|------|
| メール通知 | Resend経由でメールを送信する |
| 管理画面内お知らせ | `admin_notifications` コレクションに通知レコードを自動生成し、管理画面のベルアイコンに表示する。詳細は[管理_お知らせ管理仕様書](/document/specification/management/管理 お知らせ管理仕様書.md)を参照 |

#### メール通知の内容

| 項目 | 内容 |
|------|------|
| 件名 | 【ゲムコメ】新しいお問い合わせが届きました / 【ゲムコメ】お問い合わせに返信がありました |
| 本文 | 通知種別・受付日時・お問い合わせ種別・詳細画面へのリンク |

#### 管理画面内通知の遷移先

| 通知種別 | 遷移先 |
|---------|-------|
| 新規お問い合わせ | 該当お問い合わせ詳細画面 |
| ユーザーからの返信 | 該当お問い合わせ詳細画面 |
| ユーザーによる再オープン | 該当お問い合わせ詳細画面 |

### 7.2 ユーザーへの通知

管理者（オーナー・運営者）からの返信時に、お問い合わせ送信者へメール通知を送信する。

| 通知種別 | タイミング | 通知先 |
|---------|----------|-------|
| 管理者（オーナー・運営者）からの返信 | 管理者（オーナー・運営者）がスレッドにメッセージを送信した時点 | お問い合わせ送信者のメールアドレス |
| スレッドのクローズ | 管理者（オーナー・運営者）がスレッドをクローズした時点 | お問い合わせ送信者のメールアドレス |

> **補足**: 未ログインユーザーへの通知はメールのみ。スレッドが存在しないため、管理画面内通知は発生しない。

---

## 8. 削除ポリシー

お問い合わせレコードおよびスレッドのメッセージは削除を行わない。クローズ後も履歴として永続的に保持する。
